
# 程序计数器

## 先说结论

**程序计数器（Program Counter Register）** 是一块较小的内存空间，记录当前线程执行的字节码行号，是线程私有的、唯一不会发生 OOM 的区域。

## 深度解析

### 核心特性

| 特性 | 说明 |
|------|------|
| **线程私有** | 每个线程都有独立的程序计数器 |
| **生命周期** | 与线程共存亡 |
| **大小固定** | 约等于一个指针的大小（32/64位系统） |
| **OOM 可能** | 不会！这是唯一不会 OOM 的区域 |

### 作用机制

```java
// 字节码执行示意
0: iload_1        // PC = 0
1: iload_2        // PC = 1
2: iadd           // PC = 2
3: istore_3       // PC = 3
```

- **执行 Java 方法**：记录正在执行的字节码指令地址
- **执行 Native 方法**：计数器值为 undefined（空）
- **分支/循环/跳转**：依赖程序计数器定位目标指令

### 多线程场景

```
Thread-1: PC → 指向字节码位置 N
Thread-2: PC → 指向字节码位置 M
Thread-3: PC → 指向字节码位置 K
         ↓
   各线程独立，互不干扰
```

## 易错点/踩坑

- ❌ 程序计数器不是 Java 数组，而是 JVM 内部的数据结构
- ❌ Native 方法执行时，计数器为空（undefined），不是 0
- ❌ 程序计数器是唯一不会产生 OutOfMemoryError 的区域

## 代码示例

```java
// 通过反射查看程序计数器（实际无法直接访问）
public class PCRDemo {
    public static void main(String[] args) {
        // 字节码层面：PC 寄存器自动维护
        int a = 1;
        int b = 2;
        int c = a + b;
        // 对应字节码的 PC 值自动递增
    }
}
```

```bash
# 查看字节码中的 PC 值
javap -c PCRDemo.class
```
