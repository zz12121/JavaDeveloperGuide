
# 运行时常量池

## Q1：运行时常量池是什么？

**A**：运行时常量池（Runtime Constant Pool）是方法区的一部分，是 Class 文件常量池在运行时的表示。存储编译期生成的字面量和符号引用，在类加载后进入方法区。

---

## Q2：运行时常量池存储哪些内容？

**A**：
- **字面量**：字符串字面量、整数、浮点数常量
- **符号引用**：类的全限定名、字段名、方法名
- **方法引用**：指向方法的符号引用
- **字段引用**：指向字段的符号引用
- **接口引用**：接口方法的符号引用

---

## Q3：运行时常量池和字符串常量池的区别？

**A**：

| 区别 | 运行时常量池 | 字符串常量池 |
|------|-------------|-------------|
| **位置** | 方法区/元空间 | 堆（JDK8+） |
| **内容** | 各种类型的常量 | 仅字符串 |
| **来源** | Class 文件 | 字符串字面量 + intern() |
| **生命周期** | 类加载时创建 | JVM 进程级别 |

---

## Q4：常量池如何工作？

**A**：常量池的工作流程：
1. **编译期**：Java 编译器将字面量和符号引用写入 Class 文件常量池
2. **类加载**：类加载器将 Class 常量池内容加载到方法区的运行时常量池
3. **解析阶段**：JVM 将符号引用解析为直接引用（内存地址）

---

## Q5：String.intern() 对常量池的影响？

**A**：`intern()` 方法：
- 将字符串放入字符串常量池
- 如果池中已有，返回池中的引用
- JDK6 常量池在永久代，JDK7+ 在堆中

```java
String s1 = new String("hello");
String s2 = s1.intern(); // 添加到字符串常量池
String s3 = "hello";
System.out.println(s2 == s3); // true
```

---

## Q6：运行时常量池会 OOM 吗？

**A**：会。JDK7 之前在永久代，JDK8+ 在元空间，都可能 OOM。典型场景：
- 大量动态生成的类（如 CGLIB、反射）
- 大量字符串字面量
- 大量 `String.intern()` 调用
