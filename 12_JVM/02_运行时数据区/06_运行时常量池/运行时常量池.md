
# 运行时常量池

## 先说结论

**运行时常量池（Runtime Constant Pool）** 是方法区的一部分，存储 Class 文件常量池中的内容，包括字面量（字符串、基本类型常量）和符号引用（类/方法的符号引用）。它是**线程共享**的，会抛出 OOM。

## 深度解析

### 常量池分类

```
Class 文件常量池 → 运行时常量池 → 字符串常量池
     (编译期)          (运行期)           (堆内)
```

### 运行时常量池内容

| 类型 | 示例 | 说明 |
|------|------|------|
| **字面量** | `123`, `"hello"`, `3.14` | 直接量 |
| **符号引用** | `java/lang/String` | 类/方法的符号名 |
| **方法引用** | `#2 #3` | 指向CONSTANT_Methodref |
| **字段引用** | `#4 #5` | 指向CONSTANT_Fieldref |
| **接口引用** | `#6` | 指向CONSTANT_InterfaceMethodref |

### 符号引用到直接引用的转换

```
符号引用 → 动态链接 → 直接引用
"com.example.Foo" → 类加载解析 → Foo.class 内存地址
```

### Class 文件常量池结构

```java
// 编译后的常量池
public class Example {
    String s = "hello";      // CONSTANT_String
    int i = 100;            // CONSTANT_Integer
    Object o = new Object(); // CONSTANT_Class
}
```

## 易错点/踩坑

- ❌ 运行时常量池不等于字符串常量池
- ❌ 运行时常量池在 JDK7 之前属于永久代，JDK8+ 属于元空间
- ❌ 字符串常量池 JDK8+ 在堆中，不在方法区
- ❌ 常量池是编译期确定的，但可以运行时动态添加

## 代码示例

```java
// 运行时常量池动态添加
public class RuntimePoolDemo {
    public static void main(String[] args) {
        // String.intern() 动态添加常量到字符串常量池
        String s1 = new String("hello");
        String s2 = s1.intern();
        String s3 = "hello";
        System.out.println(s2 == s3); // true
        
        // Class.forName() 可能触发常量池扩展
        for (int i = 0; i < 10000; i++) {
            Class.forName("Class" + i);
        }
    }
}
```

# 查看类常量池
