
# 本地方法栈

## 先说结论

**本地方法栈（Native Method Stack）** 与 Java 虚拟机栈类似，但专门用于执行 native 方法（JNI）。同样会抛出 `StackOverflowError` 和 `OutOfMemoryError`。

## 深度解析

### 与 Java 虚拟机栈的对比

| 特性 | Java 虚拟机栈 | 本地方法栈 |
|------|---------------|------------|
| **存储内容** | Java 方法字节码 | native 方法（C/C++） |
| **线程可见性** | 线程私有 | 线程私有 |
| **OOM 可能** | 可能 | 可能 |
| **规范要求** | JVM 规范强制实现 | JVM 规范允许不实现 |

### HotSpot 实现

```java
// HotSpot 中本地方法栈和虚拟机栈可以合一实现
// 使用 -Xoss 参数（历史参数，实际无效）
// 使用 -Xss 同时控制两者
```

### Native 方法调用流程

```
Java 方法调用 native 方法
        ↓
JNI 加载本地库（System.loadLibrary）
        ↓
查找本地库中的函数（C/C++ 实现）
        ↓
创建本地栈帧
        ↓
执行本地代码
        ↓
返回结果，栈帧销毁
```

### 常见 native 方法

```java
// Object 类
public native int hashCode();
protected native Object clone();
public final native Class<?> getClass();

// Thread 类
private native void start0();
private native void setPriority0(int newPriority);
private native synchronized void interrupt0();

// 其他
public static native void arraycopy(Object src, ...);
public static native long currentTimeMillis();
```

## 易错点/踩坑

- ❌ 本地方法栈和 Java 虚拟机栈是**两个不同的区域**
- ❌ HotSpot 将两者合并实现，但概念上仍是分离的
- ❌ native 方法不遵守 Java 的异常处理机制
- ❌ 本地方法可以访问 JVM 内部数据结构

## 代码示例

```java
// 定义 native 方法
public class NativeDemo {
    // 声明本地方法（不提供实现）
    public native void nativeMethod();
    
    public static void main(String[] args) {
        // 加载包含 native 实现的城市
        System.loadLibrary("NativeDemo");
        
        NativeDemo demo = new NativeDemo();
        demo.nativeMethod();  // 调用 native 代码
    }
}
```

```c
// NativeDemo.c - 本地实现
#include <jni.h>

JNIEXPORT void JNICALL Java_NativeDemo_nativeMethod(JNIEnv* env, jobject obj) {
    printf("Hello from native code!\n");
}
```
