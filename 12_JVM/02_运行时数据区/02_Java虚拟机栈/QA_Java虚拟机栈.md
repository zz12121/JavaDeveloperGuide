
# Java 虚拟机栈

## Q1：什么是 Java 虚拟机栈？

**A**：Java 虚拟机栈（Java Virtual Machine Stack）是线程私有的内存区域，描述方法执行的内存模型。每个方法调用都会创建一个栈帧（Stack Frame），包含局部变量表、操作数栈、动态链接和方法返回地址。

---

## Q2：栈帧由哪些部分组成？

**A**：栈帧包含四部分：
1. **局部变量表**：存储方法参数和局部变量
2. **操作数栈**：临时的操作区域，类似 CPU 寄存器
3. **动态链接**：将符号引用转换为直接引用
4. **方法返回地址**：方法返回后继续执行的位置

---

## Q3：局部变量表的大小是固定的吗？

**A**：局部变量表的大小在**编译期**确定，存储在 Class 文件的 Code 属性中。运行时大小不变。

- `long` 和 `double` 占 2 个 slot
- 其他基本类型占 1 个 slot
- `this` 在非静态方法中占用 slot 0

---

## Q4：StackOverflowError 是什么原因？

**A**：`StackOverflowError` 的原因：
- **递归调用没有正确终止条件**：每次递归都压栈，栈空间耗尽
- **栈设置过小**：`-Xss` 设置的值太小

```java
// 典型错误
public static int sum(int n) {
    return n + sum(n - 1);  // 无限递归 → StackOverflowError
}
```

---

## Q5：栈会抛出 OOM 吗？

**A**：**会**。当线程请求的栈深度超过 JVM 允许的最大深度时会抛出 `StackOverflowError`；如果 JVM 允许动态扩展栈，但在尝试扩展时无法申请到足够内存，则抛出 `OutOfMemoryError`。

---

## Q6：栈是线程安全的吗？

**A**：栈是**线程私有**的，每个线程都有自己的栈，互不影响，所以**不存在线程安全问题**。需要关注的是多个线程访问共享对象时的线程安全问题。

---

## Q7：`-Xss` 参数设置的是全局还是单个线程？

**A**：`-Xss` 设置的是**单个线程**的栈大小，不是全局的。如果设置 `-Xss256k`，每个新创建的线程都有 256KB 的栈空间。
