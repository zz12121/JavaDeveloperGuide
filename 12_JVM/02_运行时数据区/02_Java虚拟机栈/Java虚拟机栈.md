
# Java 虚拟机栈

## 先说结论

**Java 虚拟机栈（Java Virtual Machine Stack）** 是线程私有的内存区域，存储方法调用的栈帧，每个栈帧包含局部变量表、操作数栈、动态链接和方法返回地址。**StackOverflowError** 和 **OOM** 都可能在这里发生。

## 深度解析

### 栈帧结构

```
┌─────────────────────────────┐
│        栈帧（Stack Frame）    │
├─────────────────────────────┤
│  局部变量表（Local Variables）│  ← 方法参数 + 局部变量
├─────────────────────────────┤
│  操作数栈（Operand Stack）   │  ← 临时操作区域
├─────────────────────────────┤
│  动态链接（Dynamic Linking） │  ← 方法引用解析
├─────────────────────────────┤
│  方法返回地址               │  ← 方法结束后回到哪里
└─────────────────────────────┘
```

### 局部变量表

| 类型 | 占用空间 |
|------|----------|
| byte、short、char、boolean | 1 slot (32位) |
| int、float | 1 slot |
| long、double | 2 slots |
| reference（对象引用） | 1 slot |

### 配置参数

```bash
# 栈大小（单个线程）
-Xss256k      # 默认 1MB
-Xss512k

# 调整示例
java -Xss256k -Xmx512m MyApp
```

### 方法调用流程

```
main() 调用 methodA()
    ↓
main 栈帧入栈
    ↓
methodA 栈帧入栈
    ↓
methodA 调用 methodB()
    ↓
methodA 栈帧暂停，methodB 栈帧入栈
    ↓
methodB 执行完毕返回
    ↓
methodB 栈帧出栈，methodA 继续
    ↓
methodA 执行完毕返回
    ↓
main 栈帧出栈
```

## 易错点/踩坑

- ❌ 栈是线程私有的，不存在线程安全问题
- ❌ `-Xss` 设置的是单个线程的栈大小，不是整个 JVM 的
- ❌ 递归调用没有结束条件会导致 StackOverflowError
- ❌ 局部变量表大小在编译期确定，运行期不变

## 代码示例

```java
public class StackDemo {
    // 局部变量表 slot 计算
    public void method(int a, long b, double c, Object o) {
        // slot 0: this (非静态方法)
        // slot 1: a (int)
        // slot 2-3: b (long，占2个slot)
        // slot 4-5: c (double，占2个slot)
        // slot 6: o (reference)
        
        int x = a + 1;  // 操作数栈操作
    }
    
    // 递归导致 StackOverflowError
    public static int recursive(int n) {
        if (n <= 1) return n;
        return n * recursive(n - 1);  // 没有终止条件会爆栈
    }
}
```

```bash
# 查看栈帧信息
javap -v StackDemo.class | grep -A 20 "Stack map"
```
