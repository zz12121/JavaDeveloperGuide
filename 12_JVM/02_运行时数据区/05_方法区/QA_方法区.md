
# 方法区（Method Area）

## Q1：方法区存储哪些内容？

**A**：方法区存储：
- 类的元信息（类名、修饰符、继承关系）
- 字段信息（字段名、类型、修饰符）
- 方法信息（方法名、参数、字节码）
- 常量（final static 常量）
- 静态变量（static 变量）
- JIT 编译器编译后的代码缓存
- 运行时常量池引用

---

## Q2：JDK7 和 JDK8 的方法区实现有什么区别？

**A**：

| 特性 | JDK7（永久代） | JDK8+（元空间） |
|------|---------------|----------------|
| **位置** | 堆内 | 本地内存（堆外） |
| **大小** | 固定（参数设置） | 动态扩展 |
| **OOM 原因** | 永久代满 | 类太多 |
| **字符串常量池** | 永久代内 | 堆内 |
| **静态变量** | 永久代 | 堆内 |

---

## Q3：为什么 JDK8 要移除永久代？

**A**：
1. **内存溢出问题**：永久代大小固定，容易 OOM
2. **GC 复杂**：Full GC 才会回收永久代
3. **调优困难**：需要同时调优堆和永久代
4. **元空间优势**：使用本地内存，可以动态扩展

---

## Q4：方法区会进行垃圾回收吗？

**A**：会，但回收条件苛刻：
- **类的卸载**：没有实例引用，且 ClassLoader 被回收
- **常量池清理**：没有字符串引用常量池中的常量
- **HotSpot 实现**：元空间 GC 在 Metaspace GC 时发生

---

## Q5：什么情况下会导致方法区 OOM？

**A**：
- **JDK7**：大量类加载（如 CGLIB 动态代理）、字符串常量池过大
- **JDK8+**：大量类加载导致元空间不足
```java
// JDK8 元空间 OOM 示例
while (true) {
    // 使用反射或代理生成大量类
    Class<?> cls = Class.forName("Test" + i);
}
```

---

## Q6：静态变量存储在哪里？

**A**：
- **JDK7 之前**：静态变量存储在永久代
- **JDK8+**：静态变量（对象引用）存储在堆中，因为对象在堆中

**注意**：`static final String` 常量如果是字面量，仍在字符串常量池。
