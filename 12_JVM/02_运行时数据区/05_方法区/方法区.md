
# 方法区（Method Area）

## 先说结论

**方法区（Method Area）** 是 JVM 中的一块逻辑区域，存储类的元信息（类名、修饰符、字段、方法）、常量、静态变量、即时编译器编译后的代码等。**JDK7** 的实现叫永久代，**JDK8+** 改用元空间（Metaspace）。

## 深度解析

### JDK7 vs JDK8+ 对比

| 特性 | JDK7（永久代） | JDK8+（元空间） |
|------|---------------|----------------|
| **位置** | 堆内（JVM 内存） | 堆外（本地内存） |
| **实现** | 固定大小 | 动态扩展 |
| **OOM 原因** | 字符串常量池/静态变量过多 | 类加载过多 |
| **参数** | `-XX:PermSize` `-XX:MaxPermSize` | `-XX:MetaspaceSize` `-XX:MaxMetaspaceSize` |
| **GC** | Full GC 时收集 | 卸载类时收集 |

### 方法区存储内容

```java
// 方法区存储的结构
class Person {
    private String name;      // 实例变量（在堆中）
    private static int count; // 静态变量（在方法区）
    
    public void say() { }     // 方法信息
    public Person() { }       // 构造函数
    static { }               // 静态代码块
}
```

**方法区存储**：
- 类的元信息（类名、修饰符、继承关系）
- 字段信息（字段名、类型、修饰符）
- 方法信息（方法名、参数、字节码）
- 常量（final static 常量）
- 静态变量（static 变量）
- 即时编译器编译后的代码缓存
- 运行时常量池引用

### 配置参数

```bash
# JDK8+ 元空间配置
-XX:MetaspaceSize=256m    # 初始元空间大小
-XX:MaxMetaspaceSize=512m  # 最大元空间大小

# JDK7 永久代配置（已废弃）
-XX:PermSize=128m
-XX:MaxPermSize=256m
```

## 易错点/踩坑

- ❌ JDK8 移除了永久代，但方法区的概念仍然存在
- ❌ 静态变量在 JDK8 中仍存在，但移动到了堆中
- ❌ 字符串常量池 JDK8 移到了堆，但不属于方法区
- ❌ 元空间使用本地内存，不受堆大小限制

## 代码示例

```java
// 模拟方法区 OOM（JDK7）
public class MethodAreaOOM {
    public static void main(String[] args) {
        // JDK7 中，不断生成类会导致 PermGen OOM
        while (true) {
            // CGLIB 动态生成大量类
            Enhancer enhancer = new Enhancer();
            enhancer.setSuperclass(MethodAreaOOM.class);
            enhancer.setUseCache(false);
            enhancer.setCallback(new MethodInterceptor() {
                @Override
                public Object intercept(Object obj, Method method, 
                        Object[] args, MethodProxy proxy) {
                    return proxy.invokeSuper(obj, args);
                }
            });
            enhancer.create();
        }
    }
}
```

