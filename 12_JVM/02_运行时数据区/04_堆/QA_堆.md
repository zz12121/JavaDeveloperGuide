
# 堆（Heap）

## Q1：堆的作用是什么？

**A**：堆是 JVM 中存放**对象实例和数组**的区域，是垃圾回收的主要工作区。JDK8+ 后，字符串常量池也移到了堆中。

---

## Q2：堆内存是如何划分的？

**A**：JDK8+ 堆内存分为：
- **年轻代**（约 1/3）：Eden区 + Survivor S0/S1
- **老年代**（约 2/3）：存放长期存活对象
- 可以通过 `-Xmn`、`-Xms`、`-Xmx` 等参数调整

---

## Q3：对象在堆中的分配流程？

**A**：
1. 对象优先在 **Eden 区**分配
2. Eden 区满时触发 **Minor GC**
3. 存活对象复制到 **Survivor 区**，年龄+1
4. 年龄达到阈值（默认15）后晋升到**老年代**
5. 大对象（超过 Survivor 一半）直接进入老年代

---

## Q4：JDK7 和 JDK8 的堆结构有什么区别？

**A**：
- **JDK7**：堆 = 年轻代 + 老年代 + 永久代，字符串常量池在永久代
- **JDK8**：堆 = 年轻代 + 老年代，永久代被移除改为**元空间**，字符串常量池移到堆中

---

## Q5：如何避免堆溢出？

**A**：
1. 合理设置 `-Xmx` 最大堆大小
2. 使用 `-XX:+HeapDumpOnOutOfMemoryError` 导出堆快照
3. 使用 MAT 等工具分析内存泄漏
4. 避免内存泄漏（缓存未清理、大量静态集合）

---

## Q6：什么是 TLAB？有什么作用？

**A**：TLAB（Thread Local Allocation Buffer）是 Eden 区为每个线程分配的私有缓冲区。

**作用：**
- 避免多线程在 Eden 区分配对象时的竞争
- 减少 CAS 原子操作的开销
- 提高对象分配效率

**工作流程：**
1. 线程优先在自己的 TLAB 中分配对象
2. TLAB 满后，在公共 Eden 区分配（CAS 竞争）
3. 如果 Eden 区也满，触发 Minor GC

---

## Q7：大对象直接进入老年代的阈值是多少？

**A**：大对象阈值由 `-XX:PretenureSizeThreshold` 参数控制。

```bash
# 设置阈值为 1MB
-XX:PretenureSizeThreshold=1048576
```

**注意事项：**
- 单位是字节
- 仅对 Serial、ParNew 收集器有效
- G1、CMS 等收集器会自适应调整
- 超过此大小的数组或大对象直接在老年代分配

---

## Q8：对象晋升老年代的年龄阈值是多少？

**A**：对象晋升老年代的年龄阈值由 `-XX:MaxTenuringThreshold` 参数控制，默认值为 15。

**晋升条件（满足任一即可）：**
1. 对象年龄达到 `MaxTenuringThreshold`（默认 15）
2. Survivor 区相同年龄所有对象大小之和超过 Survivor 区 50%（动态年龄判定）

**动态年龄判定示例：**
```
Survivor 区：age1(30%) + age2(25%) = 55% > 50%
则 age3 及以上对象全部晋升老年代
```

---

## Q9：什么是逃逸分析？有什么好处？

**A**：逃逸分析是 JVM 确定对象作用域的分析技术。

**逃逸类型：**
- **不逃逸**：对象只在方法内使用
- **逃逸**：对象被返回到方法外或被其他线程引用

**优化效果：**
1. **栈上分配**：不逃逸对象在栈上分配，方法结束自动释放
2. **标量替换**：将对象拆解为成员变量直接分配
3. **同步消除**：消除不必要的同步锁

**相关参数：**
```bash
# 启用逃逸分析（默认开启）
-XX:+DoEscapeAnalysis

# 启用标量替换
-XX:+EliminateAllocations
```
