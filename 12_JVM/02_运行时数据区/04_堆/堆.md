
# 堆（Heap）

## 先说结论

**堆**是 JVM 中最大的一块内存区域，存放**所有对象实例和数组**，是垃圾回收的主要工作区。JDK8 及之后，字符串常量池也移到了堆中。

## 深度解析

### 堆内存结构（JDK8+）

| 区域 | 作用 | 特点 |
|------|------|------|
| **Eden 区** | 新对象分配区 | 对象优先在此分配 |
| **Survivor S0/S1** | Minor GC 后存活对象 | 8:1 比例 |
| **Old/Tenured 区** | 长期存活对象 | 大对象直接进入老年代 |

### 堆配置参数

```bash
# 初始堆大小
-Xms256m

# 最大堆大小
-Xmx512m

# 年轻代大小
-Xmn128m

# Eden与Survivor比例
-XX:SurvivorRatio=8
```

### 对象分配流程

```
new 对象 → TLAB/Eden区 → Minor GC → 
    存活 → Survivor → 年龄达到阈值 → Old区
    死亡 → 回收
```

### TLAB（线程本地分配缓冲区）

**什么是 TLAB？**
- TLAB（Thread Local Allocation Buffer）是 Eden 区为每个线程分配的私有缓冲区
- 线程在 TLAB 内分配对象，避免多线程竞争

```
┌─────────────────────────────────────────────────────────┐
│                        Eden 区                           │
├─────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐    │
│  │  Thread-1   │  │  Thread-2   │  │  Thread-3   │    │
│  │   TLAB      │  │   TLAB      │  │   TLAB      │    │
│  │  (私有区域)  │  │  (私有区域)  │  │  (私有区域)  │    │
│  └──────────────┘  └──────────────┘  └──────────────┘    │
│                      公共区域（竞争分配）                    │
└─────────────────────────────────────────────────────────┘
```

**为什么需要 TLAB？**
- 多线程同时在 Eden 区分配对象时，存在竞争
- 竞争导致性能下降（CAS 保证原子性）
- TLAB 让线程优先在私有区域分配，减少竞争

**TLAB 参数配置：**
```bash
# 启用 TLAB（默认开启）
-XX:+UseTLAB

# TLAB 大小
-XX:TLABSize=1024k

# TLAB 动态调整
-XX:ResizeTLAB

# TLAB  refill 阈值（占 TLAB 空间的百分比）
-XX:TLABRefillWasteFraction=64
```

**TLAB 分配失败处理：**
- TLAB 剩余空间不足时，在公共 Eden 区分配
- 使用 CAS 保证线程安全
- 如果eden区也满，触发 Minor GC

### 大对象直接进入老年代

**为什么需要大对象直接进入老年代？**
- 大对象在 Survivor 区之间复制成本高
- 容易导致 Survivor 区空间不足，触发过早晋升

**阈值参数：**
```bash
# 大对象阈值（字节），超过此值直接进入老年代
-XX:PretenureSizeThreshold=1048576  # 1MB

# 注意：此参数仅对 Serial、ParNew 生效
# G1、CMS 等垃圾收集器会自适应调整
```

**典型场景：**
```java
// 大数组直接进入老年代
byte[] largeArray = new byte[10 * 1024 * 1024]; // 10MB
```

### 对象晋升老年代的详细条件

**年龄阈值：**
```bash
# 对象晋升老年代的年龄阈值（默认15）
-XX:MaxTenuringThreshold=15

# 实际晋升年龄由动态计算决定
```

**晋升条件（满足任一即可）：**
1. 对象年龄达到 `MaxTenuringThreshold`
2. Survivor 区相同年龄所有对象大小之和超过 Survivor 区的 50%
3. 动态年龄判定：从小到大累加年龄，达到某年龄时对象总大小超过 Survivor 区 50%，该年龄以上的对象全部晋升

```
Survivor 区：[age1: 30%] [age2: 25%] [age3: 10%] [age4: 5%] ...
累加：30% + 25% = 55% > 50%，则 age3 及以上对象晋升老年代
```

### 逃逸分析与栈上分配

**什么是逃逸分析？**
- JVM 通过逃逸分析确定对象的作用域
- 如果对象只在方法内部使用，不会逃逸到其他线程

**逃逸状态：**
```java
public class EscapeAnalysis {
    
    // 不逃逸：只在方法内使用
    public void method1() {
        byte[] buffer = new byte[1024]; // 栈上分配
    }
    
    // 逃逸：返回到方法外部
    public byte[] method2() {
        return new byte[1024]; // 堆分配
    }
    
    // 逃逸：被其他线程引用
    public static byte[] shared; // 类静态变量
    
    public void method3() {
        shared = new byte[1024]; // 堆分配
    }
}
```

**栈上分配原理：**
```bash
# 启用逃逸分析（默认开启）
-XX:+DoEscapeAnalysis

# 启用标量替换（默认开启）
-XX:+EliminateAllocations
```

**栈上分配 vs 堆分配：**
| 特性 | 栈上分配 | 堆分配 |
|------|----------|--------|
| 分配位置 | 线程栈 | JVM 堆 |
| 释放方式 | 方法结束自动释放 | GC 回收 |
| 分配速度 | 快 | 较慢 |
| 适用场景 | 局部变量、无逃逸对象 | 共享对象、生命周期跨方法 |

**标量替换：**
- 将对象拆解为其成员变量（标量）直接分配在栈上
- 对象本身不在堆中创建

```java
// 标量替换前
public Point createPoint() {
    Point p = new Point(1, 2); // 可能栈上分配
    return p;
}

// 标量替换后（编译器优化）
public int createPoint_x() { return 1; }
public int createPoint_y() { return 2; }
```

## 易错点/踩坑

- ❌ JDK8 移除了永久代（PermGen），改用元空间（Metaspace），字符串常量池移到了堆
- ❌ 对象不一定在 Eden 区分配，大对象可能直接进老年代
- ❌ Survivor 区 S0/S1 永远有一个是空的
- ❌ TLAB 大小不是固定的，JVM 会动态调整
- ❌ 逃逸分析在 Server 模式默认开启，Client 模式可能关闭
- ❌ `-XX:PretenureSizeThreshold` 对 G1 不生效

## 代码示例

```java
// 查看堆内存使用
public class HeapDemo {
    public static void main(String[] args) {
        long heapMax = Runtime.getRuntime().maxMemory(); // 最大堆
        long heapTotal = Runtime.getRuntime().totalMemory(); // 当前堆
        long heapFree = Runtime.getRuntime().freeMemory(); // 空闲堆
        System.out.println("最大堆: " + heapMax / 1024 / 1024 + "MB");
        System.out.println("当前堆: " + heapTotal / 1024 / 1024 + "MB");
        System.out.println("空闲堆: " + heapFree / 1024 / 1024 + "MB");
    }
}
```

```java
// TLAB 效果演示（JVM 参数：-XX:+UseTLAB -XX:TLABSize=512k）
public class TLABDemo {
    // 大量小对象分配，观察 TLAB 行为
    public static void main(String[] args) {
        long start = System.nanoTime();
        Object[] objects = new Object[1_000_000];
        for (int i = 0; i < 1_000_000; i++) {
            objects[i] = new Object(); // 在 TLAB 中分配
        }
        long end = System.nanoTime();
        System.out.println("分配耗时: " + (end - start) / 1_000_000 + "ms");
    }
}
```

```java
// 大对象直接进入老年代演示
// JVM 参数：-XX:PretenureSizeThreshold=1048576
public class LargeObjectDemo {
    public static void main(String[] args) {
        // 这个数组会直接进入老年代（>1MB）
        byte[] largeArray = new byte[2 * 1024 * 1024]; // 2MB
        System.out.println("大对象分配完成");
    }
}
```

```java
// 逃逸分析演示
public class EscapeAnalysisDemo {
    // 添加 JVM 参数观察：-XX:+DoEscapeAnalysis -XX:+EliminateAllocations
    
    // 场景1：不逃逸，可以栈上分配
    public void localAllocation() {
        // 对象不逃逸方法，栈上分配
        byte[] buffer = new byte[1024];
        buffer[0] = 1;
    }
    
    // 场景2：逃逸到堆
    public Object escapeToHeap() {
        // 返回对象引用，逃逸
        return new byte[1024];
    }
    
    // 场景3：被其他线程可见
    public static Object shared = null;
    public void escapeToOtherThread() {
        shared = new byte[1024]; // 逃逸
    }
}
```
