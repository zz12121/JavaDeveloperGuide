
# Class 文件常量池

## 先说结论

**Class 文件常量池**是 .class 文件中的数据结构，存储编译期产生的各种**字面量和符号引用**。在类加载时进入方法区成为**运行时常量池**。

## 深度解析

### 常量池结构

```
┌──────────────────────────────────────────────────────────────────┐
│                    Class 文件常量池                               │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│  constant_pool_count: n+1                                        │
│  constant_pool[1] ~ constant_pool[n]                             │
│                                                                   │
│  ┌────────┬────────┬────────┬────────┬────────┬─────────────┐   │
│  │ #1     │ #2     │ #3     │ #4     │ #5     │ ...         │   │
│  │CONSTANT│CONSTANT│CONSTANT│CONSTANT│CONSTANT│             │   │
│  │_UTF8   │_Integer│_Class  │_String │_Method │             │   │
│  │ "name" │  100   │ #6     │ #7     │ ref     │             │   │
│  └────────┴────────┴────────┴────────┴────────┴─────────────┘   │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 常量池项类型

| 类型 | tag | 说明 |
|------|-----|------|
| CONSTANT_Utf8 | 1 | UTF-8 编码字符串 |
| CONSTANT_Integer | 3 | int 常量 |
| CONSTANT_Float | 4 | float 常量 |
| CONSTANT_Long | 5 | long 常量 |
| CONSTANT_Double | 6 | double 常量 |
| CONSTANT_Class | 7 | 类/接口引用 |
| CONSTANT_String | 8 | 字符串引用 |
| CONSTANT_Fieldref | 9 | 字段引用 |
| CONSTANT_Methodref | 10 | 方法引用 |
| CONSTANT_InterfaceMethodref | 11 | 接口方法引用 |
| CONSTANT_NameAndType | 12 | 名称和类型 |
| CONSTANT_MethodHandle | 15 | 方法句柄 |
| CONSTANT_MethodType | 16 | 方法类型 |
| CONSTANT_InvokeDynamic | 18 | 动态调用 |

### 三种常量池的关系

```
┌──────────────────────────────────────────────────────────────────┐
│                    三种常量池的关系                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│  .class 文件                                                      │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │                 Class 文件常量池                              │ │
│  │   编译期生成，存储字面量和符号引用                            │ │
│  └────────────────────────────────────────────────────────────┘ │
│                              ↓ 类加载                             │
│  方法区 / 元空间                                                   │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │                 运行时常量池                                  │ │
│  │   类加载后生成，存储解析后的直接引用                          │ │
│  └────────────────────────────────────────────────────────────┘ │
│                              ↓                                   │
│  堆                                                               │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │                 字符串常量池                                  │ │
│  │   存储字符串字面量（StringTable）                            │ │
│  └────────────────────────────────────────────────────────────┘ │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

## 易错点/踩坑

- ❌ Class 文件常量池在编译期确定，运行时常量池在类加载时创建
- ❌ 常量池索引从 1 开始（不是 0）
- ❌ 运行时常量池可能动态添加内容（如 String.intern()）
- ❌ 常量池大小影响 class 文件大小

## 代码示例

```java
// 源码
public class Demo {
    String s = "hello";
    int i = 100;
    Object o = new Object();
}
```

```bash
# 查看常量池
javap -v Demo.class | grep -A 100 "Constant pool"

# 输出示例
Constant pool:
   #1 = Methodref          #6.#17  // java/lang/Object."<init>":()V
   #2 = String             #18     // java/lang/String
   #3 = Fieldref           #19.#20 // Demo.s:Ljava/lang/String;
   #4 = Integer            100
   #5 = Utf8               s
   #6 = Class              #21     // java/lang/Object
   ...
```

## 图解/流程

```
常量池类型转换流程

编译期（.class 文件）
CONSTANT_Class → 符号引用 "java/lang/Object"
CONSTANT_String → 符号引用 "hello"

    ↓ 类加载时解析

运行期（方法区）
Class 对象 → 直接引用
String 对象 → 直接引用（字符串常量池）
```
