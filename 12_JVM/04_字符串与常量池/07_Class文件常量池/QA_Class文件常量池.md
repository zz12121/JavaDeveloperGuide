
# Class 文件常量池

## Q1：什么是 Class 文件常量池？

**A**：Class 文件常量池是 .class 文件中的结构，存储编译期产生的：
- **字面量**：字符串、整数、浮点数
- **符号引用**：类、方法、字段的名称和描述符

---

## Q2：Class 文件常量池有哪些类型？

**A**：主要类型：

| 类型 | tag | 说明 |
|------|-----|------|
| CONSTANT_Utf8 | 1 | UTF-8 字符串 |
| CONSTANT_Integer | 3 | int 常量 |
| CONSTANT_Class | 7 | 类引用 |
| CONSTANT_String | 8 | 字符串引用 |
| CONSTANT_Fieldref | 9 | 字段引用 |
| CONSTANT_Methodref | 10 | 方法引用 |
| CONSTANT_NameAndType | 12 | 名称和类型 |

---

## Q3：常量池索引为什么从 1 开始？

**A**：常量池索引从 1 开始是为了：
- **节省空间**：索引 0 有特殊用途（如表示"不引用任何常量"）
- **历史原因**：JVM 规范规定
- **操作便利**：条件跳转指令可以使用 0 表示无效索引

---

## Q4：三种常量池的关系？

**A**：

| 常量池 | 位置 | 生命周期 | 内容 |
|--------|------|----------|------|
| Class 文件常量池 | .class 文件 | 编译期 | 符号引用 |
| 运行时常量池 | 方法区/元空间 | 运行期 | 直接引用 |
| 字符串常量池 | 堆（JDK8+） | JVM 进程 | 字符串实例 |

---

## Q5：符号引用如何转换为直接引用？

**A**：在类加载的**解析阶段**：
```java
// 符号引用："java/lang/Object"
//     ↓ 解析
// 直接引用：Object 类在堆中的内存地址
```

解析时机：
- **静态解析**：类加载时（final、static 等）
- **动态解析**：运行时（多态、泛型）

---

## Q6：常量池会导致 class 文件过大吗？

**A**：会。优化方法：
- 减少长字符串字面量
- 减少重复常量
- 使用混淆工具压缩常量池

---

## Q7：常量池会抛出 OOM 吗？

**A**：Class 文件常量池不会 OOM，但运行时常量池会：
- **JDK7 之前**：运行时常量池在永久代，可能 OOM
- **JDK8+**：运行时常量池在元空间，可能 OOM
