
# 字符串拼接优化

## 先说结论

Java 字符串拼接有两种优化：**编译期常量折叠**（字面量拼接）和**运行时 StringBuilder**（变量拼接）。理解这两种机制是面试高频考点。

## 深度解析

### 编译期常量折叠

```java
// 编译前
String s = "a" + "b" + "c";

// 编译后（字节码）
String s = "abc";  // 编译器直接优化为字面量
```

**触发条件**：
- 拼接两侧都是**编译期常量**
- 常量包括：字面量、final 变量、基本类型

### 运行时 StringBuilder 优化

```java
// 编译前
String a = "hello";
String s = a + "world";

// 编译后（字节码）
String s = new StringBuilder().append(a).append("world").toString();
```

**触发条件**：
- 拼接包含**变量**（非 final）
- JDK5+ 使用 StringBuilder
- JDK4 使用 StringBuffer（synchronized）

### 不同拼接方式对比

| 拼接方式 | 示例 | 优化类型 | 对象数 |
|----------|------|----------|--------|
| 字面量拼接 | `"a" + "b"` | 编译期 | 0 |
| final 拼接 | `final a="a"; a + "b"` | 编译期 | 0 |
| 变量拼接 | `a + "b"` | 运行时 StringBuilder | 3+ |
| StringBuilder | `sb.append().append()` | 运行时 | 1 |
| concat() | `"a".concat("b")` | 运行时 | 1-2 |

### 循环拼接优化

```java
// 低效：循环内创建 StringBuilder
String result = "";
for (int i = 0; i < 100; i++) {
    result += "a";  // 每次循环都创建新的 StringBuilder
}

// 高效：循环外创建 StringBuilder
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 100; i++) {
    sb.append("a");
}
String result = sb.toString();
```

## 易错点/踩坑

- ❌ 变量拼接不等于 StringBuilder（在某些情况下 JIT 可能进一步优化）
- ❌ 循环内拼接不要用 `+=`，会产生大量中间对象
- ❌ `"a" + "b"` 编译期优化，`new StringBuilder()` 运行时创建
- ❌ StringBuilder 不是线程安全的

## 代码示例

```java
public class StringConcatDemo {
    public static void main(String[] args) {
        // 1. 编译期常量折叠
        String s1 = "a" + "b";           // → "ab"
        String s2 = "ab";                // 同一对象
        System.out.println(s1 == s2);    // true
        
        // 2. final 变量优化
        final String a = "hello";
        String s3 = a + "world";         // → "helloworld"
        
        // 3. 变量拼接
        String b = "hello";
        String s4 = b + "world";         // StringBuilder
        
        // 4. 字符串 concat
        String s5 = "a".concat("b");     // 内部也是 new String()
    }
}
```
