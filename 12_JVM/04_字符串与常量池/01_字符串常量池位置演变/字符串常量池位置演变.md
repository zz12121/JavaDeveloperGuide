
# 字符串常量池位置演变

## 先说结论

字符串常量池位置经历了重大变化：**JDK6 在永久代** → **JDK7 移到堆中** → **JDK8 彻底移除永久代**。这是面试高频考点。

## 深度解析

### 位置演变

```
JDK6：永久代（PermGen）
┌──────────────────────────────────────────────────────┐
│  JVM 堆                                                │
│  ┌──────────────┬──────────────────────────────────┐ │
│  │  年轻代       │            老年代                  │ │
│  └──────────────┴──────────────────────────────────┘ │
│  ┌──────────────────────────────────────────────────┐│
│  │              永久代（PermGen）                     ││
│  │  - 类元信息                                        ││
│  │  - 字符串常量池 ← 重点！                           ││
│  │  - 静态变量                                        ││
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘

JDK7：堆中
┌──────────────────────────────────────────────────────┐
│  JVM 堆                                                │
│  ┌──────────────┬──────────────────────────────────┐ │
│  │  年轻代       │            老年代                  │ │
│  └──────────────┴──────────────────────────────────┘ │
│  ┌──────────────────────────────────────────────────┐│
│  │              字符串常量池 ← 移到堆中！             ││
│  │  - 类元信息（永久代移除）                          ││
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘

JDK8+：元空间
┌──────────────────────────────────────────────────────┐
│  JVM 堆                                                │
│  ┌──────────────┬──────────────────────────────────┐ │
│  │  年轻代       │            老年代                  │ │
│  └──────────────┴──────────────────────────────────┘ │
│  ┌──────────────────────────────────────────────────┐│
│  │              字符串常量池（仍在堆中）              ││
│  └──────────────────────────────────────────────────┘│
│                                                          │
│  本地内存：元空间（Metaspace）                            │
│  ┌──────────────────────────────────────────────────┐│
│  │  - 类元信息                                        ││
│  │  - JIT 代码                                        ││
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

### 关键变化

| JDK 版本 | 字符串常量池位置 | 永久代 | 影响 |
|----------|----------------|--------|------|
| JDK6 | 永久代 | 有 | 容易 OOM |
| JDK7 | 堆中 | 有（但字符串池移出） | 更安全 |
| JDK8+ | 堆中 | 无（元空间替代） | 完全分离 |

### 为什么要移动？

1. **永久代 OOM**：字符串常量池和类加载过多导致 PermGen OOM
2. **调优复杂**：需同时调优堆和永久代
3. **GC 效率低**：永久代 Full GC 才回收
4. **内存浪费**：字符串和类在同一区域

## 易错点/踩坑

- ❌ JDK8 移除了永久代，但字符串常量池仍在堆中
- ❌ 字符串常量池和运行时常量池是不同概念
- ❌ `-XX:PermSize` 参数 JDK8 已废弃
- ❌ 元空间使用本地内存，不受 -Xmx 限制

## 代码示例

```java
// 测试字符串常量池位置（JDK6 vs JDK7+）
public class StringPoolLocation {
    public static void main(String[] args) {
        // 字符串拼接产生新对象
        String s1 = new String("a") + new String("b");
        
        // intern() 放入常量池
        s1.intern();
        
        // 字面量 "ab" 直接从常量池获取
        String s2 = "ab";
        
        // JDK7+: true, JDK6: false
        System.out.println(s1 == s2);
    }
}
```

```bash
# JDK6 参数（已废弃）
java -XX:PermSize=128m -XX:MaxPermSize=256m

# JDK8+ 参数
java -XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=256m
```
