
# 直接指针访问

## 先说结论

**直接指针访问**是 HotSpot 默认的对象访问方式。reference 直接指向堆中对象，对象头中包含类型指针。优点是访问速度快（一次寻址），缺点是 GC 移动对象时需要更新所有 reference。

## 深度解析

### 直接指针访问原理

```
┌──────────────────────────────────────────────────────────────────┐
│                      直接指针访问方式                              │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│  栈帧（本地变量表）                                                │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ reference ────────────────────────────────────────────────┼─┼─│
│  │               ↓                                             │ │  │
│  └─────────────────────────────────────────────────────────────┘ │  │
│                          │                                       │  │
│                          ↓                                       │  │
│  ┌─────────────────────────────────────────────────────────────┐ │  │
│  │                         堆（Heap）                           │ │  │
│  │  ┌──────────────────────────────────────────────────────┐  │ │  │
│  │  │                 对象（Object）                       │  │ │  │
│  │  │  ┌────────────────┬─────────────────────────────────┐ │  │ │  │
│  │  │  │  Mark Word     │ Class Pointer ────────────────┐ │  │ │  │
│  │  │  │  (8 bytes)     │  (4 bytes)                 │   │  │ │  │
│  │  │  ├────────────────┴───────────────────────────────┼─┤  │ │  │
│  │  │  │              实例数据                          │  │  │ │  │
│  │  │  └────────────────────────────────────────────────┘  │ │  │
│  │  └──────────────────────────────────────────────────────┘  │ │  │
│  └─────────────────────────────────────────────────────────────┘ │  │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 访问流程

```
1. 读取 reference（栈帧）
       ↓
2. reference 直接指向对象地址
       ↓
3. 通过 ClassPointer 访问类型信息
       ↓
4. 访问对象字段/方法
```

### 与句柄访问对比

| 特性 | 直接指针 | 句柄访问 |
|------|----------|----------|
| **reference 指向** | 对象本身 | 句柄 |
| **寻址次数** | 1 次 | 2 次 |
| **访问速度** | 快 | 慢 |
| **GC 移动对象** | 需更新所有 reference | 只需更新句柄 |
| **稳定性** | 较低 | 高 |
| **代表 JVM** | HotSpot | Classic VM |

## 易错点/踩坑

- ❌ HotSpot 默认使用直接指针，不是句柄
- ❌ 直接指针访问比句柄快，但 GC 压力大
- ❌ 逃逸分析后对象可能栈上分配，没有 reference
- ❌ 关闭压缩指针会影响对象头和引用大小

## 代码示例

```java
public class DirectPointerDemo {
    public static void main(String[] args) {
        Object obj = new Object();
        
        // JVM 内部：
        // reference 直接指向堆中对象
        // 无需经过句柄池
        
        // 访问 obj 的类信息
        Class<?> cls = obj.getClass();
        // 通过对象的 ClassPointer 访问
    }
}
```
