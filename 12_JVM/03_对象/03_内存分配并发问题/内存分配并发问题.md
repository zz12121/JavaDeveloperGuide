
# 内存分配并发问题

## 先说结论

多线程同时分配内存时存在并发问题，JVM 通过 **CAS + 失败重试** 或 **TLAB** 解决。TLAB 效率更高，是目前主流方案。

## 深度解析

### 并发分配问题

```
Thread-1: 读取指针位置 100
Thread-2: 读取指针位置 100  ← 同时读取
Thread-1: 分配对象，指针移动到 120
Thread-2: 分配对象，指针移动到 140  ← 覆盖 Thread-1 的对象！
```

### 解决方案一：CAS + 失败重试

```java
// 伪代码
do {
    // 读取当前指针
    memoryPointer = oldPointer;
    // 计算新指针位置
    newPointer = memoryPointer + objectSize;
} while (!compareAndSet(memoryPointer, newPointer));
// 失败则重试
```

| 特性 | 说明 |
|------|------|
| **原理** | 乐观锁 + 失败重试 |
| **缺点** | 高并发时重试频繁，CPU 消耗高 |
| **适用** | 分配不频繁的场景 |

### 解决方案二：TLAB

```java
// 每个线程有独立的 TLAB
Thread-1 TLAB: [====分配====][剩余空间]
Thread-2 TLAB: [====分配====][剩余空间]
Thread-3 TLAB: [====分配====][剩余空间]
```

| 特性 | 说明 |
|------|------|
| **原理** | 线程私有缓冲区，无需同步 |
| **位置** | Eden 区 |
| **大小** | 默认约 1% Eden 区 |
| **优点** | 分配速度快，无竞争 |
| **缺点** | 空间利用率略低 |

### TLAB 分配流程

```
new 对象
    ↓
检查 TLAB 剩余空间
    ↓
┌─────────────────────────────────┐
│  剩余足够？                      │
│  ├── YES → TLAB 内分配，结束    │
│  └── NO  → 重新分配 TLAB        │
│          ↓                      │
│     TLAB 分配成功？             │
│     ├── YES → TLAB 内分配       │
│     └── NO  → CAS 同步分配      │
└─────────────────────────────────┘
```

## 易错点/踩坑

- ❌ TLAB 分配失败不一定立即重试
- ❌ TLAB 大小不合适会影响分配效率
- ❌ `-XX:UseTLAB` 在某些 JVM 版本默认关闭
- ❌ TLAB 剩余空间可能产生内存碎片

## 代码示例

```java
// 查看 TLAB 信息
public class TLABDemo {
    public static void main(String[] args) {
        // 分配大量小对象，观察 TLAB 行为
        for (int i = 0; i < 100000; i++) {
            new Object();
        }
    }
}
```

```bash
# JVM 参数
-XX:+PrintTLAB          # 打印 TLAB 信息
-XX:TLABSize=64k        # 设置 TLAB 大小
-XX:-UseTLAB            # 禁用 TLAB
-XX:TLABRefillWasteFraction=100  # TLAB 回收阈值
```

```bash
# 输出示例
TLAB: gc thread: 0x00007f8d2c01d800 [id: 12]
      desired_size: 62KB + 2KB = 64KB
      refill waste: 992B
      slow delta: 3KB
      allocation fraction: 0.128
```
