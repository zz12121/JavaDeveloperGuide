
# 内存分配方式

## 先说结论

对象内存分配有两种方式：**指针碰撞**（Bump the Pointer）和**空闲列表**（Free List）。具体使用哪种取决于 GC 算法的堆内存是否规整。线程安全通过 **CAS + 失败重试** 或 **TLAB** 保证。

## 深度解析

### 指针碰撞（Bump the Pointer）

```
┌────────────┬────────────────────────────┐
│ 已分配区域  │         空闲区域            │
└────────────┴────────────────────────────┘
            ↑ 指针位置
            │
  分配对象 → 指针移动 → 对象放入
```

- **适用场景**：堆内存规整（标记-复制、标记-整理算法）
- **GC 算法**：Serial、ParNew
- **优点**：分配速度快
- **缺点**：需要内存规整

### 空闲列表（Free List）

```
┌────┐    ┌──────┐   ┌──────┐   ┌────┐
│████│    │████  │   │████  │   │████│
└────┘    └──────┘   └──────┘   └────┘
  已用     空闲列表： [20KB] [30KB] [15KB]
```

- **适用场景**：堆内存不规整（标记-清除算法）
- **GC 算法**：CMS、标记-清除
- **优点**：适应不规整内存
- **缺点**：分配速度较慢

### TLAB（Thread Local Allocation Buffer）

```java
// TLAB 工作原理
每个线程在 Eden 区预分配一小块内存
  ↓
线程在自己的 TLAB 中分配对象
  ↓
TLAB 满了 → 重新分配 TLAB 或使用同步方式
```

| 特性 | 说明 |
|------|------|
| **启用参数** | `-XX:+UseTLAB` |
| **默认启用** | HotSpot 默认启用 |
| **TLAB 大小** | `-XX:TLABSize` |
| **分配方式** | 线程私有，无需同步 |
| **比例** | 约 1% 的 Eden 区 |

## 易错点/踩坑

- ❌ 指针碰撞不适用于标记-清除算法
- ❌ TLAB 分配失败会回退到同步分配
- ❌ 逃逸分析后可栈上分配，不走堆分配
- ❌ 大对象可能直接进入老年代

## 代码示例

```java
public class AllocationDemo {
    public static void main(String[] args) {
        // 小对象：优先在 Eden 区分配
        Object small = new Object();
        
        // 大对象：直接进入老年代
        // -XX:PretenureSizeThreshold=1000000 (约1MB)
        byte[] large = new byte[2 * 1024 * 1024]; // 2MB
        
        // TLAB 查看（需要 JVM 参数）
        // -XX:+PrintTLAB
    }
}
```

```bash
# JVM 参数
-Xms256m -Xmx256m -XX:+UseTLAB -XX:TLABSize=64k -XX:+PrintTLAB
```

## 图解/流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    对象内存分配流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  new Object()                                                    │
│       ↓                                                          │
│  检查类加载（未加载则先加载）                                       │
│       ↓                                                          │
│  逃逸分析（未逃逸？→ 栈上分配）                                     │
│       ↓                                                          │
│  计算对象大小                                                     │
│       ↓                                                          │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  TLAB 分配？                                               │  │
│  │  ├── YES → TLAB 内分配 → 完成                              │  │
│  │  └── NO  → CAS 同步分配 / 空闲列表分配 → 完成               │  │
│  └───────────────────────────────────────────────────────────┘  │
│       ↓                                                          │
│  对象头设置 → 零值初始化 → <init> 执行                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
