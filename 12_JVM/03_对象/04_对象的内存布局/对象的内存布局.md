
# 对象内存布局

## 先说结论

对象在堆中的内存布局分为三部分：**对象头（Header）**、**实例数据（Instance Data）**、**对齐填充（Padding）**。对象头包含 Mark Word 和 Class Pointer，数组还有长度字段。

## 深度解析

### 64位 JVM 对象布局

```
┌────────────────────────────────────────────┐
│                对象内存布局                  │
├────────────────┬───────────────────────────┤
│   对象头        │      实例数据              │
│   (Header)     │   (Instance Data)          │
├────────────────┼───────────────────────────┤
│ Mark Word (64位)│  父类字段                  │
│ Class Pointer  │  子类字段                   │
│ 数组长度(可选)  │  ...                       │
├────────────────┴───────────────────────────┤
│              对齐填充 (Padding)              │
└────────────────────────────────────────────┘
```

### 对象头结构（64位）

| 组成部分 | 大小 | 说明 |
|----------|------|------|
| **Mark Word** | 8 bytes | 存储对象哈希码、GC 年龄、锁状态 |
| **Class Pointer** | 4/8 bytes | 指向类元数据的指针（开启压缩4字节） |
| **数组长度** | 4 bytes | 仅数组对象有 |

### Mark Word 内容（64位）

| 锁状态 | 存储内容（25bit） | GC 年龄（4bit） | 锁标志位（4bit） |
|--------|------------------|-----------------|------------------|
| 无锁 | 对象哈希码 | 年龄 | 01 |
| 偏向锁 | 线程ID + epoch | 年龄 | 01 |
| 轻量级锁 | 指向栈中锁记录的指针 | - | 00 |
| 重量级锁 | 指向互斥量的指针 | - | 10 |
| GC 标记 | 空 | - | 11 |

### 实例数据

```java
public class User {
    long id;        // 8 bytes
    int age;        // 4 bytes
    String name;    // 4/8 bytes (reference)
    boolean vip;    // 1 byte（实际4字节）
    // 对齐填充到8字节边界
}
```

### 对齐填充

- **作用**：保证对象大小是 8 的倍数
- **原理**：CPU 按 8 字节访问效率最高
- **规则**：实例数据 → 对齐填充 → 使总大小为 8 的倍数

## 易错点/踩坑

- ❌ 对象大小不是简单相加，需要考虑对齐填充
- ❌ 压缩指针（-XX:+UseCompressedOops）可以节省 4 字节
- ❌ 数组对象比普通对象多 4 字节存储长度
- ❌ 继承时父类字段在子类字段之前

## 代码示例

```java

import java.lang.instrument.Instrumentation;

public class ObjectSizeDemo {
    static Instrumentation inst;
    
    public static void premain(String args, Instrumentation instP) {
```

# 对象头 MarkWord

## 先说结论

**Mark Word** 是对象头的核心部分（64位占 8 bytes），存储对象的**哈希码、GC 年龄、锁状态**等信息。它是一个动态数据结构，根据对象状态存储不同内容。

## 深度解析

### 64位 Mark Word 结构

```
┌──────────────────────────────────────────────────────────────────┐
│                        64 位 Mark Word                            │
├────────────────────────┬─────────────────┬──────────────────────┤
│        25 bit          │     4 bit       │        1 bit        │
│   哈希码/线程ID/分代年龄  │    分代年龄     │     是否偏向锁       │
│                        │   (GC age)      │    (biased_lock)     │
├────────────────────────┴─────────────────┴──────────────────────┤
│                          2 bit   │         4 bit                │
│                          锁标志位 │    保留位（针对不同锁状态）   │
└──────────────────────────────────────────────────────────────────┘
```

### 不同状态下的存储内容

|锁状态|Mark Word 内容|锁标志位|
|---|---|---|
|**无锁**|对象哈希码(25bit) + GC年龄(4bit) + 锁标志|01|
|**偏向锁**|线程ID(54bit) + epoch(2bit) + GC年龄(4bit) + 锁标志|01|
|**轻量级锁**|指向栈中锁记录的指针(62bit) + 锁标志|00|
|**重量级锁**|指向 Monitor 的指针(62bit) + 锁标志|10|
|**GC 标记**|空(62bit) + 锁标志|11|

### 偏向锁原理

```java
// 偏向锁开启参数
-XX:+UseBiasedLocking  // JDK6+ 默认开启

// 第一次获取锁
// Mark Word 记录线程ID，以后该线程进入不再需要CAS
Thread-1: [thread_id=1234][epoch=0][age=0][01] ← 偏向Thread-1
Thread-2: 进入同步块 → 偏向撤销 → 轻量级锁升级
```

### 锁升级流程

```
无锁状态
    ↓ (CAS 偏向某线程)
偏向锁（单线程，轻量级开销）
    ↓ (其他线程竞争)
轻量级锁（自旋，线程忙等）
    ↓ (自旋超过阈值 / 线程数过多)
重量级锁（系统级Mutex，线程阻塞）
```

## 易错点/踩坑

- ❌ Mark Word 是紧凑的，所有数据压在一个字中
- ❌ 偏向锁在多线程竞争时会撤销，消耗性能
- ❌ 偏向锁撤销发生在安全点（STW）
- ❌ hashCode() 会导致偏向锁撤销

## 代码示例

```java
public class MarkWordDemo {
    public static void main(String[] args) {
        Object obj = new Object();
        
        // 未锁定状态：Mark Word 存储哈希码
        System.out.println("Before hashCode: " + System.identityHashCode(obj));
        obj.hashCode();
        
        // hashCode() 导致偏向锁撤销
        // 撤销后 Mark Word 不再存储偏向信息
        
        // synchronize 后进入锁定状态
        synchronized (obj) {
            // Mark Word 变为轻量级锁或重量级锁
        }
    }
}
```

# 对象头 ClassPointer

## 先说结论

**Class Pointer（类指针）** 是对象头的一部分，指向方法区中类的元数据（Klass 元对象）。通过它 JVM 可以确定对象的类型。64位 JVM 可开启压缩指针（CompressedOops）节省空间。

## 深度解析

### 类指针位置

```
┌─────────────────────────────────────────────┐
│                对象头 (Header)               │
├─────────────────┬───────────────────────────┤
│   Mark Word     │     Class Pointer         │
│   (8 bytes)     │     (4/8 bytes)           │
└─────────────────┴───────────────────────────┘
                           ↓
              指向方法区类元数据（Klass）
```

### 类指针作用

```java
Object obj = new Object();

// obj 的 ClassPointer 指向方法区的 Object Klass
// 通过 ClassPointer 可以：
// 1. 判断对象类型（ instanceof ）
// 2. 访问类的方法、字段
// 3. 动态代理、方法分发
```

### 压缩指针（Compressed Oops）

|模式|指针大小|可寻址范围|
|---|---|---|
|**未压缩**|8 bytes|2^64（约 16EB）|
|**压缩**|4 bytes|2^32 × 8 = 32GB|

```bash
# 开启压缩指针（JDK7+ 64位默认开启）
-XX:+UseCompressedOops

# 关闭压缩指针
-XX:-UseCompressedOops
```

### 压缩原理

```
普通指针：    0x0000007b8a0c1234  →  直接地址
压缩指针：    0x12345678          →  地址 = 压缩值 × 8
```

- 压缩指针通过**乘 8**恢复实际地址
- 要求堆起始地址是 8 的倍数（通常是）

## 易错点/踩坑

- ❌ 类指针指向的是 **Klass 元对象**，不是 Class 类
- ❌ 压缩指针最大支持约 32GB 堆
- ❌ -XX:+UseCompressedClassPointers 控制类指针压缩
- ❌ 类指针大小影响对象头大小

## 代码示例

```java
public class ClassPointerDemo {
    public static void main(String[] args) {
        Object obj = new Object();
        
        // 获取对象头信息
        // ClassPointer 可以通过反射获取类的元数据
        
        System.out.println(obj.getClass().getName());
        // Object
        
        // instanceof 检查
        System.out.println(obj instanceof Object); // true
        System.out.println(obj instanceof String); // false
    }
}
```
