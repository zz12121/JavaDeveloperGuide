
# new 对象完整流程

## 先说结论

new 对象经历 **类加载检查 → 分配内存 → 初始化零值 → 设置对象头 → 执行 init** 五个步骤。

## 深度解析

### 对象创建流程

```
1. 类加载检查
   ↓
2. 分配内存（指针碰撞/空闲列表 + TLAB）
   ↓
3. 初始化零值（零-fill）
   ↓
4. 设置对象头（Mark Word + Class Pointer）
   ↓
5. 执行 <init>（构造函数）
```

### 内存分配方式

| 方式 | 原理 | 适用场景 |
|------|------|----------|
| **指针碰撞** | 堆内存规整，用指针分隔已用/空闲 | Serial、ParNew |
| **空闲列表** | 堆内存碎片，维护空闲块列表 | CMS、标记-清除 |

### TLAB（Thread Local Allocation Buffer）

- 每个线程在 Eden 区预分配一小块内存
- 线程在 TLAB 内分配，无需同步
- `-XX:+UseTLAB` 启用，`-XX:TLABSize` 设置大小

## 易错点/踩坑

- ❌ new 出来的对象不一定在堆上，可能经过逃逸分析后在栈上分配
- ❌ 对象创建前一定会类加载检查，未加载则触发加载
- ❌ TLAB 分配失败时，需要同步方式分配

## 代码示例

```java
// 编译后的字节码
public User(java.lang.String);
    descriptor: (Ljava/lang/String;)V
    flags: ACC_PUBLIC
    Code:
      stack=2, locals=2, args_size=2
         0: aload_0
         1: invokespecial #1  // Object.<init>
         4: aload_1
         5: putfield      #2  // name 赋值
         8: return
```

## 图解/流程

```
┌──────────────────────────────────────┐
│        new 对象完整流程               │
├──────────────────────────────────────┤
│ 1. 类加载检查                         │
│    - 是否已加载？未加载则触发加载     │
│         ↓                            │
│ 2. 分配内存                           │
│    - 指针碰撞 / 空闲列表               │
│    - TLAB 加速分配                    │
│         ↓                            │
│ 3. 初始化零值                        │
│    - 所有字段赋默认值                  │
│    - 零/null/false                   │
│         ↓                            │
│ 4. 设置对象头                         │
│    - Mark Word + Class Pointer        │
│         ↓                            │
│ 5. 执行 <init>                        │
│    - 构造函数、代码块初始化           │
└──────────────────────────────────────┘
```
