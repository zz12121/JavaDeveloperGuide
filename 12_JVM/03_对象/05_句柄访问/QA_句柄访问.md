
# 句柄访问

## Q1：什么是句柄访问？

**A**：句柄访问是一种对象访问方式，通过**句柄池**中间层访问对象：
- reference 指向句柄
- 句柄包含两个指针：对象实例指针 + 类型数据指针
- 通过句柄间接访问对象

---

## Q2：句柄的结构是什么？

**A**：句柄包含两部分：
| 组成部分 | 指向 | 作用 |
|----------|------|------|
| **对象实例指针** | 堆中对象 | 指向对象实际数据 |
| **类型数据指针** | 方法区Klass | 指向类元信息 |

---

## Q3：句柄访问的优缺点？

**A**：

**优点**：
- GC 移动对象时，只需修改句柄中的指针
- reference 本身不需要更新，稳定性高

**缺点**：
- 多一次间接寻址，访问速度较慢
- 需要额外的句柄池空间

---

## Q4：HotSpot 使用哪种访问方式？

**A**：HotSpot 默认使用**直接指针访问**，不是句柄访问。原因是：
- 直接指针访问速度更快（少一次寻址）
- 现代 GC 算法（如 G1）处理引用更新效率高

---

## Q5：什么情况下使用句柄访问？

**A**：句柄访问适用于：
- **早期 JVM**：如 Classic VM
- **需要频繁移动对象的 GC**：如标记-整理
- **追求 reference 稳定性**：避免频繁更新 reference

---

## Q6：句柄访问的完整流程？

**A**：
1. 从栈帧读取 reference
2. 通过 reference 找到句柄
3. 从句柄获取对象实例指针
4. 访问对象的字段或方法
5. 如需类型信息，从句柄获取类型数据指针

---

## Q7：直接指针和句柄如何选择？

**A**：

| 场景 | 推荐方式 |
|------|----------|
| **一般情况** | 直接指针（HotSpot 默认） |
| **GC 频繁整理** | 句柄（减少 reference 更新） |
| **追求极致性能** | 直接指针 |
