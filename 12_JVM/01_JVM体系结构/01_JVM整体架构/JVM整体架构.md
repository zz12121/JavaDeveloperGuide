
# JVM 整体架构

## 先说结论

JVM 主要由**类加载器**、**运行时数据区**、**执行引擎**、**本地接口**四部分组成，负责将字节码加载到内存、分配资源、执行指令。

## 深度解析

### 核心组件

| 组件 | 职责 |
|------|------|
| **ClassLoader（类加载器）** | 负责从文件系统/网络加载 .class 文件，生成 Class 对象 |
| **Runtime Data Area（运行时数据区）** | 内存区域，包括堆、栈、方法区等 |
| **Execution Engine（执行引擎）** | 解释器 + JIT 编译器，将字节码转换为机器码执行 |
| **Native Interface（本地接口）** | 调用本地方法（C/C++），连接 native 库 |

### 执行流程

```
.java源文件 → javac编译 → .class字节码 
    ↓
ClassLoader加载 → 运行时数据区分配内存
    ↓
执行引擎：解释器/JIT → 机器码 → CPU执行
```

## 易错点/踩坑

- ❌ JVM 不是单一的内存区域，而是一个完整的运行时容器
- ❌ 执行引擎不只有解释器，还有 JIT 编译器（热点代码优化）

## 代码示例

```bash
# 查看 JVM 启动时的类加载信息
java -verbose:class YourClass
```
