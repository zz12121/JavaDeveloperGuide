
# JDK / JRE / JVM 区别

## 先说结论

**JDK**（Java Development Kit）= JRE + 开发工具（javac、jdb、jar等），是开发环境。
**JRE**（Java Runtime Environment）= JVM + Java SE API（核心类库），是运行环境。
**JVM**（Java Virtual Machine）= 负责执行字节码的虚拟机，跨平台的核心。
简单说：**JDK > JRE > JVM**，开发用 JDK，运行用 JRE，执行靠 JVM。

## 深度解析

### 三者关系图

```
+--------------------------------------------------+
|                    JDK                             |
|  +----------------------------------------+      |
|  |               JRE                        |      |
|  |  +-------------------------------+     |      |
|  |  |            JVM                 |     |      |
|  |  |  +----------+ +----------+     |     |      |
|  |  |  |  类加载器  | | 执行引擎  |     |     |      |
|  |  |  +----------+ +----------+     |     |      |
|  |  |  +-----------------------+   |     |      |
|  |  |  |    运行时数据区          |   |     |      |
|  |  |  +-----------------------+   |     |      |
|  |  +-------------------------------+     |      |
|  |  +-------------------------------+     |      |
|  |  |  Java SE 核心类库 (rt.jar)   |     |      |
|  |  +-------------------------------+     |      |
|  +----------------------------------------+      |
|  +----------------------------------------+     |
|  |  开发工具 (javac/jdb/jar/javadoc)       |     |
|  +----------------------------------------+     |
+--------------------------------------------------+
```

### 对比表

| 特性 | JDK | JRE | JVM |
|------|-----|-----|-----|
| **全称** | Java Development Kit | Java Runtime Environment | Java Virtual Machine |
| **定位** | 开发工具包 | 运行时环境 | 虚拟机 |
| **包含** | JRE + 开发工具 | JVM + 核心类库 | 类加载器+执行引擎+内存区 |
| **能否编译** | 能（javac） | 不能 | 不能 |
| **能否运行** | 能 | 能 | 能（通过java命令） |
| **能否开发** | 能 | 不能 | 不能 |

### 版本变化注意点

- **JDK 11 起**：不再提供单独的 JRE 下载
- **JDK 9 起**：模块化系统，rt.jar 拆分为多个模块
- **Oracle JDK 8**：商用需付费许可（2019年1月起）

## 易错点/踩坑

- **JDK 包含 JRE，不是并列关系**
- **JVM 是 JRE 的一部分，不是独立产品**
- **JRE 不包含 javac 等开发工具**
- 生产服务器通常只安装 JRE
