
# JVM运行流程

## Q1：从 .java 到执行的完整流程是什么？

**A**：三大阶段：

1. **编译阶段**：`javac` 将 `.java` 编译为 `.class` 字节码（平台无关）
2. **类加载阶段**：ClassLoading -> Linking（验证+准备+解析） -> Initialization（`<clinit>`）
3. **执行阶段**：执行引擎将字节码转为机器码（解释器逐行执行 + JIT 编译热点代码）

---

## Q2：类加载的五个阶段分别做什么？

**A**：

| 阶段 | 做什么 |
|------|--------|
| **加载** | 读取 .class 二进制流 -> 生成 Class 对象 |
| **验证** | 校验字节码格式正确性 |
| **准备** | 为 static 变量分配内存并赋**默认值** |
| **解析** | 符号引用替换为直接引用 |
| **初始化** | 执行 `<clinit>()`，赋**程序员指定值** |

---

## Q3：准备阶段和初始化阶段的区别？

**A**：赋值时机不同。准备阶段给 static 变量赋**默认值**（int->0），初始化才赋**程序员写的值**（如 `static int a = 42` 的 42）。
