
# JVM运行流程

## 先说结论

Java 源代码从 `.java` 到 CPU 执行，经历 **编译 -> 加载 -> 验证 -> 准备 -> 解析 -> 初始化 -> 执行** 七个阶段。
核心流程为：**源码经 javac 编译为字节码 -> ClassLoader 加载到 JVM -> 执行引擎解释/JIT 编译为机器码 -> CPU 执行**。

## 深度解析

### 各阶段详解

| 阶段 | 操作 | 说明 |
|------|------|------|
| **编译** | `javac` | `.java` -> `.class` 字节码（与平台无关） |
| **加载** | ClassLoader | 读取二进制流 -> 生成 Class 对象 |
| **验证** | Verifier | 文件格式/元数据/字节码/符号引用校验 |
| **准备** | Allocator | static 变量分配内存并设默认值（0/null/false） |
| **解析** | Resolver | 常量池符号引用替换为直接引用 |
| **初始化** | `<clinit>` | 执行静态代码块和变量赋值 |
| **执行** | 执行引擎 | 解释器逐行执行 + JIT 编译热点代码 |

### 准备阶段 vs 初始化阶段

```java
// 准备阶段后: a=0, b=null
// 初始化阶段后: a=42, b=new ArrayList<>()
public static int a = 42;
public static List<String> b = new ArrayList<>();
```

关键区别：
- **准备阶段**：赋**默认值**（0/null/false）
- **初始化**：赋**程序员指定的初值**

## 易错点/踩坑

- **javac 不是 JVM 的一部分**，它是 JDK 的开发工具
- **类加载 != 初始化**：加载只是读入字节码，初始化才是执行 `<clinit>`
- **解释器和 JIT 不是二选一**：默认混合模式
