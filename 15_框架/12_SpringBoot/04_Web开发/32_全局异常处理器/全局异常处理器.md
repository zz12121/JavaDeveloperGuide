# 全局异常处理器

## 先说结论

Spring Boot 提供默认的错误页面，同时支持通过 `@ControllerAdvice` 实现全局异常处理，**统一返回 JSON 或自定义页面**。

## 深度解析

### 默认行为

```
请求错误
        ↓
ErrorController 处理
        ↓
返回 JSON 或错误页面
        ↓
/error
```

### 自定义错误页面

```
src/main/resources/
├── static/
│   └── error/
│       ├── 404.html
│       ├── 500.html
│       └── error.html  ← 通用的错误页
```

### @ControllerAdvice 全局处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    public Result handleBusiness(BusinessException e) {
        return Result.error(e.getMessage());
    }
    
    @ExceptionHandler(Exception.class)
    public Result handleException(Exception e) {
        return Result.error("系统错误");
    }
}
```

## 易错点/踩坑

- ❌ 异常未被捕获 → 返回 Spring Boot 默认错误页
- ❌ 异常处理返回 null → 可能导致空响应
- ❌ 异常处理顺序错误 → 子类异常应在父类之前

## 代码示例

### 完整异常处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    public Result handleBusiness(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }
    
    @ExceptionHandler(ValidationException.class)
    public Result handleValidation(ValidationException e) {
        return Result.error(400, e.getMessage());
    }
    
    @ExceptionHandler(Exception.class)
    public Result handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error(500, "系统错误，请稍后重试");
    }
}
```
