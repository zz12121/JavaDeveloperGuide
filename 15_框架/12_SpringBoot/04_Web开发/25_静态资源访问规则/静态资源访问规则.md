# 静态资源访问规则

## 先说结论

Spring Boot 自动配置静态资源访问，默认从 `/static`、`/public`、`/resources`、`/META-INF/resources` 目录提供静态文件。

## 深度解析

### 默认静态资源目录

| 目录 | 说明 |
|------|------|
| `classpath:/static/` | 推荐使用 |
| `classpath:/public/` | 公开目录 |
| `classpath:/resources/` | 资源目录 |
| `classpath:/META-INF/resources/` | 第三方 jar 资源 |

### 访问规则

```
请求 URL: GET /css/style.css
        ↓
查找目录顺序:
1. classpath:/static/css/style.css ✓
2. classpath:/public/css/style.css
3. classpath:/resources/css/style.css
4. classpath:/META-INF/resources/css/style.css
```

### 目录结构

```
src/main/resources/
├── static/
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   └── app.js
│   └── images/
│       └── logo.png
└── templates/
    └── index.html
```

### 访问方式

```html
<!-- 方式1：省略 /static 前缀（推荐） -->
<link rel="stylesheet" href="/css/style.css">

<!-- 方式2：完整路径 -->
<link rel="stylesheet" href="/static/css/style.css">
```

## 易错点/踩坑

- ❌ 静态资源 404 → 检查是否放在正确目录
- ❌ 与 Controller 路径冲突 → 静态资源优先于 Controller
- ❌ 忘记加 `/` → 相对路径可能出错
