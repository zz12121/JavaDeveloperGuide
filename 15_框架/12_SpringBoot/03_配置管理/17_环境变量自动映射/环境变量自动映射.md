# 环境变量自动映射

## 先说结论

Spring Boot 自动将环境变量映射为配置属性，`LOGGING_PATTERN_CONSOLE` → `logging.pattern.console`，支持多种格式转换。

## 深度解析

### 映射规则

| 环境变量 | 配置属性 |
|----------|----------|
| `SERVER_PORT` | `server.port` |
| `SPRING_PROFILES_ACTIVE` | `spring.profiles.active` |
| `APP_CONFIG_FILE` | `app.config.file` |
| `DATASOURCE_URL` | `datasource.url` |

### 转换规则

```
LOGGING_PATTERN_CONSOLE
         ↓
logging.pattern.console

1. 转小写
2. 下划线(_) 替换为 点(.)
3. 下划线前缀移除
```

### 特殊规则

| 格式 | 示例 |
|------|------|
| 全大写下划线 | `SERVER_PORT` → `server.port` |
| 下划线转点 | `APP_DB_HOST` → `app.db.host` |
| 多下划线 | `APP_DB_HOST_PORT` → `app.db.host.port` |

### 环境变量使用示例

```bash
# Linux/Mac
export SERVER_PORT=8080
export SPRING_PROFILES_ACTIVE=prod
export APP_DB_URL=jdbc:mysql://localhost:3306/test

# Windows
set SERVER_PORT=8080

# 运行
java -jar app.jar
```

### Docker 环境变量

```yaml
# docker-compose.yml
services:
  app:
    environment:
      - SERVER_PORT=8080
      - SPRING_DATASOURCE_URL=jdbc:mysql://db:3306/test
```

## 易错点/踩坑

- ❌ `app.name` 对应 `APP_NAME` 而不是 `APP_NAME_ENV` — 中间下划线可能混淆
- ❌ 数字处理 — `APP_PORT_8080` 不是 `app.port.8080`
- ❌ 环境变量覆盖配置文件 — 优先级高，可能导致本地调试困惑
