# 配置元数据（spring-configuration-metadata.json）

## 先说结论

配置元数据是 Spring Boot 提供的 IDE 智能提示支持，通过生成 metadata 文件，让你在 application.yml 中获得属性补全。

## 深度解析

### 元数据文件位置

```
META-INF/
└── spring-configuration-metadata.json
```

### 自动生成

Spring Boot 2.2+ 使用 `spring-boot-configuration-processor` 自动生成：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-configuration-processor</artifactId>
    <optional>true</optional>
</dependency>
```

### 元数据格式

```json
{
  "groups": [
    {
      "name": "server",
      "type": "org.springframework.boot.web.server.ServerProperties",
      "sourceType": "org.springframework.boot.web.server.ServerProperties"
    }
  ],
  "properties": [
    {
      "name": "server.port",
      "type": "java.lang.Integer",
      "description": "Server port.",
      "sourceMethod": "getPort()",
      "defaultValue": 8080
    }
  ],
  "hints": [
    {
      "name": "server.port",
      "values": [
        { "value": 8080, "description": "Default port" },
        { "value": 443, "description": "HTTPS port" }
      ]
    }
  ]
}
```

### IDE 提示效果

- 属性名自动补全
- 属性类型显示
- 默认值提示
- 枚举值建议

### 手动定义元数据

```json
{
  "properties": [
    {
      "name": "app.mode",
      "type": "java.lang.String",
      "description": "Application mode",
      "defaultValue": "dev"
    }
  ]
}
```

## 易错点/踩坑

- ❌ 缺少 configuration-processor 依赖 — 元数据不会自动生成
- ❌ 修改配置类后元数据未更新 — 重新编译
- ❌ 元数据与实际配置不符 — 手动维护可能出错
