# 多环境配置

## 先说结论

多环境配置通过 profile 机制实现 dev/test/prod 环境隔离，每个环境有独立的配置文件。

## 深度解析

### 环境划分

| 环境 | 用途 | 典型配置 |
|------|------|----------|
| dev | 本地开发 | localhost、DEBUG 日志 |
| test | 单元测试 | H2 内存数据库 |
| staging | 预发布 | 模拟生产环境 |
| prod | 生产环境 | 远程数据库、INFO 日志 |

### 配置文件结构

```
src/main/resources/
├── application.yml              # 公共配置
├── application-dev.yml         # 开发环境
├── application-test.yml         # 测试环境
└── application-prod.yml        # 生产环境
```

### 公共配置（application.yml）

```yaml
# 公共配置，所有环境共享
server:
  port: 8080

spring:
  application:
    name: myapp

# 日志配置
logging:
  level:
    root: INFO
```

### 开发环境（application-dev.yml）

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/dev_db
    username: dev
    password: dev123

logging:
  level:
    root: DEBUG
    com.example: DEBUG
```

### 生产环境（application-prod.yml）

```yaml
spring:
  datasource:
    url: jdbc:mysql://prod-server:3306/prod_db
    username: prod
    password: ${DB_PASSWORD}

logging:
  level:
    root: WARN
    com.example: INFO
```

### 环境切换方式

| 方式 | 配置 |
|------|------|
| 命令行 | `--spring.profiles.active=dev` |
| 环境变量 | `SPRING_PROFILES_ACTIVE=dev` |
| IDEA | Run Configuration 中设置 |

## 易错点/踩坑

- ❌ profile 配置覆盖不了公共配置 — profile 先加载，公共配置后加载
- ❌ 生产密码写死在配置中 — 用环境变量或配置中心
- ❌ 忘记激活 profile — 默认不激活任何 profile
