# 命令行参数配置

## 先说结论

命令行参数是优先级最高的外部化配置方式，通过 `--key=value` 形式传递，适合临时覆盖或容器化部署场景。

## 深度解析

### 基本语法

```bash
java -jar app.jar --server.port=9090
java -jar app.jar --spring.profiles.active=prod
java -jar app.jar --app.name="My App"
```

### 多参数

```bash
java -jar app.jar \
  --server.port=9090 \
  --spring.profiles.active=prod \
  --app.max-connections=100
```

### 参数格式转换

| 命令行参数 | 配置属性 |
|------------|----------|
| `--server.port=8080` | `server.port=8080` |
| `--app.config.file=/path` | `app.config.file=/path` |
| `--SPRING.PROFILES.ACTIVE=dev` | `spring.profiles.active=dev` |

### SpringApplication 传参

```java
public static void main(String[] args) {
    // 手动添加参数
    args = new String[]{"--server.port=9090"};
    SpringApplication.run(Application.class, args);
}
```

### 参数展开（2.4+）

```bash
# 2.4 之前
java -jar app.jar --spring.config.location=file:./config/

# 2.4+ 支持展开
java -jar app.jar --spring.config.additional-location=file:./config/
```

## 易错点/踩坑

- ❌ 参数有空格没加引号 — `--app.name=My App` 应改为 `--app.name="My App"`
- ❌ 大小写混淆 — Linux 环境变量区分大小写
- ❌ 优先级问题 — 命令行参数会覆盖所有配置文件
