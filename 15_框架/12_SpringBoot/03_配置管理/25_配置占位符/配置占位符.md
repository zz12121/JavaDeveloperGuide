# 配置占位符

## 先说结论

配置占位符 `${property:default}` 允许引用其他配置属性并提供默认值，简化配置管理和复用。

## 深度解析

### 基本语法

```yaml
app:
  name: myapp
  description: ${app.name} is running  # 引用 app.name
  port: ${server.port:8080}           # 引用 server.port，默认为 8080
```

### 默认值语法

```yaml
# 有默认值
${property:default_value}
${property:${other:default}}

# 无默认值（属性不存在会报错）
${property}
```

### 嵌套占位符

```yaml
app:
  env: prod
  db:
    url: jdbc:mysql://${app.env}-server:3306/db
    # 结果: jdbc:mysql://prod-server:3306/db
```

### 实际应用

```yaml
spring:
  datasource:
    url: jdbc:mysql://${DB_HOST:localhost}:${DB_PORT:3306}/${DB_NAME:test}
    username: ${DB_USER:root}
    password: ${DB_PASSWORD}

  redis:
    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}
```

### 循环引用

```yaml
# 错误示例 - 循环引用
app:
  name: ${app.title}
  title: ${app.name}
```

## 易错点/踩坑

- ❌ 循环引用 — `${a:${b:a}}` 可能导致无限循环
- ❌ 默认值写错 — 默认值不存在时不报错，可能导致问题
- ❌ 占位符嵌套过多 — 难以维护
