# 外部化配置（Externalized Configuration）

## 先说结论

外部化配置是将配置从代码中分离出来，使应用可以在不同环境使用不同配置，无需重新构建。

## 深度解析

### 核心概念

```
代码 → 配置 → 环境
         ↑
    外部化配置
```

### 配置外部化方式

| 方式 | 说明 |
|------|------|
| 配置文件 | application.yml/properties |
| 命令行参数 | `--key=value` |
| 环境变量 | `KEY=VALUE` |
| 配置中心 | Nacos/Apollo/Consul |
| Spring Cloud Config | 分布式配置管理 |

### 多环境配置

```yaml
# application-dev.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/dev_db

# application-prod.yml
spring:
  datasource:
    url: jdbc:mysql://prod-server:3306/prod_db
```

### 激活环境

```bash
java -jar app.jar --spring.profiles.active=prod
```

### 配置中心架构

```
┌─────────────┐
│  配置中心   │
│  Nacos/Apollo│
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  应用实例1  │
│  应用实例2  │
│  应用实例3  │
└─────────────┘
```

## 易错点/踩坑

- ❌ 敏感配置明文存储 — 使用加密或配置中心
- ❌ 环境配置覆盖不生效 — 检查优先级
- ❌ 配置变更后未刷新 — 配合 @RefreshScope
