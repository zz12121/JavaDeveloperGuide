# 配置加密（Jasypt）

## 先说结论

生产环境的敏感配置（数据库密码、API 密钥等）不应明文存储，可使用 Jasypt 进行加密保护。

## 深度解析

### Jasypt 依赖

```xml
<dependency>
    <groupId>com.github.ulisesbocchio</groupId>
    <artifactId>jasypt-spring-boot-starter</artifactId>
    <version>3.0.5</version>
</dependency>
```

### 基本加密使用

```yaml
# application.yml
spring:
  datasource:
    password: ENC(Xkq9T2y7Qzx7N5w4R3bL6p1m8s2t5u0=)
    #                       ↑ Jasypt 加密后的密文

jasypt:
  encryptor:
    password: mySecretKey    # 加密密钥（不应提交到代码）
    algorithm: PBEWithMD5AndDES
```

### 自定义加密属性

```java
@Configuration
public class JasyptConfig {
    @Bean
    public StringEncryptor stringEncryptor() {
        PooledPBEStringEncryptor encryptor = new PooledPBEStringEncryptor();
        encryptor.setConfig(encryptorConfig());
        return encryptor;
    }

    private SimpleStringPBEConfig encryptorConfig() {
        SimpleStringPBEConfig config = new SimpleStringPBEConfig();
        config.setPassword("mySecretKey");
        config.setAlgorithm("PBEWithMD5AndDES");
        config.setKeyObtentionIterations("1000");
        config.setPoolSize(1);
        return config;
    }
}
```

### 环境变量注入密钥

```yaml
# 不在配置文件中明文写密钥
jasypt:
  encryptor:
    password: ${JASYPT_PASSWORD}
```

```bash
export JASYPT_PASSWORD=mySecretKey
```

### 命令行加密

```bash
java -cp jasypt-*.jar com.github.ulisesbocchio.jasypt.Main password=mySecret algorithm=PBEWithMD5AndDES input=myPassword
```

## 易错点/踩坑

- ❌ 加密密钥写在代码中 — 密钥应该通过环境变量传入
- ❌ 使用弱算法 — 推荐 `PBEWithHMACSHA512` 或更强的算法
- ❌ 密钥泄露 — 一旦泄露需更换所有密文
