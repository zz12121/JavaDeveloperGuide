# 条件注解总结对比

## 先说结论

Spring Boot 提供了 10+ 个 `@ConditionalOn*` 条件注解，分为 Class、Bean、Property、Resource、Web、Expression 六大类。理解每个注解的检查时机和用途，是掌握自动配置原理的关键。

## 深度解析

### 条件注解分类图

```
@Conditional* 条件注解体系
    │
    ├── Class 相关
    │   ├── @ConditionalOnClass          — classpath 有 Xxx
    │   └── @ConditionalOnMissingClass   — classpath 无 Xxx
    │
    ├── Bean 相关
    │   ├── @ConditionalOnBean           — 容器中有 Xxx Bean
    │   └── @ConditionalOnMissingBean    — 容器中无 Xxx Bean
    │
    ├── Property 相关
    │   └── @ConditionalOnProperty       — 配置属性满足条件
    │
    ├── Resource 相关
    │   └── @ConditionalOnResource       — 资源文件存在
    │
    ├── Web 环境相关
    │   ├── @ConditionalOnWebApplication — 是 Web 应用
    │   └── @ConditionalOnNotWebApplication — 非 Web 应用
    │
    └── Expression 相关
        └── @ConditionalOnExpression    — SpEL 表达式为 true
```

### 检查时机对比

| 注解类型 | 检查时机 | 说明 |
|----------|----------|------|
| `@ConditionalOnClass` | **最早**，classpath 扫描阶段 | 最快速，直接检查 .class 是否存在 |
| `@ConditionalOnResource` | classpath 扫描阶段 | 资源文件检查 |
| `@ConditionalOnBean` | Bean 注册阶段 | 需要其他 Bean 已注册 |
| `@ConditionalOnMissingBean` | Bean 注册阶段 | 依赖 Bean 注册顺序 |
| `@ConditionalOnProperty` | Bean 实例化阶段 | Environment 读取 |
| `@ConditionalOnExpression` | Bean 实例化阶段 | SpEL 表达式求值 |

### 选择指南

| 场景 | 推荐注解 |
|------|----------|
| 需要某个 jar 依赖 | `@ConditionalOnClass` |
| 不需要某个 jar 依赖 | `@ConditionalOnMissingClass` |
| 依赖其他 Bean | `@ConditionalOnBean` |
| 提供默认 Bean | `@ConditionalOnMissingBean` |
| 功能开关 | `@ConditionalOnProperty` |
| 复杂条件 | `@ConditionalOnExpression` |

## 易错点/踩坑

- ❌ **@ConditionalOnBean 在类级别** — 可能因 Bean 尚未注册而误判
- ❌ **@ConditionalOnMissingBean 在方法级别** — 更安全，因为方法执行时其他 Bean 已注册
