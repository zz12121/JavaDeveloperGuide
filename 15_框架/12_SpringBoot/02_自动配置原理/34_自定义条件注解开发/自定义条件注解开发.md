# 自定义条件注解开发

## 先说结论

Spring Boot 的条件注解基于 `@Conditional` 和 `Condition` 接口实现，可以扩展自定义条件注解，满足特定的自动配置需求。

## 深度解析

### 实现步骤

```java
// 1. 定义条件注解
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Conditional(OnMyCondition.class)
@Documented
public @interface ConditionalOnMyCondition {
    String value();
}
```

```java
// 2. 实现 Condition
public class OnMyCondition implements Condition {

    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        // 获取注解属性
        Map<String, Object> attributes = metadata
            .getAnnotationAttributes(ConditionalOnMyCondition.class.getName());

        String value = (String) attributes.get("value");

        // 获取配置属性
        String configValue = context.getEnvironment().getProperty("my.condition.value");

        // 自定义匹配逻辑
        return value.equals(configValue);
    }
}
```

```java
// 3. 使用自定义条件注解
@AutoConfiguration
@ConditionalOnMyCondition(value = "enabled")
public class MyAutoConfiguration { }
```

### Condition 接口详解

```java
public interface Condition {
    // 返回 true 表示条件满足
    boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata);
}
```

| 方法 | 用途 |
|------|------|
| `context.getEnvironment()` | 访问配置属性 |
| `context.getRegistry()` | 访问 Bean 注册表 |
| `context.getBeanFactory()` | 访问 Bean 工厂 |
| `context.getClassLoader()` | 获取类加载器 |
| `metadata.getAnnotationAttributes()` | 获取注解属性 |

### 常用工具类

```java
// 类存在性检查
ClassUtils.isPresent(className, classLoader);

// Bean 存在性检查
context.getRegistry().containsBeanDefinition(beanName);

// 属性值读取
context.getEnvironment().getProperty("property.name");
```
