# 自动配置条件注解概述

## 先说结论

Spring Boot 提供了一套完整的 `@Conditional*` 条件注解体系，用于控制自动配置类的加载时机。核心思想是"按需加载"——只有当特定条件满足时，对应的自动配置才会生效。

## 深度解析

### 条件注解分类

| 分类 | 注解 | 条件 |
|------|------|------|
| **Class 相关** | `@ConditionalOnClass` | classpath 中存在指定类 |
| | `@ConditionalOnMissingClass` | classpath 中不存在指定类 |
| **Bean 相关** | `@ConditionalOnBean` | 容器中存在指定 Bean |
| | `@ConditionalOnMissingBean` | 容器中不存在指定 Bean |
| **属性相关** | `@ConditionalOnProperty` | 配置属性满足条件 |
| **资源相关** | `@ConditionalOnResource` | 指定资源文件存在 |
| **环境相关** | `@ConditionalOnWebApplication` | 是 Web 应用环境 |
| | `@ConditionalOnNotWebApplication` | 非 Web 应用环境 |
| **表达式** | `@ConditionalOnExpression` | SpEL 表达式为 true |

### 注解继承关系

```
@Conditional（Spring 框架原生注解）
    │
    ├── @ConditionalOnClass
    ├── @ConditionalOnMissingClass
    ├── @ConditionalOnBean
    ├── @ConditionalOnMissingBean
    ├── @ConditionalOnProperty
    ├── @ConditionalOnResource
    ├── @ConditionalOnWebApplication
    ├── @ConditionalOnNotWebApplication
    └── @ConditionalOnExpression
```

### 条件注解组合使用

```java
// 组合多个条件：需要 classpath 有 Xxx 且 yyy 配置为 true
@AutoConfiguration
@ConditionalOnClass(Xxx.class)
@ConditionalOnProperty(name = "my.xxx.enabled", havingValue = "true")
public class MyAutoConfiguration { }
```

### 条件判断流程

```
扫描自动配置类
    │
    ▼
遍历每个 @Conditional* 注解
    │
    ├── @ConditionalOnClass → 检查 classpath
    ├── @ConditionalOnBean → 检查已注册 Bean
    ├── @ConditionalOnProperty → 检查配置属性
    └── ...
    │
    ├── 全部通过 → 注册 Bean
    └── 任一失败 → 跳过（静默）
```

## 易错点/踩坑

- ❌ **条件不满足时静默跳过** — 不会抛异常，只是不注册，可能导致意外
- ❌ **`@ConditionalOnMissingBean` 容易误判** — 用户代码在自动配置之前加载会导致判断错误
- ❌ **条件注解放在 @Bean 方法上** — 整个配置类都会被跳过
