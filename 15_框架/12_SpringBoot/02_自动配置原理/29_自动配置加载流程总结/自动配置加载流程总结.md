# 自动配置加载流程总结

## 先说结论

Spring Boot 自动配置通过 `@EnableAutoConfiguration` → `AutoConfigurationImportSelector` → 读取配置 → 条件过滤 → Bean 注册的完整链路实现"零配置"启动。

## 深度解析

### 完整流程图

```
应用启动
    │
    ▼
SpringApplication.run()
    │
    ▼
prepareContext() → 加载主启动类为 BeanDefinition
    │
    ▼
refresh(context)
    │
    ▼
ConfigurationClassPostProcessor.postProcessBeanDefinitionRegistry()
    │
    ▼
AutoConfigurationImportSelector.getAutoConfigurationEntry()
    │
    ▼
├─ 读取 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
│   （2.7+）
├─ 或读取 META-INF/spring.factories
│   （2.7 之前，兼容）
│
▼ AutoConfigurationImportFilter 过滤
├─ OnClassCondition
├─ OnBeanCondition
└─ OnWebApplicationCondition
    │
    ▼
遍历候选配置类
    │
    ▼
逐个执行 @Conditional* 条件判断
    │
    ├── 通过 → 注册 BeanDefinition
    └── 未通过 → 静默跳过
    │
    ▼
Spring Boot 应用启动完成
```

### 关键节点

| 阶段 | 关键类/方法 |
|------|-------------|
| 配置读取 | `SpringFactoriesLoader.loadFactoryNames()` |
| 过滤 | `AutoConfigurationImportFilter.match()` |
| 条件判断 | `@Conditional*` 注解 |
| Bean 注册 | `BeanDefinitionRegistry.registerBeanDefinition()` |
