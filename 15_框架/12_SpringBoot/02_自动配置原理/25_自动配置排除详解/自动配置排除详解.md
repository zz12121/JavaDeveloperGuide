# 自动配置排除详解

## 先说结论

Spring Boot 提供多种方式排除不需要的自动配置类，包括注解方式、配置文件方式，以及基于条件的排除策略。排除后，对应的 Bean 不会被注册。

## 深度解析

### 排除方式汇总

| 方式 | 配置位置 | Spring Boot 版本 |
|------|----------|------------------|
| `exclude` 属性 | `@SpringBootApplication` | 1.x - 2.x |
| `excludeName` 属性 | `@SpringBootApplication` | 2.x+ |
| `spring.autoconfigure.exclude` | `application.yml` | 2.x+ |
| `@AutoConfiguration` 的 exclude | 配置文件 | 3.0+ |

### 注解方式

```java
// 方式一：exclude 属性（推荐）
@SpringBootApplication(exclude = {
    DataSourceAutoConfiguration.class,
    WebMvcAutoConfiguration.class
})
public class MyApplication { }

// 方式二：excludeName 属性（Spring Boot 2.1+）
@SpringBootApplication(excludeName = {
    "org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration",
    "org.springframework.boot.autoconfigure.web.servlet.WebMvcAutoConfiguration"
})
public class MyApplication { }
```

### 配置文件方式

```properties
# application.yml（Spring Boot 2.x）
spring:
  autoconfigure:
    exclude:
      - org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
      - org.springframework.boot.autoconfigure.web.servlet.WebMvcAutoConfiguration
```

```yaml
# application.yml（Spring Boot 3.x）
spring:
  autoconfigure:
    exclude:
      - org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
```

### Spring Boot 3.0 变化

Spring Boot 3.0 移除了 `@SpringBootApplication` 的 `exclude` 属性，只能使用 `excludeName` 或配置文件：

```java
// Spring Boot 3.0
@SpringBootApplication(excludeName = "org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration")
```

## 易错点/踩坑

- ❌ **exclude 类名写错** — 不存在的类不会报错，只是无效
- ❌ **3.0 仍使用 exclude 属性** — 编译报错
- ❌ **排除后再配置** — 排除后即使手动配置也会失败
