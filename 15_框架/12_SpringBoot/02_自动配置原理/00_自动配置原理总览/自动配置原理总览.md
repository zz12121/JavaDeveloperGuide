# 自动配置原理

## 先说结论

Spring Boot 通过 `@EnableAutoConfiguration` 触发自动配置扫描，从 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`（2.7+）或 `META-INF/spring.factories` 读取配置类列表，配合 `@Conditional*` 系列条件注解实现"智能配置"——只加载环境中存在的依赖对应的配置类。

## 深度解析

### 核心组件

| 组件 | 作用 | 位置 |
|------|------|------|
| `@EnableAutoConfiguration` | 启用自动配置机制 | `SpringBootApplication` 内置 |
| `AutoConfigurationImportSelector` | 加载自动配置类列表 | `spring-boot-autoconfigure` |
| `AutoConfigurationImportFilter` | 过滤无效配置 | `spring-boot` |
| `AutoConfiguration` 注解 | 标记自动配置类 | `org.springframework.boot.autoconfigure` |

### 自动配置核心注解体系

```
┌──────────────────────────────────────────────────────────────┐
│                      @AutoConfiguration                      │
│                     (标注在配置类上)                          │
│                                                              │
│    ┌──────────────────────────────────────────────────┐      │
│    │           @Conditional* 条件注解家族               │      │
│    └──────────────────────────────────────────────────┘      │
│                                                              │
│  @AutoConfigureBefore/@After  — 加载顺序                      │
│  @AutoConfigureOrder         — 显式排序                      │
└──────────────────────────────────────────────────────────────┘
```

### 条件注解全家桶

| 条件注解 | 条件满足才加载 |
|----------|---------------|
| `@ConditionalOnClass` | classpath 存在指定类 |
| `@ConditionalOnMissingClass` | classpath 不存在指定类 |
| `@ConditionalOnBean` | 容器中存在指定 Bean |
| `@ConditionalOnMissingBean` | 容器中不存在指定 Bean |
| `@ConditionalOnProperty` | 配置属性满足条件 |
| `@ConditionalOnResource` | 存在指定资源文件 |
| `@ConditionalOnWebApplication` | 是 Web 应用环境 |
| `@ConditionalOnNotWebApplication` | 非 Web 应用环境 |
| `@ConditionalOnExpression` | SpEL 表达式为 true |

### 自动配置加载流程

```
应用启动
    │
    ▼
SpringApplication.run()
    │
    ▼
refresh(context)
    │
    ▼
ConfigurationClassPostProcessor
    │
    ▼
AutoConfigurationImportSelector.getAutoConfigurationEntry()
    │
    ▼
读取 imports 文件 / spring.factories
    │
    ▼
AutoConfigurationImportFilter 过滤
    │
    ▼
@Conditional* 逐个条件判断
    │
    ▼
通过 → 注册为 BeanDefinition
失败 → 跳过（不抛异常）
```

### 自动配置排序

- `@AutoConfigureBefore(A.class)` — 在 A 之前加载
- `@AutoConfigureAfter(A.class)` — 在 A 之后加载
- `@AutoConfigureOrder(100)` — 显式指定顺序，数值越小越先

## 易错点/踩坑

- ❌ **spring.factories 在 2.7 已废弃** — 很多旧教程和源码仍是此格式，需迁移到 `AutoConfiguration.imports`
- ❌ **自动配置类的条件不满足时静默跳过** — 不会抛异常，只是不注册，排查困难
- ❌ **`@ConditionalOnMissingBean` 容易失效** — 如果用户代码先执行，条件判断会误判
- ❌ **`@AutoConfigureBefore/After` 只对自动配置类有效** — 对普通 `@Configuration` 类无效

## 代码示例

```java
// 自定义自动配置类（旧格式 - 2.7 之前）
@Configuration
@AutoConfigureBefore(DataSourceAutoConfiguration.class)
public class MyAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public MyService myService() {
        return new MyServiceImpl();
    }
}
```

```properties
# spring.factories（已废弃，仅 2.7 以下可用）
# org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
# com.example.MyAutoConfiguration
```

```properties
# AutoConfiguration.imports（2.7+ 推荐）
com.example.MyAutoConfiguration
```
