# 自动配置与手动配置优先级

## 先说结论

Spring Boot 的配置优先级是：**用户手动配置 > 自动配置**。通过 `@ConditionalOnMissingBean` 等机制保证用户配置不会被自动配置覆盖。

## 深度解析

### 优先级链

```
高
│
├── 命令行参数
├── 环境变量
├── application.yml (Profile)
├── application.yml
├── application.properties
├── @PropertySource
│
├── 自动配置（@ConditionalOnMissingBean 保证）
│
└── 默认值
    │
低
```

### 实现机制

**@ConditionalOnMissingBean 的核心作用**：

```java
@AutoConfiguration
public class RedisAutoConfiguration {

    @Bean
    @Primary
    @ConditionalOnMissingBean(StringRedisTemplate.class)  // 关键
    public StringRedisTemplate stringRedisTemplate() {
        return new StringRedisTemplate();
    }
}
```

```java
@Configuration
public class MyRedisConfig {

    @Bean
    public StringRedisTemplate stringRedisTemplate() {
        return new CustomStringRedisTemplate();  // 用户自定义
    }
}
```

结果：用户的 `MyRedisConfig` 先生效，`@ConditionalOnMissingBean` 检测到 Bean 已存在，自动配置的 `stringRedisTemplate()` 跳过。

### Bean 注册顺序

| 顺序 | 来源 | 说明 |
|------|------|------|
| 1 | @ComponentScan | 用户代码，按包顺序 |
| 2 | @Import 普通配置 | 用户手动导入的配置 |
| 3 | 自动配置 | DeferredImportSelector 后处理 |
| 4 | @Import(AutoConfiguration) | 最后处理 |

## 易错点/踩坑

- ❌ **Bean 名称冲突** — 即使有 `@ConditionalOnMissingBean`，同名 Bean 会冲突
- ❌ **@Primary 冲突** — 多个 `@Primary` Bean 会导致注入失败
