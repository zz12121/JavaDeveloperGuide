# 启动失败分析

## 先说结论

Spring Boot 通过 **FailureAnalyzer** 机制，在启动失败时提供友好的错误提示，分析常见错误原因并给出解决方案。

## 深度解析

### 失败分析流程

```
┌─────────────────────────────────────────────────────────────┐
│                    失败分析流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   启动失败                                                   │
│       │                                                     │
│       ↓                                                     │
│   捕获异常                                                   │
│       │                                                     │
│       ↓                                                     │
│   遍历所有 FailureAnalyzer                                  │
│       │                                                     │
│       ├── PortInUseFailureAnalyzer                         │
│       ├── DataSourceBeanCreationFailureAnalyzer            │
│       ├── ContextFailureAnalyzer                           │
│       └── ...                                              │
│       │                                                     │
│       ↓                                                     │
│   匹配并生成友好提示                                         │
│       │                                                     │
│       ↓                                                     │
│   输出分析报告                                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 内置 FailureAnalyzer

| 分析器 | 处理的异常 |
|--------|-----------|
| `PortInUseFailureAnalyzer` | 端口被占用 |
| `DataSourceBeanCreationFailureAnalyzer` | DataSource 创建失败 |
| `NoSuchBeanDefinitionFailureAnalyzer` | Bean 不存在 |
| `BeanCurrentlyInCreationFailureAnalyzer` | 循环依赖 |
| `ApplicationContextFailedToRefresh` | Context 刷新失败 |
| `ValidationConfigurationFailureAnalyzer` | 验证配置失败 |

### 自定义 FailureAnalyzer

```java
// 实现接口
public class MyFailureAnalyzer implements FailureAnalyzer {
    
    @Override
    public FailureAnalysis analyze(Throwable cause) {
        if (cause instanceof MyException) {
            MyException ex = (MyException) cause;
            return new FailureAnalysis(
                "配置错误",           // 标题
                "请检查 application.yml 中的 xxx 配置",  // 描述
                ex                   // 原因
            );
        }
        return null;  // 不处理，返回 null
    }
}
```

### 注册方式

**spring.factories**：
```properties
org.springframework.boot.SpringApplicationFailureAnalyzer=\
  com.example.MyFailureAnalyzer
```

**或 Java 8+**：
```java
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication app = new SpringApplication(Application.class);
        app.addListeners(new FailureAnalyzingApplicationListener());
        app.run(args);
    }
}
```

## 易错点/踩坑

- ❌ 自定义 FailureAnalyzer 未正确注册
- ❌ FailureAnalyzer 返回 null 导致不处理
- ❌ 异常匹配逻辑错误

### 自定义分析器

```java
public class ConfigMissingFailureAnalyzer implements FailureAnalyzer {
    
    @Override
    public FailureAnalysis analyze(Throwable cause) {
        Throwable rootCause = cause.getCause();
        
        if (rootCause instanceof MissingPropertyException) {
            MissingPropertyException ex = (MissingPropertyException) rootCause;
            String message = String.format(
                "缺少必需的配置项: %s%n请在 application.yml 中添加:",
                ex.getPropertyName()
            );
            return new FailureAnalysis("配置缺失", message, ex);
        }
        return null;
    }
}
```
