# 泛型约束

## 核心约束一览

由于**类型擦除**，Java 泛型存在以下限制：

### 1. 不能实例化类型参数

```java
public class Box<T> {
    // ❌ 不能 new T()
    T value = new T();
    
    // ✅ 变通：通过 Class 对象反射创建
    T value = clazz.getDeclaredConstructor().newInstance();
    
    // ✅ 变通：通过 Supplier 工厂
    T value = supplier.get();
}

// 实际用法
public <T> T create(Class<T> clazz) throws Exception {
    return clazz.getDeclaredConstructor().newInstance();
}
```

### 2. 不能创建泛型数组

```java
// ❌ 不能创建
T[] arr = new T[10];
List<String>[] lists = new ArrayList<String>[10];

// ✅ 变通1：Object 数组 + 强转（有警告）
@SuppressWarnings("unchecked")
T[] arr = (T[]) new Object[10];

// ✅ 变通2：反射（推荐，精确）
@SuppressWarnings("unchecked")
T[] arr = (T[]) Array.newInstance(componentType, size);

// ✅ 变通3：用 List 代替数组
List<T> list = new ArrayList<>();
```

### 3. 静态成员不能使用类的类型参数

```java
public class Box<T> {
    // ❌ 静态字段不能用类型参数
    private static T instance;
    
    // ❌ 静态方法不能用类型参数（用的是类声明的 T）
    public static T getInstance() { return null; }
    
    // ✅ 静态方法用自己声明的类型参数（泛型方法）
    public static <U> U create(U u) { return u; }
}
```

### 4. 不能对类型参数使用 instanceof

```java
public <T> boolean isType(Object obj) {
    // ❌ 不能用 instanceof T
    return obj instanceof T;
    
    // ✅ 变通：通过 Class 对象
    return clazz.isInstance(obj);
}
```

### 5. 不能捕获或抛出类型参数异常

```java
// ❌ 泛型类不能继承 Throwable
public class MyException<T> extends Exception { }

// ❌ 不能 catch 类型参数
public <T extends Exception> void method() {
    try { ... } catch (T e) { ... }  // ❌
}

// ✅ 可以在方法签名中 throws 类型参数
public <T extends Exception> void rethrow(Exception e) throws T {
    throw (T) e;  // 警告，但允许
}
```

### 6. 类型参数不支持基本类型

```java
// ❌ 不合法
Box<int> box = new Box<>();
List<double> list = new ArrayList<>();

// ✅ 使用包装类型
Box<Integer> box = new Box<>();
List<Double> list = new ArrayList<>();
```
