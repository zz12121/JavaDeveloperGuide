# 泛型擦除机制（Type Erasure）

## 什么是类型擦除

Java 泛型是在**编译器层面**实现的，编译器完成类型检查后，会将所有泛型类型参数从字节码中**擦除**：

- `T` → `Object`（无界）
- `T extends Number` → `Number`（擦除为上界）
- `T extends Comparable & Serializable` → `Comparable`（擦除为第一个上界）

```java
// 源代码
public class Box<T> {
    private T value;
    public T getValue() { return value; }
}

// 擦除后（字节码等价）
public class Box {
    private Object value;
    public Object getValue() { return value; }
}
```

## 擦除的影响

### 1. 运行时无法获取类型参数

```java
List<String> list = new ArrayList<>();
// ❌ 无法获取运行时类型参数
// list.getClass().getGenericType()  不是这样用的

// 运行时只知道是 ArrayList
System.out.println(list.getClass());  // class java.util.ArrayList

// instanceof 检查无类型参数
if (list instanceof List) { ... }           // ✅
// if (list instanceof List<String>) { ... } // ❌ 编译错误
```

### 2. 不能创建泛型数组

```java
// ❌ 编译错误
T[] arr = new T[10];
List<String>[] arr = new ArrayList<String>[10];

// ✅ 变通方案
@SuppressWarnings("unchecked")
T[] arr = (T[]) new Object[10];

// 或者用反射
T[] arr = (T[]) Array.newInstance(clazz, size);
```

### 3. 不能用于 instanceof

```java
// ❌ 编译错误：不能对泛型参数用 instanceof
if (obj instanceof T) { ... }

// ✅ 通过 Class 对象检查
if (clazz.isInstance(obj)) { ... }
```

### 4. static 成员不能用类型参数

```java
// ❌ 编译错误
static T instance;
static T getInstance() { ... }
```

## 擦除后的强转（编译器自动插入）

```java
// 源代码
List<String> list = new ArrayList<>();
String s = list.get(0);

// 等价字节码（编译器自动插入 checkcast）
List list = new ArrayList();
String s = (String) list.get(0);  // 编译器自动加强转
```

## 如何在运行时获取泛型信息

```java
// 通过超类类型令牌（Super Type Token）保留泛型
// 匿名子类会把泛型类型写入字节码
Type type = new ArrayList<String>() {}.getClass()
                .getGenericSuperclass();
// 获得 java.util.ArrayList<java.lang.String>

// Jackson/Gson 中的 TypeReference 就是此原理
TypeReference<List<String>> ref = new TypeReference<List<String>>() {};
```
