# 泛型的本质（类型参数化）

## 什么是泛型

泛型（Generics）是 **JDK 5** 引入的特性，本质是**类型参数化**——将类型作为参数传递，使代码能复用于不同类型，同时保证**编译期类型安全**。

```java
// 没有泛型：需要手动强转，运行时可能 ClassCastException
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);  // 手动强转，不安全

// 有泛型：编译期检查，类型安全
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);  // 无需强转，编译器保证类型
```

## 泛型的三大优势

| 优势 | 说明 |
|------|------|
| 类型安全 | 编译期检查类型，避免运行时 `ClassCastException` |
| 消除强转 | 代码更简洁，不需要手动 `(Type)` 强制转换 |
| 代码复用 | 同一套逻辑可以适配多种类型（算法与数据类型解耦） |

## 泛型的四种使用形式

| 形式 | 语法 | 说明 |
|------|------|------|
| 泛型类 | `class Box<T>` | 类定义时声明类型参数 |
| 泛型接口 | `interface Comparable<T>` | 接口定义时声明类型参数 |
| 泛型方法 | `<T> T method(T t)` | 方法独立声明类型参数 |
| 泛型通配符 | `? extends T` / `? super T` | 使用时限定类型范围 |

## 类型参数命名惯例

| 字母 | 含义 |
|------|------|
| `T` | Type（通用类型） |
| `E` | Element（集合元素） |
| `K` | Key（键） |
| `V` | Value（值） |
| `N` | Number（数字） |
| `R` | Return（返回值） |

## 泛型在运行时

由于**类型擦除**，泛型只在**编译期**存在，运行时类型参数被擦除为原始类型（`Object` 或上界）。

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();

// 运行时两者是同一个类
System.out.println(list1.getClass() == list2.getClass());  // true
System.out.println(list1.getClass());  // class java.util.ArrayList
```
