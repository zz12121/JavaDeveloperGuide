# 对象序列化（Serializable / Externalizable）

## 什么是序列化

将 Java 对象转换为**字节序列**的过程叫**序列化**，反方向叫**反序列化**。

```
Java 对象 → 字节序列 → 文件/网络传输 → 字节序列 → Java 对象
  序列化                             反序列化
```

## Serializable 接口

`Serializable` 是一个**标记接口**（无任何方法），表示该类的对象可以被序列化：

```java
public interface Serializable {
    // 没有任何方法！
}
```

### 序列化示例

```java
public class User implements Serializable {
    private String name;
    private int age;

    // 必须有无参构造器（反序列化时需要）
    public User() {}

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// 序列化
User user = new User("Tom", 25);
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("user.ser"))) {
    oos.writeObject(user);
}

// 反序列化
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("user.ser"))) {
    User restored = (User) ois.readObject();
    System.out.println(restored.name);  // Tom
    System.out.println(restored.age);   // 25
}
```

## 序列化条件

一个对象能被序列化，需要满足：

1. 类实现 `Serializable` 接口
2. 所有非 transient 的字段也必须是可序列化的（或基本类型）
3. 如果父类没实现 `Serializable`，父类必须有**无参构造器**（反序列化时需要）

如果父类只有有参构造器且没有无参构造器，反序列化**一定会失败**，并抛出 `java.io.InvalidClassException`。
### 详细原理解析
#### 1. 反序列化的构造过程
当你反序列化一个对象时，JVM 不会调用该对象的构造器。但是，**JVM 会向上追溯继承链，直到找到第一个未实现 `Serializable`的父类，然后调用该父类的无参构造器**。
- **已实现 `Serializable`的类**：其字段由字节流直接恢复，不调用构造器。
- **未实现 `Serializable`的父类**：其字段必须通过正常的 Java 对象创建流程初始化，即**必须调用构造器**。
#### 2. 问题场景模拟
```java
// 父类：未实现 Serializable，且只有有参构造器
class Parent {
    private int value;
    public Parent(int value) { // 只有有参构造器
        this.value = value;
    }
}

// 子类：实现了 Serializable
class Child extends Parent implements Serializable {
    private String name;
    public Child(String name, int value) {
        super(value); // 编译没问题
        this.name = name;
    }
}
```
**尝试反序列化 `Child`对象时会发生什么？**
1. JVM 开始反序列化 `Child`。
2. `Child`实现了 `Serializable`，跳过其构造器。
3. JVM 向上检查父类 `Parent`。
4. `Parent`**没有**实现 `Serializable`，因此 JVM 必须调用 `Parent`的构造器来初始化这部分状态。
5. JVM 尝试寻找 `Parent()`无参构造器 —— **找不到**。
6. **结果**：抛出 `InvalidClassException`，错误信息通常包含：`no valid constructor`。
### 解决方案
针对父类没有无参构造器的情况，有以下几种解决方式：

|方案|做法|适用场景|
|---|---|---|
|**最佳方案**​|**给父类添加一个 `protected`或 `public`的无参构造器**​|你能修改父类代码。这是最标准、最推荐的做法。|
|**次优方案**​|**让父类也实现 `Serializable`**​|父类的设计本就应该是可序列化的，且其状态也需要被持久化。|
|**妥协方案**​|**在子类中实现 `readObject`方法手动处理**​|极特殊场景，非常复杂且不推荐，需要反射强行赋值。|

## Externalizable 接口

`Externalizable` 继承 `Serializable`，提供**自定义序列化控制**：

```java
public interface Externalizable extends Serializable {
    void writeExternal(ObjectOutput out) throws IOException;
    void readExternal(ObjectInput in) throws IOException, ClassNotFoundException;
}
```

```java
public class User implements Externalizable {
    private String name;
    private int age;

    public User() {}  // 必须有无参构造器

    @Override
    public void writeExternal(ObjectOutput out) throws IOException {
        out.writeUTF(name);
        out.writeInt(age);
    }

    @Override
    public void readExternal(ObjectInput in) throws IOException, ClassNotFoundException {
        name = in.readUTF();
        age = in.readInt();
    }
}
```

## Serializable vs Externalizable

| 维度 | Serializable | Externalizable |
|------|-------------|----------------|
| 控制粒度 | JVM 自动，全部字段 | 开发者自定义 |
| 性能 | 较低（反射 + 内部序列化逻辑） | **更高**（手动控制） |
| 无参构造器 | 不要求 | **必须** |
| 灵活性 | 低 | 高（可选择性序列化字段） |
| 推荐 | ✅ 大多数场景 | 特殊性能要求 |

## 序列化注意事项

1. `static` 字段**不参与**序列化（属于类，不属于对象）
2. `transient` 字段不参与序列化
3. 序列化不调用构造器，反序列化也不调用构造器（`Externalizable` 除外）
4. 序列化的对象图：如果对象引用了其他对象，这些对象也会被递归序列化
