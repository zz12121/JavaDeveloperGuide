# 自定义异常

## 为什么需要自定义异常

1. 业务语义清晰：`UserNotFoundException` 比 `RuntimeException("用户不存在")` 更表意
2. 便于统一处理：全局异常处理器可按类型精确匹配
3. 携带更多上下文：可扩展错误码、错误描述等字段

## 自定义异常模板

### 推荐方式：继承 RuntimeException（Unchecked）

```java
/**
 * 业务异常基类
 */
public class BusinessException extends RuntimeException {
    
    private final int code;      // 错误码
    private final String msg;    // 错误信息
    
    public BusinessException(int code, String msg) {
        super(msg);
        this.code = code;
        this.msg = msg;
    }
    
    // 保留原始异常（支持异常链）
    public BusinessException(int code, String msg, Throwable cause) {
        super(msg, cause);
        this.code = code;
        this.msg = msg;
    }
    
    public int getCode() { return code; }
    public String getMsg() { return msg; }
}
```

### 具体业务异常

```java
public class UserNotFoundException extends BusinessException {
    public UserNotFoundException(Long userId) {
        super(404, "用户不存在，id=" + userId);
    }
}

public class OrderStatusException extends BusinessException {
    public OrderStatusException(String status) {
        super(400, "订单状态非法：" + status);
    }
}
```

## 最佳实践

### 1. 错误码枚举

```java
public enum ErrorCode {
    USER_NOT_FOUND(1001, "用户不存在"),
    PARAM_INVALID(1002, "参数非法"),
    SYSTEM_ERROR(5000, "系统错误");
    
    private final int code;
    private final String message;
    
    ErrorCode(int code, String message) {
        this.code = code;
        this.message = message;
    }
}

public class BusinessException extends RuntimeException {
    private final ErrorCode errorCode;
    
    public BusinessException(ErrorCode errorCode) {
        super(errorCode.getMessage());
        this.errorCode = errorCode;
    }
}
```

### 2. Spring 全局异常处理器配合

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(BusinessException.class)
    public Result<?> handleBusiness(BusinessException e) {
        return Result.fail(e.getCode(), e.getMsg());
    }
    
    @ExceptionHandler(UserNotFoundException.class)
    public Result<?> handleUserNotFound(UserNotFoundException e) {
        return Result.fail(404, e.getMessage());
    }
}
```

## 注意事项

- 自定义异常**务必提供多个构造方法**，包含 `(String message, Throwable cause)` 以支持异常链
- 不要用异常控制正常业务流程（if-else 的事不要用 try-catch 代替，性能差）
- 异常信息要有意义，包含足够的上下文信息（如 userId、orderId 等）
