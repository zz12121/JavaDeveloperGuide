# 异常链（Exception Chaining）

## 什么是异常链

将捕获的原始异常作为新异常的 **cause（原因）** 一并抛出，形成异常链，保留完整的错误传播路径。

```
ServiceException: 查询用户失败
  caused by: DataAccessException: 数据库查询出错
    caused by: SQLException: Table 'user' not found
```

## 为什么需要异常链

❌ **不使用异常链（信息丢失）：**
```java
try {
    dao.findUser(id);
} catch (SQLException e) {
    // 丢弃原始异常，只抛出新异常
    throw new ServiceException("查询用户失败");
}
// 日志中看不到 SQL 错误原因，排查困难！
```

✅ **使用异常链（信息保留）：**
```java
try {
    dao.findUser(id);
} catch (SQLException e) {
    // 将原始异常作为 cause 传入
    throw new ServiceException("查询用户失败", e);
}
```

## 实现方式

### 方式一：构造方法传入 cause

```java
// Throwable 提供的构造方法
Throwable(String message, Throwable cause)
Throwable(Throwable cause)

// 使用示例
throw new RuntimeException("外层描述", originalException);
```

### 方式二：initCause() 方法

```java
RuntimeException e = new RuntimeException("外层描述");
e.initCause(originalException);  // 只能调用一次
throw e;
```

## 获取异常链信息

```java
try {
    // ...
} catch (ServiceException e) {
    Throwable cause = e.getCause();               // 获取直接原因
    while (cause != null) {
        System.out.println(cause.getMessage());
        cause = cause.getCause();                 // 遍历整条链
    }
    e.printStackTrace();  // 打印完整异常链堆栈
}
```

## 实战：分层异常转换

```java
// DAO 层：SQL 异常
public User findById(Long id) throws DataAccessException {
    try {
        // JDBC 操作
    } catch (SQLException e) {
        throw new DataAccessException("DB 查询失败", e);
    }
}

// Service 层：业务异常
public UserDTO getUser(Long id) {
    try {
        User user = userDao.findById(id);
        if (user == null) throw new UserNotFoundException(id);
        return convert(user);
    } catch (DataAccessException e) {
        throw new ServiceException("获取用户失败，id=" + id, e);
    }
}
```

## Suppressed Exception（抑制异常）

try-with-resources 使用的特殊机制：

```java
try (MyResource r = new MyResource()) {
    throw new Exception("主异常");
    // close() 抛 Exception → 被抑制，附加到主异常
} catch (Exception e) {
    e.getSuppressed();  // 获取所有被抑制的异常
}
```

> 区别：`getCause()` 是因果关系；`getSuppressed()` 是"同时发生但被压制"的关系
