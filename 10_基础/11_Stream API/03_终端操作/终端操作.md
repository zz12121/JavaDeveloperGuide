## 核心结论

终端操作触发 Stream 流水线的**真正执行**，分为**非短路操作**（处理所有元素）和**短路操作**（不必处理所有元素即可返回结果）。一个 Stream 只能被消费一次。

---

## 深度解析

### 1. 收集操作

#### collect — 最灵活的终端操作
```java
// 收集为 List
List<String> list = stream.collect(Collectors.toList());

// 收集为 Set
Set<String> set = stream.collect(Collectors.toSet());

// 收集为 Map
Map<String, Integer> map = list.stream()
    .collect(Collectors.toMap(User::getName, User::getAge));

// 分组
Map<String, List<User>> groupByCity = users.stream()
    .collect(Collectors.groupingBy(User::getCity));

// 分区（二分组）
Map<Boolean, List<User>> partition = users.stream()
    .collect(Collectors.partitioningBy(u -> u.getAge() >= 18));

// 拼接字符串
String joined = list.stream().collect(Collectors.joining(", "));
```

### 2. 遍历与匹配

```java
// forEach
list.stream().forEach(System.out::println);

// anyMatch — 至少一个匹配（短路）
boolean hasLongName = list.stream().anyMatch(s -> s.length() > 10);

// allMatch — 全部匹配（短路）
boolean allPositive = nums.stream().allMatch(n -> n > 0);

// noneMatch — 全部不匹配（短路）
boolean noneNull = list.stream().noneMatch(Objects::isNull);

// findFirst — 返回第一个（短路）
Optional<String> first = list.stream().filter(s -> s.length() > 3).findFirst();

// findAny — 返回任意一个（短路，并行流中有优势）
Optional<String> any = list.parallelStream().findAny();
```

### 3. 聚合操作

```java
// count
long count = list.stream().count();

// min / max（返回 Optional）
Optional<Integer> min = nums.stream().min(Integer::compareTo);
Optional<User> oldest = users.stream().max(Comparator.comparing(User::getAge));

// sum / average / summaryStatistics（IntStream/LongStream/DoubleStream 专属）
int sum = nums.stream().mapToInt(Integer::intValue).sum();
IntSummaryStatistics stats = nums.stream().mapToInt(Integer::intValue).summaryStatistics();
// stats.getSum() / stats.getAverage() / stats.getMax() / stats.getMin() / stats.getCount()
```

### 4. 归约操作

#### reduce — 将流归纳为一个值

```java
// 两参数 reduce：初始值 + BinaryOperator
int sum = nums.stream().reduce(0, Integer::sum);

// 一参数 reduce：无初始值，返回 Optional
Optional<Integer> total = nums.stream().reduce(Integer::sum);

// 三参数 reduce：用于并行流（combiner 合并各线程结果）
int sum2 = nums.parallelStream().reduce(0, Integer::sum, Integer::sum);

// 求最大值
Optional<Integer> max = nums.stream().reduce(Integer::max);

// 字符串拼接
String result = list.stream().reduce("", String::concat);
```

### 5. 转换为数组

```java
String[] arr = list.stream().toArray(String[]::new);
int[] intArr = nums.stream().mapToInt(Integer::intValue).toArray();
```

### 6. 终端操作分类

| 类型 | 操作 | 是否短路 |
|------|------|----------|
| 收集 | `collect` | 否 |
| 遍历 | `forEach` | 否 |
| 匹配 | `anyMatch`/`allMatch`/`noneMatch` | **是** |
| 查找 | `findFirst`/`findAny` | **是** |
| 聚合 | `count`/`min`/`max`/`sum` | 否 |
| 归约 | `reduce` | 否 |
| 数组 | `toArray` | 否 |

> **关键理解**：短路操作在找到结果后立即停止处理剩余元素，类似 `||` 和 `&&` 的短路行为。

### 7. Collectors 高级用法

#### collectingAndThen — 先收集再转换结果

```java
// 先收集为 List，再转为不可变列表（JDK 16+ 更推荐用 toList()）
List<String> immutable = list.stream()
    .collect(Collectors.collectingAndThen(
        Collectors.toList(),  // 第一步：收集为 List
        List::copyOf          // 第二步：转换为不可变列表
    ));

// 实战：先分组，再计算每组数量
Map<String, Long> cityCount = users.stream()
    .collect(Collectors.collectingAndThen(
        Collectors.groupingBy(User::getCity),
        Map::size  // 转换为城市数量
    ));
```

#### reducing — 归约收集器

```java
// reducing 等价于 reduce，但作为 Collector 可与其他收集器组合
List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

// 两参数 reducing
Optional<Integer> max = numbers.stream()
    .collect(Collectors.reducing(Integer::max));

// 三参数 reducing（带初始值）
Integer sum = numbers.stream()
    .collect(Collectors.reducing(0, Integer::sum, Integer::sum));

// 实战：按部门分组，再求每组工资总和
Map<String, Integer> totalSalaryByDept = employees.stream()
    .collect(Collectors.groupingBy(
        Employee::getDepartment,
        Collectors.reducing(0, Employee::getSalary, Integer::sum)
    ));
```

#### partitioningBy 高级用法

```java
// 分区后再统计数量
Map<Boolean, Long> agePartition = users.stream()
    .collect(Collectors.partitioningBy(
        u -> u.getAge() >= 18,
        Collectors.counting()  // 统计每组人数
    ));

// 分区后再收集为特定类型
Map<Boolean, List<String>> nameByAge = users.stream()
    .collect(Collectors.partitioningBy(
        u -> u.getAge() >= 18,
        Collectors.mapping(User::getName, Collectors.toList())
    ));
```

### 8. Stream 与集合互转

#### Stream → 集合

```java
// toList() — JDK 16+，返回不可变列表（推荐）
List<String> immutableList = list.stream().toList();

// toSet() — JDK 16+，返回不可变集合
Set<String> immutableSet = list.stream().toSet();

// toCollection(Supplier) — 收集到指定集合类型
LinkedList<String> linkedList = list.stream().collect(Collectors.toCollection(LinkedList::new));
ArrayDeque<String> arrayDeque = list.stream().collect(Collectors.toCollection(ArrayDeque::new));
ConcurrentHashMap.KeySetView<String, Boolean> concurrentSet =
    list.stream().collect(Collectors.toCollection(ConcurrentHashMap::newKeySet));

// toMap — 收集为 Map
Map<String, Integer> map = list.stream()
    .collect(Collectors.toMap(Function.identity(), String::length));

// 收集为不可变 Map（JDK 10+）
Map<String, Integer> immutableMap = list.stream()
    .collect(Collectors.toMap(Function.identity(), String::length, (a, b) -> a,
        Map::of));
```

#### 集合 → Stream

```java
// List / Set
List<String> list = Arrays.asList("a", "b", "c");
Stream<String> stream = list.stream();

// 数组
String[] array = {"a", "b", "c"};
Stream<String> stream1 = Arrays.stream(array);
Stream<String> stream2 = Stream.of(array);

// Map（需分别处理 key 和 value）
Map<String, Integer> map = Map.of("a", 1, "b", 2);

// 处理所有键值对
Stream<Map.Entry<String, Integer>> entryStream = map.entrySet().stream();

// 只处理 key 或 value
Stream<String> keys = map.keySet().stream();
Stream<Integer> values = map.values().stream();

// 合并 key 和 value
List<String> combined = map.entrySet().stream()
    .map(e -> e.getKey() + "=" + e.getValue())
    .collect(Collectors.toList());
```

#### toList() vs Arrays.asList() vs List.of() 的区别

| 方法 | 返回类型 | 可变性 | 特点 |
|------|----------|--------|------|
| `toList()` | `List<T>` (不可变) | JDK 16+，底层调用 `ImmutableCollections.List` | 最推荐，无法添加/修改/删除 |
| `Arrays.asList()` | `Arrays$ArrayList` | 可修改但**不可增删** | 固定长度，修改元素会反映到底层数组 |
| `List.of()` | `List<T>` (不可变) | JDK 9+，底层调用 `ImmutableCollections.List` | 与 toList() 类似 |

```java
List<String> list = List.of("a", "b", "c");
list.add("d");  // 抛 UnsupportedOperationException

List<String> mutable = new ArrayList<>(list);  // 转可变集合
mutable.add("d");  // OK
```

#### Stream 被消费后再次使用的问题

```java
// ⚠️ Stream 只能遍历一次
Stream<String> stream = list.stream();
stream.forEach(System.out::println);
stream.forEach(System.out::println);  // 抛 IllegalStateException

// ✅ 如果需要多次使用，先收集
List<String> collected = list.stream().toList();  // JDK 16+
collected.forEach(System.out::println);
collected.forEach(System.out::println);  // OK

// ✅ 或者使用 Supplier 惰性创建多个 Stream
Supplier<Stream<String>> streamSupplier = () -> list.stream();
streamSupplier.get().forEach(System.out::println);  // OK
streamSupplier.get().forEach(System.out::println);  // OK
```

### 9. 自定义 Collector 实现

通过 `Collector.of()` 自定义收集器：

```java
// 示例：自定义收集器，将字符串收集为逗号分隔的字符串
Collector<String, ?, String> commaJoiner = Collector.of(
    StringBuilder::new,                    // supplier: 创建容器
    (sb, s) -> {                           // accumulator: 累加元素
        if (sb.length() > 0) sb.append(", ");
        sb.append(s);
    },
    (sb1, sb2) -> {                        // combiner: 合并两个容器（并行时）
        if (sb1.length() > 0 && sb2.length() > 0) sb1.append(", ");
        sb1.append(sb2);
        return sb1;
    },
    StringBuilder::toString                // finisher: 转换为最终结果
);

// 使用
String result = list.stream().collect(commaJoiner);
```

#### Collector 接口方法说明

| 方法 | 说明 | 并行时的要求 |
|------|------|--------------|
| `supplier()` | 创建结果容器 | - |
| `accumulator()` | 将元素加入容器 | 必须为**结合律**的 |
| `combiner()` | 合并两个容器 | 必须为**结合律**和**交换律**的 |
| `finisher()` | 将容器转为最终结果 | - |
| `characteristics()` | 标记收集器特性 | - |

#### 常用 Characteristics

```java
// UNORDERED: 不保证顺序，收集器可优化（如 toSet）
// CONCURRENT: 允许并行模式下同时调用 accumulator
// IDENTITY_FINISH: finisher 是恒等函数，可省略

// 示例：创建线程安全的收集器
Collector<String, ?, Set<String>> concurrentSetCollector = Collector.of(
    () -> Collections.newSetFromMap(new ConcurrentHashMap<>()),
    Set::add,
    (s1, s2) -> { s1.addAll(s2); return s1; },
    Function.identity(),
    Collector.Characteristics.CONCURRENT,
    Collector.Characteristics.UNORDERED
);
```
