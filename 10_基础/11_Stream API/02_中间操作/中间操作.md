## 核心结论

中间操作是**惰性求值**的，调用后不会立即执行，而是返回一个新的 Stream。只有终端操作触发时，整个流水线才会真正执行。中间操作分为**无状态**和**有状态**两类。

---

## 深度解析

### 1. 无状态操作（处理每个元素不依赖其他元素）

#### filter — 过滤
```java
list.stream().filter(s -> s.length() > 3)
```

#### map — 映射转换
```java
list.stream().map(String::toUpperCase)
list.stream().map(Integer::parseInt)
```

#### flatMap — 扁平化映射（一对多展开）

**基础用法**：
```java
// 将嵌套集合展平
List<List<Integer>> nested = Arrays.asList(
    Arrays.asList(1, 2), Arrays.asList(3, 4)
);
nested.stream().flatMap(Collection::stream) // [1, 2, 3, 4]

// 字符串拆分
Stream.of("hello world", "foo bar")
      .flatMap(s -> Arrays.stream(s.split(" ")))
      .toArray(); // ["hello", "world", "foo", "bar"]
```

**高级字符串处理场景**：

```java
// 1. 去除空字符串（split 后过滤空串）
List<String> words = Stream.of("a", "b,,c", "")
    .flatMap(s -> Arrays.stream(s.split(",")))
    .filter(s -> !s.isEmpty())
    .collect(Collectors.toList());
// 结果: ["a", "b", "c"]

// 2. 统计所有字符出现次数
String text = "hello world";
Map<Character, Long> charCount = text.chars()
    .mapToObj(c -> (char) c)
    .collect(Collectors.groupingBy(c -> c, Collectors.counting()));
// 结果: {h=1, e=1, l=3, o=2, ' '=1, w=1, r=1, d=1}

// 3. 多行文本按行拆分并处理
String multiline = "line1\nline2\nline3";
List<String> processed = Arrays.stream(multiline.split("\n"))
    .map(String::trim)
    .filter(s -> !s.isEmpty())
    .collect(Collectors.toList());

// 4. 嵌套对象展开（订单 → 商品）
List<Order> orders = getOrders();
List<Item> allItems = orders.stream()
    .flatMap(order -> order.getItems().stream())  // Order → Stream<Item>
    .collect(Collectors.toList());

// 5. 去重后的扁平化
List<List<String>> nestedNames = Arrays.asList(
    Arrays.asList("Alice", "Bob"),
    Arrays.asList("Bob", "Charlie")
);
List<String> uniqueNames = nestedNames.stream()
    .flatMap(List::stream)
    .distinct()
    .collect(Collectors.toList());
// 结果: ["Alice", "Bob", "Charlie"]

// 6. 条件映射（map vs flatMap 选择）
// 场景：返回Optional时用flatMap展平
List<Optional<String>> optionals = Arrays.asList(
    Optional.of("a"), Optional.empty(), Optional.of("b")
);
List<String> values = optionals.stream()
    .flatMap(Optional::stream)  // Optional 为空则产生空流
    .collect(Collectors.toList());
// 结果: ["a", "b"]
```

#### peek — 调试用，不影响流
```java
list.stream()
    .peek(System.out::println)
    .filter(s -> s.length() > 3)
    .collect(Collectors.toList());
```

### 2. 有状态操作（需要知道其他元素的信息）

#### distinct — 去重（依赖 equals/hashCode）
```java
list.stream().distinct()
```

#### sorted — 排序
```java
list.stream().sorted()                          // 自然排序
list.stream().sorted(Comparator.reverseOrder()) // 逆序
list.stream().sorted(Comparator.comparing(User::getAge)) // 按属性
```

#### limit — 截取前 N 个
```java
list.stream().limit(5)
```

#### skip — 跳过前 N 个
```java
list.stream().skip(3)
```

### 3. 中间操作分类总结

| 操作 | 类型 | 状态 | 说明 |
|------|------|------|------|
| `filter` | 无状态 | 转换 | 按条件过滤元素 |
| `map` | 无状态 | 转换 | 元素类型/值转换 |
| `flatMap` | 无状态 | 转换 | 扁平化一对多映射 |
| `peek` | 无状态 | 观察 | 不修改流，用于调试 |
| `distinct` | **有状态** | 有状态 | 去重，需缓冲所有元素 |
| `sorted` | **有状态** | 有状态 | 排序，需缓冲所有元素 |
| `limit` | **有状态** | 短路有状态 | 截断，有缓冲上限 |
| `skip` | **有状态** | 有状态 | 跳过，需缓冲跳过的元素 |

> **面试关键**：有状态操作在并行流中性能较差，因为需要全局协调（如 sorted 需要所有元素到齐后才能排序）。

### 4. 典型组合示例

```java
// 链式调用：过滤 → 转换 → 去重 → 排序 → 截取
List<String> result = users.stream()
    .filter(u -> u.getAge() >= 18)                // 过滤未成年
    .map(User::getName)                            // 提取姓名
    .distinct()                                    // 去重
    .sorted(Comparator.comparing(String::length))   // 按名字长度排序
    .limit(10)                                     // 最多10个
    .collect(Collectors.toList());                 // 终端操作
```
