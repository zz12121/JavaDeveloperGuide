# 抽象类（abstract class）vs 接口（interface）

## 先说结论

抽象类和接口都是实现**抽象**的方式。抽象类定义"是什么"（IS-A），体现模板设计；接口定义"能做什么"（CAN-DO），体现行为契约。一个类只能继承一个抽象类，但可以实现多个接口。

JDK 8 引入 **default 方法**和 **static 方法**解决了接口无法扩展的问题。
JDK 9 进一步引入 **private 方法**让 default 方法之间可以共享代码。

## 深度解析

### 核心区别

| 维度 | 抽象类（abstract class） | 接口（interface） |
|------|------------------------|------------------|
| 继承/实现 | extends（单继承） | implements（多实现） |
| 构造方法 | ✅ 可以有 | ❌ 不能有 |
| 实例变量 | ✅ 可以有任意修饰符 | ❌ 只有 public static final（常量） |
| 方法 | 可以有抽象方法和具体方法 | JDK 8 前全是抽象方法 |
| 访问修饰符 | 任意修饰符 | 默认 public abstract |
| 设计意图 | IS-A 关系（模板） | CAN-DO 关系（契约） |

### 什么时候用抽象类
- 多个类有**共同的属性和状态**需要共享
- 需要定义**模板方法**（部分实现 + 部分留给子类）
- 类之间有明确的**父子关系**

### 什么时候用接口
- 定义**行为契约**，不关心实现细节
- 需要**多重继承**的效果（一个类实现多个接口）
- 跨继承树的类需要**共同行为**（如 Serializable、Comparable）

### JDK 8：default 方法

- 用 `default` 关键字修饰，有方法体实现
- 实现类可以直接使用，也可以重写
- 解决接口升级问题：新增方法不会破坏已有实现类
- **菱形继承冲突**：一个类实现两个接口，同名 default 方法必须显式重写

### JDK 8：static 方法

- 接口中的 static 方法属于**接口本身**，不属于任何实现类
- 只能通过**接口名.方法名**调用，实现类不能继承
- 通常作为**工具方法**放在接口中

### JDK 9：private 方法

- 用 `private` 修饰，只能在接口内部使用
- 解决多个 default 方法之间的**代码复用**问题
- 有 private 方法和 private static 方法两种

### 接口演进历程

| 版本    | 新增特性       | 说明        |
| ----- | ---------- | --------- |
| JDK 8 | default 方法 | 接口有默认实现   |
| JDK 8 | static 方法  | 接口级别的静态方法 |
| JDK 9 | private 方法 | 接口内部代码复用  |

## 易错点/踩坑

- ❌ "接口只是抽象类的替代"：两者设计意图不同，抽象类强调血缘关系，接口强调能力契约
- ❌ "抽象类不能有具体方法"：抽象类可以有具体方法，只有 abstract 方法需要子类实现
- ❌ "接口不能有方法实现"：JDK 8+ 的 default 方法可以有实现
- ❌ "抽象类必须至少有一个抽象方法"：抽象类可以全部是具体方法，用 abstract 只是为了禁止实例化
- ❌ "实现类继承接口的 default 方法"：default 方法可以被"继承使用"，但不叫继承，是实现类的实例方法
- ❌ "接口的 static 方法可以被实现类调用"：不能，只能通过接口名调用
- ❌ "default 方法解决所有问题"：default 方法多了会导致接口变成"胖接口"，设计时仍应保持接口精简

## 代码示例

```java
// 抽象类 - 模板设计
public abstract class AbstractShape {
    protected String color;

    public AbstractShape(String color) {
        this.color = color;
    }

    // 具体方法（子类直接复用）
    public void display() {
        System.out.println("Color: " + color + ", Area: " + getArea());
    }

    // 抽象方法（子类必须实现）
    public abstract double getArea();
}

// 接口 - 行为契约
public interface Drawable {
    void draw();  // 抽象方法
}

public interface Resizeable {
    void resize(double factor);
}

// 继承抽象类 + 实现多个接口
public class Circle extends AbstractShape implements Drawable, Resizeable {
    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double getArea() { return Math.PI * radius * radius; }

    @Override
    public void draw() { System.out.println("Drawing Circle"); }

    @Override
    public void resize(double factor) { radius *= factor; }
}
```

```java
public interface MyInterface {

    // JDK 8 之前：抽象方法
    void abstractMethod();

    // JDK 8：default 方法
    default void defaultMethod() {
        System.out.println("Default implementation");
        helperMethod();  // JDK 9+ 可调用 private 方法
    }

    // JDK 8：static 方法
    static void staticMethod() {
        System.out.println("Static utility method");
    }

    // JDK 9：private 方法（供 default 方法复用）
    private void helperMethod() {
        System.out.println("Helper logic");
    }
}

// 使用
public class MyClass implements MyInterface {
    @Override
    public void abstractMethod() {
        System.out.println("Implementation");
    }
    // defaultMethod() 可以直接使用或重写
}

// 调用
MyClass obj = new MyClass();
obj.defaultMethod();       // 通过实例调用 default
MyInterface.staticMethod(); // 通过接口名调用 static

// 菱形冲突解决
public class MultiImpl implements InterfaceA, InterfaceB {
    @Override
    public void sameDefaultMethod() {
        InterfaceA.super.sameDefaultMethod();  // 显式选择
    }
}
```

                │  + color        │
                │  + display()    │
                │  # getArea()    │
                └────────┬────────┘
                         │ extends
              ┌──────────┴──────────┐
              ▼                     ▼
