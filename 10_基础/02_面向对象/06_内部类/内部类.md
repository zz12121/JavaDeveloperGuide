# 内部类（成员内部类/静态内部类/局部内部类/匿名内部类）

## 先说结论

内部类是定义在另一个类内部的类，主要用于**逻辑封装**和**回调实现**。
四种内部类中，**匿名内部类**在 Lambda 出现前使用最广泛，**静态内部类**在工程中最常用（如 Builder 模式）。

## 深度解析

### 四种内部类

| 类型 | 定义位置 | 访问外部类 | 持有外部类引用 | 创建方式 |
|------|---------|-----------|--------------|---------|
| 成员内部类 | 类内方法外 | 所有成员（含 private） | ✅ 隐式持有 | `Outer.Inner obj = outer.new Inner()` |
| 静态内部类 | 类内 static | 只有静态成员 | ❌ 不持有 | `Outer.Inner obj = new Outer.Inner()` |
| 局部内部类 | 方法内 | 方法内的 final 变量 | ✅ 隐式持有 | 方法内 new |
| 匿名内部类 | 表达式中 | 同局部内部类 | ✅ 隐式持有 | `new Interface() { ... }` |

### 匿名内部类（重点）

- 没有名字的内部类，一次性使用
- 可以实现接口或继承类
- Lambda 表达式是匿名内部类的简化写法（只适用于函数式接口）

### 静态内部类 vs 成员内部类

- 静态内部类**不持有外部类引用**，更轻量，不会内存泄漏
- 成员内部类隐式持有外部类引用，可能导致外部类无法被 GC

## 易错点/踩坑

- ❌ "匿名内部类只能实现接口"：匿名内部类也可以**继承一个类**
- ❌ "局部内部类可以访问方法的所有局部变量"：只能访问 **effectively final** 的局部变量
- ❌ "静态内部类可以访问外部类的实例变量"：只能访问外部类的**静态成员**
- ❌ "成员内部类必须通过外部类实例创建"：是的，这是正确的，因为成员内部类隐式持有外部类引用

## 代码示例

```java
public class Outer {
    private int x = 10;
    private static int sx = 20;

    // 1. 成员内部类
    class MemberInner {
        public void print() {
            System.out.println(x);   // ✅ 可访问外部类私有成员
            System.out.println(sx);
        }
    }

    // 2. 静态内部类
    static class StaticInner {
        public void print() {
            // System.out.println(x);  // ❌ 不能访问实例变量
            System.out.println(sx);    // ✅ 可访问静态成员
        }
    }

    // 3. 局部内部类
    public void method() {
        final int y = 30;
        class LocalInner {
            public void print() {
                System.out.println(x + y);  // y 必须是 effectively final
            }
        }
        new LocalInner().print();
    }

    // 4. 匿名内部类
    public Runnable getRunnable() {
        return new Runnable() {
            @Override
            public void run() {
                System.out.println("Anonymous: " + x);
            }
        };
        // Lambda 等价写法：return () -> System.out.println("Lambda: " + x);
    }
}
```

```java
class Outer {
    public Runnable test() {
        int count = 10;      // ① 局部变量初始化
        
        class Inner implements Runnable {
            public void run() {
                System.out.println(count);  // 访问局部变量
            }
        }
        
        count = 20;           // ② 修改局部变量（在创建对象之前！）
        count = 20;  // ❌ 编译错误：Variable 'count' is accessed from within inner class, needs to be final or effectively final (等效是不可修改的)
        return new Inner();   // ③ 创建内部类对象
    }
}
//编译后的局部内部类，使用的变量被编译为final
class Outer$Inner {
    private final int val$count;

    Outer$Inner(int capturedValue) {
        this.val$count = capturedValue;  // 值拷贝，不是引用
    }
}
```

## 图解/流程

```
Outer
├── MemberInner    （持有 Outer.this 引用 → 可访问所有成员）
├── StaticInner    （不持有引用 → 只能访问静态成员）
├── method()
│   └── LocalInner （持有引用 → 只能访问 final 局部变量）
└── getRunnable()
    └── AnonymousInner（持有引用 → 一次性使用）
```
