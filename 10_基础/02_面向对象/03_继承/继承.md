# 继承（Inheritance）— extends

## 先说结论

继承使用 `extends` 关键字实现**代码复用**，子类继承父类的非私有属性和方法。
Java 是**单继承**（一个类只能有一个直接父类），但可以**多层继承**形成继承链。

## 深度解析

### 继承的核心特性
- **代码复用**：子类直接使用父类的方法和属性
- **单继承**：一个类只能 extends 一个父类（区别于 C++ 多继承）
- **多层继承**：A → B → C，形成继承链
- **所有类默认继承 Object**：`class Person` 等价于 `class Person extends Object`

### 继承中哪些能被继承

| 父类成员 | 能否继承 | 能否访问 |
|---------|---------|---------|
| public 方法/字段 | ✅ | ✅ |
| protected 方法/字段 | ✅ | ✅ |
| default 方法/字段 | ✅ | 同包可访问 |
| private 方法/字段 | ❌ | ❌ |
| 构造方法 | ❌ | 通过 super() 调用 |
| static 方法/字段 | ✅ | 可访问（但不叫继承，是隐藏） |

### 方法重写（Override）规则
- 方法名、参数列表必须与父类**完全相同**
- 返回值类型必须与父类**相同或是其子类**（协变返回）
- 访问权限不能比父类**更严格**（可以更宽松）
- 不能重写 private/static/final 方法
- 建议加 `@Override` 注解让编译器检查

## 易错点/踩坑

- ❌ "子类继承父类的构造方法"：构造方法不能被继承，子类通过 `super()` 调用父类构造
- ❌ "子类继承父类的所有东西"：private 成员、构造方法不会被继承
- ❌ "重写 static 方法"：static 方法属于类不属于对象，子类定义同名 static 方法叫**隐藏**，不是重写
- ❌ "继承破坏封装"：继承是 IS-A 关系，如果只是想复用代码应该用**组合（HAS-A）**

## 代码示例

```java
public class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " is eating");
    }

    // final 方法不能被重写
    public final void breathe() {
        System.out.println(name + " is breathing");
    }
}

public class Dog extends Animal {
    private String breed;

    public Dog(String name, String breed) {
        super(name);  // 必须调用父类构造
        this.breed = breed;
    }

    @Override
    public void eat() {
        System.out.println(name + "(" + breed + ") is eating dog food");
    }

    // breathe() 不能被重写，因为是 final
}
```

## 图解/流程

```
Object
  │
  └── Animal
        │  name, eat(), breathe()
        │
        └── Dog
              │  breed, eat() 重写
              │
              └── Husky
                    │  特有方法
```
