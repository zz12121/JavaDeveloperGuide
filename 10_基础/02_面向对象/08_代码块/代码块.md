# 构造代码块 vs 静态代码块 vs 普通代码块

## 先说结论

三种代码块的执行时机和用途不同：**静态代码块**（static {}）类加载时执行一次，用于初始化静态资源；**构造代码块**（{}）每次创建对象时执行，在构造方法之前；**普通代码块**（方法内的 {}）方法调用时执行。

## 深度解析

### 三种代码块对比

| 类型 | 语法 | 执行时机 | 执行次数 | 用途 |
|------|------|---------|---------|------|
| 静态代码块 | `static { }` | 类加载时 | 1次（整个生命周期） | 初始化静态变量、加载配置 |
| 构造代码块 | `{ }`（类内方法外） | 每次 new 对象时 | 每次创建 | 提取构造方法公共逻辑 |
| 普通代码块 | `{ }`（方法内） | 方法被调用时 | 每次调用 | 限制变量作用域 |

### 执行顺序
```
父类静态代码块 → 子类静态代码块
    （类加载阶段，只执行一次）

父类构造代码块 → 父类构造方法
→ 子类构造代码块 → 子类构造方法
    （对象创建阶段，每次 new 都执行）
```

### 各代码块的细节
- **静态代码块**：可以有多个，按定义顺序执行
- **构造代码块**：编译器会将构造代码块的内容**复制到每个构造方法的最前面**（super() 之后）
- **普通代码块**：最常见的用法是限制变量的**作用域**，如在大方法中划分逻辑段落

## 易错点/踩坑

- ❌ "构造代码块比构造方法先执行"：是的，但它实际上是编译器帮你把代码块内容**复制到构造方法中 super() 之后**，所以本质还是在构造方法里执行
- ❌ "静态代码块在 main 方法之前执行"：不一定，静态代码块在**类首次被使用**时执行（加载、创建对象、访问静态成员等触发）
- ❌ "构造代码块可以访问 this"：可以，因为执行时对象已在创建中

## 代码示例

```java
public class Parent {
    static { System.out.println("Parent 静态代码块"); }
    { System.out.println("Parent 构造代码块"); }
    public Parent() { System.out.println("Parent 构造方法"); }
}

public class Child extends Parent {
    static { System.out.println("Child 静态代码块"); }
    { System.out.println("Child 构造代码块"); }
    public Child() { System.out.println("Child 构造方法"); }
}

// new Child() 输出顺序：
// Parent 静态代码块
// Child 静态代码块
// Parent 构造代码块
// Parent 构造方法
// Child 构造代码块
// Child 构造方法
```

## 图解/流程

```
new Child() 执行流程：

类加载阶段（仅一次）：
┌─ Parent.class 加载 → Parent 静态代码块
└─ Child.class 加载  → Child 静态代码块

对象创建阶段（每次 new）：
┌─ 分配内存
├─ Parent 构造代码块 → Parent 构造方法
└─ Child 构造代码块  → Child 构造方法
```
