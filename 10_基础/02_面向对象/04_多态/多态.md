# 多态（Polymorphism）— 重写/重载

## 先说结论

多态是**同一操作作用于不同对象，产生不同的行为**。前提三个条件：**继承 + 重写 + 父类引用指向子类对象**。分为**编译时多态**（重载）和**运行时多态**（重写）。

## 深度解析

### 运行时多态（动态绑定）
```java
Animal a = new Dog();  // 父类引用指向子类对象
a.eat();               // 调用的是 Dog 的 eat()，不是 Animal 的
```
- JVM 在运行时根据**实际对象类型**决定调用哪个方法
- 只能调用父类中声明的方法，不能调用子类特有方法
- 属性不参与多态（属性是静态绑定的，看引用类型）

### 编译时多态（静态绑定）
- 方法**重载（Overload）**：同名不同参，编译器在编译时确定调用哪个版本
- 与运行时多态不同，重载不依赖对象实际类型

### 多态的三个前提
1. **继承关系**：子类 extends 父类
2. **方法重写**：子类重写父类方法
3. **父类引用指向子类对象**：`Parent p = new Child()`

### 多态中的类型转换
```java
Animal a = new Dog();     // 向上转型（自动）
Dog d = (Dog) a;          // 向下转型（强转，需 instanceof 判断）
Cat c = (Cat) a;          // 运行时 ClassCastException
```

## 易错点/踩坑

- ❌ "多态就是重写"：重写只是实现手段，多态还需要父类引用指向子类对象
- ❌ "属性也有多态"：**属性是静态绑定的**，`Animal a = new Dog(); a.name` 访问的是 Animal 的 name
- ❌ "static 方法可以多态"：static 方法属于类，不参与多态，子类同名 static 方法是隐藏
- ❌ "向下转型不需要判断"：必须用 `instanceof` 判断，否则运行时 `ClassCastException`

## 代码示例

```java
public class Animal {
    public String name = "Animal";  // 属性（静态绑定）
    public void eat() { System.out.println("Animal eats"); }
}

public class Dog extends Animal {
    public String name = "Dog";     // 属性隐藏，不是多态
    @Override
    public void eat() { System.out.println("Dog eats bone"); }
    public void bark() { System.out.println("Woof!"); }
}

// 多态演示
Animal a = new Dog();
System.out.println(a.name);  // "Animal"（属性看编译类型）
a.eat();                     // "Dog eats bone"（方法看运行类型）

// 安全向下转型
if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();  // "Woof!"
}
```

## 图解/流程

```
编译时              运行时
───────            ─────────
Animal a            实际对象是 Dog
a.eat()  ─────→    JVM 检查实际类型 Dog
                    Dog 重写了 eat()
                    → 调用 Dog.eat()

                    如果 Dog 没有重写
                    → 沿继承链向上找 Animal.eat()
```
