# 构造方法（构造函数）

## 先说结论

构造方法用于**创建和初始化对象**，与类同名、没有返回值、在 `new` 时自动调用。每个类至少有一个构造方法，不写则编译器提供默认无参构造。

## 深度解析

### 构造方法的特点
- 方法名与**类名完全相同**
- **没有返回值**（连 void 都不写）
- 在 `new` 关键字创建对象时**自动调用**
- 可以重载（多个构造方法，参数不同）
- **不能被 override**，但可以被子类调用（super）

### 构造方法的作用
1. 分配对象内存（JVM 负责）
2. 初始化实例变量（显式赋值或默认值）
3. 执行构造方法体中的自定义逻辑
4. 返回对象引用

### 默认构造方法
- 如果没有显式定义任何构造方法，编译器自动提供**无参构造**
- 如果已定义有参构造，编译器**不再提供**无参构造
- 默认构造的内容：调用 `super()` + 实例变量默认初始化

### 构造方法 vs 普通方法

| 维度 | 构造方法 | 普通方法 |
|------|---------|---------|
| 名称 | 必须与类名相同 | 任意 |
| 返回值 | 无（连 void 都不写） | 必须有（void/具体类型） |
| 调用方式 | new 时自动调用 | 对象.方法名() |
| 作用 | 初始化对象 | 执行业务逻辑 |
| 继承 | 不能继承，不能 override | 可以继承，可以 override |

## 易错点/踩坑

- ❌ "构造方法有返回值类型 void"：构造方法没有返回值，连 void 都不写
- ❌ "定义了有参构造后还有默认无参构造"：编译器只在没有任何构造方法时才提供无参构造
- ❌ "构造方法能被 override"：构造方法不能被重写，子类通过 `super()` 调用父类构造
- ❌ "构造方法能被 static/abstract/final 修饰"：都不能，修饰构造方法没有意义

## 代码示例

```java
public class Person {
    private String name;
    private int age;

    // 无参构造
    public Person() {
        this("Unknown", 0);  // 调用本类的有参构造
    }

    // 有参构造
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 拷贝构造（Java 没有内置拷贝构造，可以自己实现）
    public Person(Person other) {
        this.name = other.name;
        this.age = other.age;
    }
}

// 使用
Person p1 = new Person();           // 无参构造
Person p2 = new Person("张三", 25);  // 有参构造
Person p3 = new Person(p2);          // 拷贝构造
```

```java
class Parent {
    public Parent() {
        System.out.println("Parent constructor start");
        doSomething(); // ⚠️ 调用可被重写的方法
        System.out.println("Parent constructor end");
    }

    public void doSomething() {
        System.out.println("Parent doSomething");
    }
}

class Child extends Parent {
    private int value = 10;

    @Override
    public void doSomething() {
        System.out.println("Child value = " + value);
    }
}

new Child();

Parent constructor start
Child value = 0   ❌ 不是 10
Parent constructor end

//对象创建时的内存布局
Child 对象
├── Parent 部分（正在构造）
├── vtable（方法表）✅ 已经是 Child 的
└── value = 0 ❌（还没执行赋值）

//### 方法调用流程
new Child()
 └─ 调用 Parent 构造器
     └─ 调用 doSomething()
         └─ vtable 指向 Child.doSomething()
             └─ 读取 value（此时还是默认值 0）

```
## 图解/流程

```
new Person("张三", 25) 执行流程：

1. 堆中分配内存空间
2. 实例变量默认初始化（name=null, age=0）
3. 调用构造方法
   ├── 调用 super()（隐式调用 Object 构造）
   ├── this.name = "张三"（显式初始化）
   └── this.age = 25
4. 返回对象引用给 p2
```
