# 类与对象的关系

## 先说结论

**类是对象的模板（蓝图），对象是类的实例**。类定义了属性和方法，对象是类的具体化，占用实际内存。

## 深度解析

### 类（Class）
- 类是具有相同属性和行为的对象的**抽象模板**
- 类是一个**数据结构**，定义了对象的特征（字段）和行为（方法）
- 类本身不占用堆内存（Class 对象在方法区），只有实例化的对象才占堆内存

### 对象（Object / Instance）
- 对象是类的**具体实例**，是存在于内存中的实体
- 每个对象有独立的属性值，但共享类的方法代码
- 对象在堆内存中分配空间，引用变量在栈中

### 类与对象的关系
```
类（模板）          对象（实例）
───────           ──────────────
定义结构    ──→   创建具体实体
不占堆内存        占用堆内存
一个类             可以创建多个对象
```

### 三步创建对象
1. **声明**：`Person p;` — 在栈上创建引用变量
2. **实例化**：`new Person();` — 在堆上分配内存
3. **初始化**：构造方法赋初值

## 易错点/踩坑

- ❌ "类就是对象"：类是模板，对象是实例，两者不同层次
- ❌ "对象存储方法代码"：方法代码在方法区，对象只存储属性值和指向方法区的引用
- ❌ "引用变量就是对象"：`p` 是引用（栈），`new Person()` 才是对象（堆），`p` 指向对象

## 代码示例

```java
public class Person {
    // 属性（字段）
    String name;
    int age;

    // 构造方法
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 方法
    public void sayHello() {
        System.out.println("我是" + name + "，今年" + age + "岁");
    }
}

// 使用
Person p1 = new Person("张三", 25);  // 创建对象
Person p2 = new Person("李四", 30);  // 同一类，不同对象
p1.sayHello();  // 每个对象有独立的属性值
```

## 图解/流程

```
栈（Stack）                    堆（Heap）
───────                       ──────────
p1 ─────────────────→ ┌──────────────────┐
                      │ Person 对象      │
p2 ─────────────────→ ├──────────────────┤
                      │ name = "李四"    │
                      │ age  = 30        │
                      ├──────────────────┤
                      │ 方法引用 ────────────→ 方法区（共享）
                      └──────────────────┘

方法区（Method Area）
───────────────────
Person.class（类信息、方法字节码）
```
