# 封装（Encapsulation）— 访问修饰符

## 先说结论

封装通过**访问修饰符**控制属性和方法的可见性，隐藏实现细节，只暴露必要的接口。
核心原则：**属性私有化（private），方法公开化（public）**。

## 深度解析

### 封装的本质
- 将数据（属性）和操作数据的方法绑定在一起
- 隐藏对象的内部实现细节，对外只提供公共接口
- 通过 getter/setter 控制属性的读写

### Java 四种访问修饰符

| 修饰符 | 同类 | 同包 | 子类 | 不同包 |
|--------|------|------|------|--------|
| `private` | ✅ | ❌ | ❌ | ❌ |
| `(default)` | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

### 封装的好处
- **安全性**：防止外部直接修改内部数据
- **可维护性**：修改内部实现不影响外部调用
- **可扩展性**：可以在 getter/setter 中添加校验逻辑

## 易错点/踩坑

- ❌ "封装就是加 private"：封装的核心思想是隐藏细节、暴露接口，private 只是一种手段
- ❌ "所有属性都要 private"：某些场景下 default/protected 更合适（如包内共享、子类继承）
- ❌ "protected 对所有子类可见"：跨包时只有通过继承关系才能访问 protected 成员，不能通过父类引用访问

## 代码示例

```java
public class Student {
    // 属性私有化
    private String name;
    private int age;
    private double score;

    // 无参构造
    public Student() {}

    // 全参构造
    public Student(String name, int age, double score) {
        this.name = name;
        this.age = age;
        this.score = score;
    }

    // getter/setter（带校验逻辑）
    public void setAge(int age) {
        if (age < 0 || age > 150) {
            throw new IllegalArgumentException("年龄不合法");
        }
        this.age = age;
    }

    public int getAge() {
        return this.age;
    }

    // 只读属性（只有 getter，没有 setter）
    public double getScore() {
        return this.score;
    }
}
```

## 图解/流程

```
外部调用者                  Student 对象
──────────                 ──────────────
    │
    │  setAge(25)
    ├──────────────────→ 校验合法性 → this.age = 25
    │
    │  getAge()
    ├──────────────────→ return this.age
    │
    │  age = -5          ❌ 编译错误（private）
    │  score = 99        ❌ 编译错误（private）
```
