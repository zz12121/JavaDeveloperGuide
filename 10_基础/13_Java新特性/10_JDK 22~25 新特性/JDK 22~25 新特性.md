# JDK 22~25 新特性

## 概述

JDK 的发布节奏自 JDK 9 起改为每六个月发布一次（每年三月和九月），每个版本都有 JEP（Java Enhancement Proposal）作为特性载体。本章节覆盖 JDK 22（2024年3月）、JDK 23（2024年9月）、JDK 24（2025年3月）、JDK 25（2025年9月）四个版本的重大新特性。

---

## JDK 22 新特性（2024年3月正式发布）

### 1.1 Unnamed Patterns and Variables（unnamed ~）（正式特性）

在 JDK 21 首次引入后，unnamed patterns/variables 在 JDK 22 中**成为正式特性**（Permanent），不再是预览版。

**核心语法：**

```java
// unnamed pattern：忽略嵌套数据的结构
switch (shape) {
    case Circle(var radius)          -> System.out.println("Circle with r=" + radius);
    case Rectangle(_, _)            -> System.out.println("Rectangle");  // unnamed
    case Triangle                    -> System.out.println("Other shape");
}

// unnamed variable：忽略不使用的循环变量
for (int i = 0, _ = initialize(); i < 10; i++) {
    System.out.println("Processing...");  // _ 替代无用的 initialize() 返回值
}

// unnamed exception 变量
try {
    processConnection();
} catch (IOException _) {
    System.out.println("Failed");
}

// try-with-resources 中忽略不需要的变量
try (var _ = ScopedValue.get()) {
    // 只关心 ScopedValue 的效果，不关心返回的 AutoCloseable
}
```

**使用场景：**
- 当数据结构有多个字段但只需要其中几个时，避免引入无意义变量名
- 遍历 Map 时只需要 key 或 value：`map.forEach((_, value) -> process(value))`
- Lambda 参数未使用时：`list.stream().filter(_ -> true)`

### 1.2 Class-File API（预览特性 → JDK 22 正式）

JDK 22 提供了标准化的 `java.lang.classfile` 包，可以**解析、修改、生成 Java 类文件**，替代了 ASM 等第三方字节码工具。

```java
import java.lang.classfile.*;
import java.lang.classfile.attributes.*;
import java.lang.classfile.constantpool.*;
import java.lang.classfile.instruction.*;

// 读取类文件
byte[] classBytes = Files.readAllBytes(Path.of("MyClass.class"));
ClassModel classModel = ClassFile.of().parse(classBytes);

classModel.forEachElement(element -> {
    if (element instanceof MethodModel method) {
        System.out.println("Method: " + method.methodName());
    }
});

// 生成类文件
var cc = ClassFile.of();
byte[] newClass = cc.build(
    ClassDesc.of("com.example.Generated"),
    builder -> builder
        .withSuperclass(ConstantPoolUtils.classDesc(Object.class))
        .withMethodBody("main",
            MethodTypeDesc.of(Descriptor.of(void.class), ConstantPoolUtils.classDesc(String[].class)),
            Scope.trusted(),
            10,  // max_stack
            5,   // max_locals
            builder2 -> {
                builder2.getstatic(
                    FieldRefDesc.of(
                        ClassDesc.of("java/lang/System"),
                        "out",
                        ConstantPoolUtils.classDesc(PrintStream.class)
                    )
                );
                builder2.ldc("Hello, World!");
                builder2.invokevirtual(
                    MethodRefDesc.of(
                        ClassDesc.of("java/io/PrintStream"),
                        "println",
                        MethodTypeDesc.of(void.class, ConstantPoolUtils.classDesc(Object.class))
                    )
                );
                builder2.return_();
            }
        )
);
```

### 1.3 Launch Multi-File Source-Code Programs（JDK 22 正式）

之前 `java HelloWorld.java` 只能运行单文件。现在支持**多文件编译运行**：

```bash
# 之前（JDK 11+）：单文件
java HelloWorld.java

# 现在（JDK 22+）：多文件（自动编译）
java --source 22 Utility.java Main.java
java Main.java  # 自动找同类文件
```

```java
// Main.java
class Main {
    public static void main(String[] args) {
        String result = Utility.format("World");
        System.out.println(Utility.greet(result));
    }
}

// Utility.java
class Utility {
    static String format(String s) { return s.toUpperCase(); }
    static String greet(String s) { return "Hello, " + s + "!"; }
}
```

### 1.4 String Templates（JDK 21 首次预览 → JDK 22 第二预览版）

```java
// JDK 22 String Templates（预览）
String name = "Alice";
int age = 30;

// STR 模板处理器：直接插值
String message = STR."Hello \{name}, you are \{age} years old.";

// FMT 模板处理器：格式化控制
String formatted = FMT."""
    Name:  \{name,-10}
    Age:   \{age,5} years
    Price: \{123.456,10.2f}
    """;

// 多行模板
String html = STR."""
    <div>
        <h1>\{title}</h1>
        <p>\{content}</p>
    </div>
    """;

// RAW + PROCESS 自定义模板处理器
TemplateProcessor processor = TemplateProcessor.of(
    (Template t, List<Object> values) -> {
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < t.interpolations().size(); i++) {
            sb.append(t.strings().get(i));
            sb.append(">>>").append(values.get(i)).append("<<<");
        }
        sb.append(t.strings().getLast());
        return sb.toString();
    }
);
String custom = RAW."Result: \{42}".process(processor);
```

### 1.5 Foreign Function & Memory API（JDK 22 第三预览版 → JDK 24 最终版）

纯 Java 调用本地原生代码，替代 JNI：

```java
import java.lang.foreign.*;

// 声明要调用的 C 函数
MemorySegment libc = SymbolLookup.libraryLookup("c", Arena.global());

FunctionDescriptor strLenType = FunctionDescriptor.of(ValueLayout.JAVA_LONG, ValueLayout.ADDRESS);
MethodHandle strlen = Linker.nativeLinker().downcallHandle(
    libc.find("strlen").orElseThrow(),
    strLenType
);

// 调用 C 的 strlen
try (Arena arena = Arena.ofConfined()) {
    MemorySegment cString = arena.allocateUtf8String("Hello Java 22!");
    long len = (long) strlen.invoke(cString);  // 输出 12
}
```

### 1.6 Implicitly Declared Classes and Instance Main Methods（JDK 22 第二预览版）

无需 `public class` 声明即可运行简单程序：

```java
// 隐式类：编译器自动生成类包装
void main() {
    System.out.println("Hello from implicit class!");
}
```

---

### 1.7 Statements before super()（JDK 22 第二预览版）

之前 `super()` 调用必须在构造方法第一行。JDK 22 允许在 `super()` 之前执行语句：

```java
class Parent {
    Parent(int x) { System.out.println("Parent: " + x); }
}

class Child extends Parent {
    Child() {
        int localVar = validate();          // ✅ JDK 22 允许
        super(localVar);                    // super() 不必是第一行
        postInit();
    }
}
```

---

### 1.8 Region Pinning for G1（JVM 内部改进）

G1 垃圾回收器的改进，减少 GC 期间因 synchronized 块导致的延迟（synchronized 块会临时"pin"对象在内存中，影响 GC）。

## JDK 23 新特性（2024年9月正式发布）

### 2.1 String Templates（第三预览版）

继续改进，添加了嵌入式表达式限制（security restrictions）。

### 2.2 Stream Gatherers（JDK 23 预览版）

Stream API 的自定义中间操作能力：

```java
// JDK 23 Stream Gatherers - 预览特性
import java.util.stream.Gatherer;

// 定义一个 window 收集器：按固定大小分组
Gatherer<Integer, ?, List<Integer>> fixedWindow(int size) {
    return Gatherer.ofSequential(
        () -> new ArrayList<>(size),
        (list, item, downstream) -> {
            list.add(item);
            if (list.size() == size) {
                downstream.push(new ArrayList<>(list));
                list.clear();
            }
            return true;
        },
        (list, downstream) -> {
            if (!list.isEmpty()) downstream.push(new ArrayList<>(list));
        }
    );
}

// 使用
List<List<Integer>> windows = IntStream.range(1, 10)
    .boxed()
    .gather(fixedWindow(3))
    .toList();
// 输出：[[1,2,3], [4,5,6], [7,8,9]]

// 内置 Gatherer
List<String> unique = Stream.of("a","b","a","c","b")
    .gather(Gatherer.ofGreedySet())  // 去重
    .toList();
```

### 2.5 Scoped Values（第二预览版）

比 `ThreadLocal` 更安全、更高效的跨线程数据共享机制（与虚拟线程配合）：

```java
// Scoped Values 类似于 ThreadLocal，但：
// 1. 不可变（更安全）
// 2. 跨虚拟线程传递时无需显式传递
// 3. 内存效率更高

static final ScopedValue<String> USER = ScopedValue.newInstance();

ScopedValue.where(USER, "Alice", () -> {
    // 在子虚拟线程中自动可见
    CompletableFuture.runAsync(() -> {
        System.out.println(USER.get());  // 输出 "Alice"
    });
});
```

### 2.6 Structured Concurrency（第二预览版 → JDK 21 第一版 → JDK 23 第二版）

```java
// JDK 23 Structured Concurrency
try (var scope = StructuredTaskScope.ShutdownOnFailure()) {
    Future<String>  user  = scope.fork(() -> findUser());
    Future<Integer> order = scope.fork(() -> fetchOrder());

    scope.join();           // 等待所有子任务完成
    scope.throwIfFailed();  // 任意一个失败就抛出

    System.out.println(user.resultNow() + ": " + order.resultNow());
}
```

## JDK 24 新特性（2025年3月正式发布）

### 3.1 压缩字符串（Preview → JDK 24 正式）

JDK 24 正式引入字符串内部编码压缩，大幅减少字符串内存占用：

```java
// 底层变化：Latin-1 字符串使用紧凑编码
// 之前：每个字符用 char（2 字节）
// 之后：Latin-1 字符用 byte（1 字节），Compact Strings 回归并增强

String latin = "Hello";        // LATIN-1 编码，5 bytes
String unicode = "你好";        // UTF-16 编码，4 chars（不变）

// API 无感知，但内存大幅节省
System.out.println(Runtime.getRuntime().freeMemory());  // 更多可用内存
```

### 3.2 Stream Gatherers（第三预览版 → JDK 24 最终版）

Stream 自定义中间操作正式化。

### 3.3 Foreign Function & Memory API（JDK 24 最终版）

FFM API 在 JDK 24 中正式成为标准 API，不再是预览版。

### 3.4 Scoped Values（JDK 24 最终版）

Scoped Values 正式成为标准 API。

### 3.5 Vector API（第九预览版）

向量计算 API，用于 SIMD 加速。

### 3.6 Class-File API（JDK 24 最终版）

Class-File API 正式化。

### 3.7 准备禁止动态加载的代理（JDK 24）

从 JDK 24 开始，禁止使用 `java.agent` 在运行时动态加载到已运行的 JVM（`-javaagent` 和 JVMTI 动态附加将被限制）。

## 面试关联

### Q1：JDK 22~25 中最重要的几个特性是什么？

**A：**
1. **Stream Gatherers（JDK 25）**：使 Stream API 可以自定义中间操作，是 JDK 8 Stream 最重要的补充
2. **压缩字符串（JDK 24~25）**：字符串内存占用大幅降低，对大型应用影响显著
3. **Foreign Function & Memory API（JDK 24）**：纯 Java 调用 native 代码，替代 JNI
4. **Scoped Values（JDK 24）**：替代 ThreadLocal 的更安全方案
5. **Class-File API（JDK 24）**：标准化字节码操作，替代 ASM

### Q2：为什么 JDK 的发布节奏变成了每六个月一个版本？

**A：**
JDK 9 将发布节奏从"特性驱动"改为"时间驱动"，每六个月发布一个版本（3月和9月）。这样确保了：
- 特性及时交付，不会无限延期
- 预览特性有足够时间稳定
- 企业可以更好地规划升级节奏
- LTS（长期支持）版本每两年一个（JDK 17/21/25）

### Q3：Scoped Values 和 ThreadLocal 的区别是什么？

**A：**

| 维度 | ThreadLocal | Scoped Values |
|------|------------|---------------|
| 可变性 | 可变，需手动清理 | 不可变，设置后不可修改 |
| 继承 | 子线程需要单独设置 | 虚拟线程自动继承父线程的值 |
| 内存效率 | 每个线程独立副本 | 共享只读值，无重复副本 |
| 适用场景 | 线程上下文数据 | 虚拟线程 + 结构化并发 |

### Q4：Stream Gatherers 和 Collector 的区别是什么？

**A：**
- `Collector` 是**终端操作**的结果收集器（在流的末端工作）
- `Gatherer` 是**中间操作**的函数式扩展（在流的中间工作，转换元素）

```java
// Gatherer: 中间操作，transforming
stream.gather(myGatherer).forEach(...);

// Collector: 终端操作，aggregating
stream.collect(myCollector);
```

### Q5：JDK 24 的压缩字符串对应用有什么影响？

**A：**
- 字符串内存占用预计降低 30~50%
- 主要影响 Latin-1 字符组成的字符串（英文为主的代码）
- 对中文字符影响较小（仍然需要 UTF-16）
- API 无变化，应用代码无需修改
