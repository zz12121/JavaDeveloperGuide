# 枚举类
## 核心结论

枚举（enum）是 Java 中用于表示一组固定常量的特殊类，本质上是**继承自 java.lang.Enum 的 final 类**。枚举天然线程安全、支持构造器、字段、方法，且可用于 switch 语句。

---

## 深度解析

### 1. 基本定义

```java
public enum Season {
    SPRING, SUMMER, AUTUMN, WINTER
}
```

### 2. 枚举的本质

反编译后，`Season` 实际上是一个 final 类：
```java
public final class Season extends Enum<Season> {
    public static final Season SPRING = new Season("SPRING", 0);
    public static final Season SUMMER = new Season("SUMMER", 1);
    public static final Season AUTUMN = new Season("AUTUMN", 2);
    public static final Season WINTER = new Season("WINTER", 3);
    // ...
}
```
每个枚举常量都是 `public static final` 的实例。

### 3. 枚举的高级用法

#### 带字段和构造器
```java
public enum HttpStatus {
    OK(200, "成功"),
    NOT_FOUND(404, "未找到"),
    SERVER_ERROR(500, "服务器错误");

    private final int code;
    private final String desc;

    HttpStatus(int code, String desc) {
        this.code = code;
        this.desc = desc;
    }

    public int getCode() { return code; }
    public String getDesc() { return desc; }
}
```

#### 带抽象方法
```java
public enum Operation {
    ADD { public double apply(double a, double b) { return a + b; } },
    SUB { public double apply(double a, double b) { return a - b; } },
    MUL { public double apply(double a, double b) { return a * b; } };

    public abstract double apply(double a, double b);
}
```

#### 实现接口
```java
public enum PayMethod implements Describable {
    ALIPAY {
        @Override
        public String describe() { return "支付宝支付"; }
    },
    WECHAT {
        @Override
        public String describe() { return "微信支付"; }
    };
}
```

### 4. 枚举常用方法

| 方法 | 说明 |
|------|------|
| `values()` | 返回所有枚举常量数组 |
| `valueOf(String)` | 根据名称获取枚举常量（不存在抛异常） |
| `name()` | 返回枚举常量名称 |
| `ordinal()` | 返回枚举常量的序号（从 0 开始） |
| `toString()` | 默认返回 name()，可重写 |

### 5. 枚举的优势

| 优势 | 说明 |
|------|------|
| 类型安全 | 编译期检查，不会传错值 |
| 不可变 | final 类 + private 构造器 |
| 线程安全 | 枚举常量在类加载时初始化，天然线程安全 |
| 支持 switch | 可直接用于 switch-case |
| 可扩展 | 支持字段、方法、实现接口 |

# 枚举实现接口
## 核心结论

枚举可以像普通类一样**实现接口**，每个枚举常量可以**分别实现**接口方法，从而实现多态行为——不同枚举值调用同一接口方法时表现出不同行为。

---

## 深度解析

### 1. 枚举实现接口

```java
public interface Describable {
    String describe();
}

public enum PayMethod implements Describable {
    ALIPAY {
        @Override
        public String describe() { return "支付宝支付"; }
    },
    WECHAT {
        @Override
        public String describe() { return "微信支付"; }
    },
    CREDIT_CARD {
        @Override
        public String describe() { return "信用卡支付"; }
    }
}
```

### 2. 使用场景

#### 策略模式

```java
public interface DiscountStrategy {
    BigDecimal apply(BigDecimal price);
}

public enum CustomerType implements DiscountStrategy {
    VIP {
        @Override
        public BigDecimal apply(BigDecimal price) {
            return price.multiply(new BigDecimal("0.8"));
        }
    },
    NORMAL {
        @Override
        public BigDecimal apply(BigDecimal price) {
            return price.multiply(new BigDecimal("0.95"));
        }
    },
    GUEST {
        @Override
        public BigDecimal apply(BigDecimal price) {
            return price; // 不打折
        }
    };
}

// 使用
BigDecimal finalPrice = CustomerType.VIP.apply(originalPrice);
```

#### 多接口实现

```java
public enum Status implements Serializable, Describable, Codeable {
    ACTIVE {
        public String describe() { return "活跃"; }
        public int getCode() { return 1; }
    },
    INACTIVE {
        public String describe() { return "停用"; }
        public int getCode() { return 0; }
    };
}
```

### 3. 两种实现方式对比

|方式|说明|适用场景|
|---|---|---|
|枚举类统一实现|枚举类 `implements` 接口，所有常量共享同一实现|所有枚举行为相同|
|每个常量分别实现（匿名内部类）|每个枚举常量 `{}` 中各自实现接口方法|每个枚举行为不同|

### 4. 注意事项

- 枚举不能继承其他类（已继承 Enum）
- 但可以同时实现多个接口
- 分别实现时，每个枚举常量实际上是一个**匿名内部类**


# 枚举单例模式
## 核心结论

**《Effective Java》推荐**使用枚举实现单例模式。枚举单例天然线程安全、防止反射攻击、防止序列化破坏，是实现单例的**最佳方式**。

---

## 深度解析

### 1. 实现

```java
public enum DatabaseConnection {
    INSTANCE;

    private Connection connection;

    private DatabaseConnection() {
        // 初始化连接
        this.connection = DriverManager.getConnection("jdbc:mysql://...");
    }

    public Connection getConnection() {
        return connection;
    }
}

// 使用
Connection conn = DatabaseConnection.INSTANCE.getConnection();
```

### 2. 为什么是最佳方式

#### 天然线程安全
JVM 保证枚举实例在类加载时初始化，且只初始化一次，天然线程安全。

#### 防止反射攻击
反射通过 `Constructor.setAccessible(true)` 绕过 private 构造器来创建新实例，但**枚举的构造器禁止反射调用**：
```java
Constructor<Singleton> constructor = Singleton.class.getDeclaredConstructor();
constructor.setAccessible(true);
constructor.newInstance(); // 抛出 IllegalArgumentException: Cannot reflectively create enum objects
```

#### 防止序列化破坏
普通单例在序列化/反序列化时会创建新实例，枚举单例不会：
```java
// 普通单例需要实现 readResolve() 才能防止序列化破坏
private Object readResolve() { return INSTANCE; }

// 枚举单例不需要，JVM 自动保证唯一性
```

### 3. 与其他单例实现对比

|方式|线程安全|防反射|防序列化|实现复杂度|
|---|---|---|---|---|
|饿汉式|✅|❌|❌|简单|
|懒汉式 + synchronized|✅|❌|❌|简单|
|DCL（双重检查锁）|✅|❌|❌|较复杂|
|静态内部类|✅|❌|❌|简单|
|**枚举**|✅|✅|✅|**最简单**|

### 4. 限制
- 不支持延迟初始化（枚举在类加载时就创建实例）
- 如果单例需要继承其他类，不能使用枚举方式
- 不太灵活，无法传参给构造器（枚举常量必须在类加载时确定）
