# 反射的原理与应用场景

## Q：什么是反射？它的原理是什么？

**A：** 反射是 Java 在运行时动态获取类的完整结构信息（字段、方法、构造器等）并操作对象的能力。

原理：JVM 在类加载阶段将 .class 文件解析为 `java.lang.Class` 对象，Class 对象中保存了类的所有元数据。反射通过 Class 对象在运行时动态操作这些信息。

## Q：反射能做什么？

**A：**
1. 运行时判断任意对象所属的类
2. 运行时实例化任意类的对象
3. 运行时调用任意对象的方法（包括私有方法）
4. 运行时读写任意对象的字段（包括私有字段）
5. 生成动态代理（AOP 的基础）
6. 读取运行时注解

## Q：反射在实际开发中有哪些应用场景？

**A：**
- **Spring**：IoC 容器通过反射创建 Bean 和注入依赖，AOP 通过反射+动态代理实现
- **ORM 框架**（MyBatis、Hibernate）：通过反射将数据库结果映射到 Java 对象
- **JSON 序列化**（Jackson、Fastjson）：通过反射读取对象字段生成 JSON
- **JUnit**：通过反射查找并执行 @Test 标注的方法
- **RPC 框架**（Dubbo）：通过反射调用服务方法
