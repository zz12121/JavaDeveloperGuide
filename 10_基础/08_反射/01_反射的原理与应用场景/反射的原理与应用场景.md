# 反射的原理与应用场景

## 核心结论

反射（Reflection）是 Java 在**运行时**动态获取类的信息（字段、方法、构造器）并操作对象的能力。Java 程序在加载类时，会将类的结构信息解析为 `java.lang.Class` 对象，反射就是通过这个 Class 对象操作类的内部成员。

## 深度解析

### 反射的原理

```
.java 源码 → 编译 → .class 字节码 → JVM 加载 → Class 对象（类元数据）
                                                    ↓
                                               反射操作
```

- JVM 在**类加载阶段**将 .class 文件解析为 `Class` 对象
- `Class` 对象中保存了类的完整结构信息：类名、父类、接口、字段、方法、构造器、注解等
- 反射通过 `Class` 对象在**运行时**动态获取和操作这些信息，突破访问权限限制

### 反射的核心类（java.lang.reflect 包）

| 类 | 作用 |
|----|------|
| `Class` | 类的元数据入口 |
| `Field` | 类的字段信息，可读写 |
| `Method` | 类的方法信息，可调用 |
| `Constructor` | 类的构造器信息，可创建对象 |
| `Annotation` | 注解信息 |
| `Array` | 数组操作 |

### 反射能做什么

1. **运行时判断**：判断任意对象所属的类
2. **运行时构造**：实例化任意类的对象
3. **运行时调用**：调用任意对象的方法（包括私有方法）
4. **运行时访问**：读写任意对象的字段（包括私有字段）
5. **动态代理**：生成代理类（AOP 的基础）
6. **注解处理**：读取运行时注解

### 反射的应用场景

| 场景 | 示例 |
|------|------|
| **Spring IoC** | 通过反射实例化 Bean、注入依赖 |
| **Spring AOP** | 通过反射（动态代理）实现切面逻辑 |
| **JDBC** | `ResultSet.getString()` 通过反射映射到 Java 对象 |
| **MyBatis** | `resultType` 自动映射结果集到实体类 |
| **JUnit** | 通过反射查找并执行 `@Test` 标注的方法 |
| **Jackson/Fastjson** | 通过反射序列化/反序列化 JSON |
| **Lombok** | 编译期通过注解处理器修改 AST |
