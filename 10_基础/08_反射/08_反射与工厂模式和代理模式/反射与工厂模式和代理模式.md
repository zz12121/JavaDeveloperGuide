# 反射与工厂模式/代理模式

## 核心结论

反射常与工厂模式和代理模式结合使用。在工厂模式中，反射可以动态实例化对象，避免 if-else 分支；在代理模式中，反射（`Method.invoke()`）是 JDK 动态代理的底层实现机制。

## 深度解析

### 反射 + 工厂模式

传统工厂模式的问题：每次新增产品都需要修改工厂类，违反开闭原则。

```java
// 传统工厂：每新增一个产品就要改这里
public class Factory {
    public static Product create(String type) {
        if ("A".equals(type)) return new ProductA();
        if ("B".equals(type)) return new ProductB();
        if ("C".equals(type)) return new ProductC(); // 新增分支
        throw new IllegalArgumentException();
    }
}
```

使用反射改进：

```java
// 反射工厂：新增产品不需要修改工厂
public class ReflectFactory {
    public static <T> T create(Class<T> clazz) {
        try {
            return clazz.getDeclaredConstructor().newInstance();
        } catch (Exception e) {
            throw new RuntimeException("创建实例失败: " + clazz.getName(), e);
        }
    }
}

// 使用
ProductA a = ReflectFactory.create(ProductA.class);
ProductB b = ReflectFactory.create(ProductB.class);
```

配置文件驱动的反射工厂：

```java
// config.properties: product=com.example.ProductA
public class ConfigFactory {
    public static Product create() throws Exception {
        Properties props = new Properties();
        props.load(new FileInputStream("config.properties"));
        String className = props.getProperty("product");
        Class<?> clazz = Class.forName(className);
        return (Product) clazz.getDeclaredConstructor().newInstance();
    }
}
```

### 反射 + 动态代理（JDK Proxy）

JDK 动态代理的底层核心就是反射调用 `Method.invoke()`：

```java
// 1. 定义接口
public interface UserService {
    void save(String name);
}

// 2. 实现类
public class UserServiceImpl implements UserService {
    public void save(String name) { System.out.println("保存用户: " + name); }
}

// 3. 创建动态代理
public class ProxyDemo {
    public static void main(String[] args) {
        UserService realService = new UserServiceImpl();

        UserService proxy = (UserService) Proxy.newProxyInstance(
            realService.getClass().getClassLoader(),
            realService.getClass().getInterfaces(),
            (proxyObj, method, args) -> {
                System.out.println("前置增强: " + method.getName());
                Object result = method.invoke(realService, args); // 反射调用
                System.out.println("后置增强");
                return result;
            }
        );

        proxy.save("张三");
        // 输出：
        // 前置增强: save
        // 保存用户: 张三
        // 后置增强
    }
}
```

### JDK 动态代理 vs CGLIB 代理

| 对比项 | JDK 动态代理 | CGLIB 代理 |
|--------|-------------|-----------|
| 实现方式 | 基于接口（`Proxy` + `InvocationHandler`） | 基于继承（生成子类） |
| 要求 | 目标类必须实现接口 | 目标类不能是 final 类 |
| 底层 | 反射调用 `Method.invoke()` | FastClass 机制，性能更好 |
| Spring 默认 | 有接口时使用 | 无接口时使用 |
