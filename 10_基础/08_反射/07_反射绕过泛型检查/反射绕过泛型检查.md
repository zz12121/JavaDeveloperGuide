# 反射绕过泛型检查

## 核心结论

Java 泛型在编译期发生**类型擦除**，运行时泛型信息不存在。通过反射可以在运行时绕过泛型类型检查，向泛型集合中添加不匹配的类型。

## 深度解析

### 泛型擦除导致绕过成为可能

```java
// 编译后，ArrayList<String> 和 ArrayList<Integer> 都是 ArrayList
List<String> list = new ArrayList<>();
// 编译后等价于
ArrayList list = new ArrayList();
```

因为泛型类型在运行时被擦除，JVM 无法在运行时做泛型类型检查。

### 反射绕过泛型示例

```java
List<String> stringList = new ArrayList<>();
stringList.add("hello");

// stringList.add(123); // 编译报错：类型不匹配

// 通过反射绕过编译期检查
Class<?> clazz = stringList.getClass();
Method addMethod = clazz.getMethod("add", Object.class);
addMethod.invoke(stringList, 123);  // 运行时成功添加 Integer

// 遍历时会出问题
for (String s : stringList) {
    System.out.println(s); // 输出 hello，然后抛出 ClassCastException
}
```

### 绕过的原理

```
编译期：List<String> → 泛型检查 → add(Integer) 编译报错
运行时：List → 类型擦除为 List → add(Object) → 反射调用无泛型限制
```

反射调用的是**原始类型**（Raw Type）的 `add(Object)` 方法，该方法没有泛型约束。

### 字段级别的反射绕过

```java
public class GenericHolder<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}

GenericHolder<String> holder = new GenericHolder<>();
holder.set("hello");

// 通过反射将 Integer 写入 String 类型的 Holder
Field field = GenericHolder.class.getDeclaredField("value");
field.setAccessible(true);
field.set(holder, 42);  // 绕过泛型
System.out.println(holder.get()); // 输出 42（如果转为 String 则抛出 ClassCastException）
```

### 安全建议

- 反射绕过泛型会破坏类型安全，导致运行时 `ClassCastException`
- 框架内部（如 Jackson、MyBatis）会谨慎使用这种能力
- 开发中应避免手动使用反射绕过泛型
