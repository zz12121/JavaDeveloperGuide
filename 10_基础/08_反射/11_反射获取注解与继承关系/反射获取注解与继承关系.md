# 反射获取注解与继承关系

## 两种注解获取方法的核心区别

这是最容易混淆的面试点：

| 方法 | 返回值 | 包含继承的注解 | 包含间接声明的注解 |
|------|:------:|:---:|:---:|
| `getAnnotation(Class)` | `T` | ✅ | ✅ |
| `getAnnotations()` | `Annotation[]` | ✅ | ✅ |
| `getDeclaredAnnotation(Class)` | `T` | ❌ | ❌ |
| `getDeclaredAnnotations()` | `Annotation[]` | ❌ | ❌ |

**带 `Declared` 的只查自身直接声明的，不查继承的。**

## 源码分析

### getAnnotation() 源码

```java
public <A extends Annotation> A getAnnotation(Class<A> annotationClass) {
    // 调用 inheritedString，构建完整的注解继承链
    MemberDecl c = memberContainer();
    AnnotationBus<?> bus = AnnotationBus.getFor(c, getSuperclass());
    return bus.findMerged(annotationClass);
}
```

### getDeclaredAnnotation() 源码

```java
public <A extends Annotation> A getDeclaredAnnotation(Class<A> annotationClass) {
    // 直接从自身 Annotations 数组中查找
    // 不涉及任何父类或接口的继承
    return declaredAnnotations().get(annotationClass);
}
```

## 继承 vs 直接声明：图解

```
class Parent {
    @MyAnno  // 父类直接标注
    public void parentMethod() {}
}

class Child extends Parent {
    @MyAnno  // 子类直接标注
    public void childMethod() {}
}
```

| 调用场景 | getAnnotation() | getDeclaredAnnotation() |
|---------|:---:|:---:|
| `parentMethod` 上找 `@MyAnno` | ✅ 找到 | ✅ 找到 |
| `childMethod` 上找 `@MyAnno` | ✅ 找到 | ✅ 找到 |
| `childMethod` 上找 `@MyAnno`（继承） | ✅ 找到 | ❌ 不找父类 |

## 实际应用场景

### 1. Spring MVC 请求映射

```java
// Spring 通过 getDeclaredAnnotations() 找 @GetMapping / @PostMapping
// 避免父类的 @RequestMapping 被误识别
Method method = handler.getClass().getDeclaredMethod("handle");
RequestMapping rm = method.getDeclaredAnnotation(RequestMapping.class);
```

### 2. MyBatis @Select 注解解析

```java
// MyBatis 只看当前方法上的 @Select，不继承
Method method = mapperInterface.getDeclaredMethod(methodName);
Select select = method.getDeclaredAnnotation(Select.class);
// 不走 getAnnotation()，因为接口方法不继承注解
```

### 3. 自定义权限校验框架

```java
@RequiresPermissions("user:read")
public User getUser(Long id) { return userService.getById(id); }

// 检查方法上的权限注解（不含继承）
RequiresPermissions rp =
    method.getDeclaredAnnotation(RequiresPermissions.class);
if (rp == null) return;  // 方法级没有，查类级

RequiresPermissions classRp =
    method.getDeclaringClass().getDeclaredAnnotation(RequiresPermissions.class);
```

## 与 @Inherited 的关系

`@Inherited` 只对**类**有意义，对方法/字段无效：

```java
@Inherited  // 元注解：子类继承此类上的注解
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnno {}

// 父类
@MyAnno
class Parent {}

// 子类（继承 Parent）
class Child extends Parent {}
// Child.class.getAnnotation(MyAnno.class) → 找到！✅
// Child.class.getDeclaredAnnotation(MyAnno.class) → 找不到！❌
```

| 场景 | getAnnotation | getDeclaredAnnotation |
|------|:---:|:---:|
| 继承的类级注解（@Inherited） | ✅ | ❌ |
| 直接的类级注解 | ✅ | ✅ |
| 方法/字段上的注解（不管 @Inherited） | ✅ | ✅（只查自身）|

## 综合对比表

```java
// 场景：父类 Parent 有 @A，子类 Child 没有
@A
class Parent {}

class Child extends Parent {}

// 父类
Parent.class.getAnnotation(A.class)             // ✅ 找到
Parent.class.getDeclaredAnnotation(A.class)      // ✅ 找到

// 子类（@Inherited 生效）
Child.class.getAnnotation(A.class)               // ✅ 找到（继承）
Child.class.getDeclaredAnnotation(A.class)       // ❌ 不找父类

// 方法
parentMethod 上 getAnnotation(A.class)           // ✅ 找到
parentMethod 上 getDeclaredAnnotation(A.class)   // ✅ 找到（方法不考虑 @Inherited）
```
