# 108 函数式接口

## 核心结论

函数式接口（Functional Interface）是**只包含一个抽象方法**的接口，可以作为 Lambda 表达式的类型。`@FunctionalInterface` 注解用于编译期检查，不写也不影响使用。函数式接口可以有 default 方法、static 方法。

## 深度解析

### 函数式接口的定义

```java
// 自定义函数式接口
@FunctionalInterface
public interface MyFunction<T, R> {
    R apply(T t);  // 唯一的抽象方法

    // ✅ default 方法不影响
    default void print(T t) {
        System.out.println(t);
    }

    // ✅ static 方法不影响
    static <T> MyFunction<T, T> identity() {
        return t -> t;
    }
}
```

### Lambda 与函数式接口的关系

Lambda 表达式必须有**目标类型（Target Type）**，即函数式接口的类型：

```java
// Lambda 的类型就是函数式接口
Runnable r = () -> System.out.println("run");         // Runnable
Comparator<String> c = (a, b) -> a.length() - b.length(); // Comparator
Predicate<Integer> p = x -> x > 0;                    // Predicate
```

### JDK 内置的函数式接口

除了四大核心接口外，JDK 还提供了大量函数式接口：

| 接口 | 抽象方法 | 说明 |
|------|---------|------|
| `Runnable` | `void run()` | 无参无返回 |
| `Callable<V>` | `V call()` | 无参有返回（可抛异常） |
| `Comparator<T>` | `int compare(T, T)` | 比较器 |
| `Consumer<T>` | `void accept(T)` | 消费者 |
| `Supplier<T>` | `T get()` | 供给者 |
| `Predicate<T>` | `boolean test(T)` | 断言 |
| `Function<T,R>` | `R apply(T)` | 函数 |
| `BiFunction<T,U,R>` | `R apply(T,U)` | 双参数函数 |
| `UnaryOperator<T>` | `T apply(T)` | 一元操作（Function 子接口） |
| `BinaryOperator<T>` | `T apply(T,T)` | 二元操作 |

### 函数式接口的继承

```java
// 函数式接口可以被继承，只要抽象方法总数仍为 1
@FunctionalInterface
public interface MyFunction<T> extends Function<T, T> {
    // Function 已有 apply 方法，不新增抽象方法 → 仍是函数式接口
}
```
