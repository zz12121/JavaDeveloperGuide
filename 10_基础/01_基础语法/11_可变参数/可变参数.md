# 可变参数（Varargs）

## 先说结论

可变参数用 `Type... args` 声明，本质是**数组**。一个方法只能有一个可变参数且必须放在最后。方法重载时优先级：**精确匹配 > 自动装箱/拆箱 > 可变参数**。

## 深度解析

### 语法
```java
public void print(String... args) { }     // 可变参数
public void print(int a, String... args) { } // 可变参数放最后
```

### 编译器实现
- `Type... args` 编译后就是 `Type[] args`，本质是数组
- 调用时可以传数组，也可以传逗号分隔的多个值
- 调用时不传参数，数组长度为 0（不是 null）

### 使用限制
- 一个方法**只能有一个**可变参数
- 可变参数**必须是最后一个**参数
- 可变参数可以与普通参数共存

### 重载优先级
当调用存在歧义时，编译器按以下优先级选择：
1. **精确匹配**（参数类型完全一致）
2. **自动装箱/拆箱**
3. **可变参数**（最低优先级）

## 易错点/踩坑

- ❌ 可变参数传 null 的陷阱：`f(null)` 如果同时有 `f(String)` 和 `f(String...)` 重载，编译器无法确定
- ❌ 可变参数与数组重载冲突：`f(int...)` 和 `f(int[])` 签名相同，不能同时存在
- ❌ `f(int...)` 不传参数时 args 不是 null，是空数组（length = 0）

## 代码示例

```java
// ========== 基本用法 ==========
public static int sum(int... nums) {
    int total = 0;
    for (int num : nums) {
        total += num;
    }
    return total;
}

sum(1, 2, 3);      // 6
sum();              // 0（空数组）
sum(new int[]{1, 2}); // 3（也可以直接传数组）

// ========== 可变参数 + 普通参数 ==========
public static void print(String prefix, String... items) {
    System.out.println(prefix);
    for (String item : items) {
        System.out.println("  - " + item);
    }
}
print("水果：", "苹果", "香蕉", "橙子");

// ========== 重载优先级 ==========
public static void f(int i) { System.out.println("精确匹配"); }
public static void f(Integer i) { System.out.println("装箱"); }
public static void f(int... args) { System.out.println("可变参数"); }

f(1);              // "精确匹配"
f(new Integer(1)); // "装箱"（精确匹配 Integer）
// f()              // "可变参数"

// ========== null 陷阱 ==========
public static void g(String s) { System.out.println("String"); }
public static void g(String... ss) { System.out.println("String..."); }

// g(null);         // 编译错误！歧义调用
g((String) null);  // "String"（显式类型转换消除歧义）
```

## 图解/流程

```
可变参数编译过程：

  源码：sum(1, 2, 3)
           │
           ▼ 编译器转换
  字节码：sum(new int[]{1, 2, 3})
           │
           ▼ 方法签名
  public static int sum(int[] nums)  ← 本质就是数组参数

重载匹配优先级：

  f(1)
    │
    ├── f(int) 精确匹配？ ──── 是 → 调用 f(int)
    │
    ├── f(Integer) 装箱？ ──── 是 → 调用 f(Integer)
    │
    └── f(int...) 可变参数？ ── 是 → 调用 f(int...)
```
