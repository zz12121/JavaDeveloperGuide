# 装箱与拆箱（Auto Boxing/Unboxing）

## 先说结论

装箱是将基本类型转为包装类（`Integer.valueOf()`），拆箱是反向操作（`xxxValue()`）。Java 5 引入自动装箱/拆箱语法糖，但 **Integer 缓存池（-128~127）** 导致 == 比较在此范围内返回 true，超出则为 false，这是面试最高频考点。

## 深度解析

### 装箱与拆箱原理
- **装箱（Boxing）**：基本类型 → 包装类，底层调用 `Integer.valueOf()` 等方法
- **拆箱（Unboxing）**：包装类 → 基本类型，底层调用 `intValue()` 等方法
- 自动装箱/拆箱是**编译器语法糖**，编译后仍然是方法调用

### Integer 缓存池（核心考点）
- `Integer.valueOf()` 对 **-128~127** 范围内的值返回缓存对象
- 超出范围则创建新对象
- `Byte`、`Short`、`Long`、`Character` 也有类似缓存（各自范围不同）
- `Float`、`Double` 没有缓存

### 各包装类缓存范围

| 包装类 | 缓存范围 | 说明 |
|--------|---------|------|
| `Byte` | -128 ~ 127 | 全部缓存 |
| `Short` | -128 ~ 127 | 部分缓存 |
| `Integer` | -128 ~ 127 | 部分缓存（可通过 -XX:AutoBoxCacheMax 调整上限） |
| `Long` | -128 ~ 127 | 部分缓存 |
| `Character` | 0 ~ 127 | 部分缓存 |
| `Float` | 无 | 不缓存 |
| `Double` | 无 | 不缓存 |
| `Boolean` | true/false | 只有两个值，全部缓存 |

## 易错点/踩坑

- ❌ `Integer a = 127; Integer b = 127; a == b` → `true`（缓存池）
- ❌ `Integer a = 128; Integer b = 128; a == b` → `false`（超出缓存，不同对象）
- ❌ `Integer a = 100; Integer b = 100; a == b` → `true`，但 `a.equals(b)` 才是正确比较方式
- ❌ 拆箱空指针：`Integer a = null; int b = a;` → **NullPointerException**（自动拆箱调 `a.intValue()`）
- ❌ `Integer` 和 `int` 混用比较时，`Integer` 会自动拆箱为 `int`，用  == 比值而非比地址

## 代码示例

```java
// ========== Integer 缓存池 ==========
Integer a1 = 127;
Integer a2 = 127;
System.out.println(a1 == a2);     // true（缓存池同一对象）
System.out.println(a1.equals(a2)); // true

Integer b1 = 128;
Integer b2 = 128;
System.out.println(b1 == b2);     // false（超出缓存，不同对象）
System.out.println(b1.equals(b2)); // true

// ========== 拆箱空指针 ==========
Integer x = null;
int y = x;  // NullPointerException！自动拆箱调 x.intValue()

// ========== 自动装箱拆箱的底层 ==========
Integer num = 10;           // 编译为 Integer.valueOf(10)
int val = num;              // 编译为 num.intValue()
Integer sum = num + 5;     // num 拆箱为 int，加 5 后装箱为 Integer

// ========== 面试高频题 ==========
Integer c1 = new Integer(100);
Integer c2 = new Integer(100);
System.out.println(c1 == c2);     // false（new 总是创建新对象）
System.out.println(c1.equals(c2)); // true
```

## 图解/流程

```
自动装箱/拆箱流程：

  int i = 10;
  Integer num = i;    ────编译器转换───→ Integer num = Integer.valueOf(i);
  int j = num;        ────编译器转换───→ int j = num.intValue();

Integer.valueOf() 内部逻辑：

  value 在 -128~127？
      │
      ├── 是 → 返回 IntegerCache.cache[] 中的缓存对象
      │
      └── 否 → new Integer(value)，每次创建新对象

  ┌──────────────────────────────────────┐
  │         IntegerCache 缓存池          │
  │  [-128, -127, ..., 126, 127]        │
  │   ↑                               ↑  │
  │ cache[0]                    cache[255]│
  └──────────────────────────────────────┘
```
