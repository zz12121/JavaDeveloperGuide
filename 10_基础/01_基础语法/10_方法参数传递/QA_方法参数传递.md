# 方法参数传递（值传递 vs 引用传递）

## Q1：Java 只有值传递？

**A**：
- 基本类型传值的副本
- 引用类型传地址的副本（仍然是值传递）
- Java 不存在引用传递（即不能在方法内改变调用者的引用指向）

---

## Q2：引用类型传递的效果？

**A**： 
- 修改对象属性 → 影响原对象（两个引用指向同一对象）
- 重新赋值（`p = new Person()`）→ 不影响原引用（副本指向了新对象）
- String 不可变：方法内"修改"String 实际是创建新对象

---

## Q3：经典面试题？

**A**：
- `swap(int a, int b)` 不能交换基本类型变量
- 原因：交换的是副本，原变量不受影响

---

## Q4：Java 是值传递还是引用传递？

**A**：Java 只有值传递。基本类型传递值的副本；引用类型传递引用地址的副本。
方法内修改引用指向的对象属性会影响原对象，但重新赋值引用不会影响原引用。
"引用传递"意味着可以修改调用者的引用本身，Java 不支持。

---

## Q5：为什么引用类型传的是地址副本，修改对象会影响原对象？

**A**：因为引用类型的"值"就是堆中对象的地址。传递地址的副本后，方法内参数和外部引用都指向同一个堆对象。
所以通过参数修改对象属性，外部也能看到变化。
但如果在方法内 `p = new Object()`，只是让参数副本指向了新对象，外部引用不变。

---

## Q6：String 是引用类型，为什么方法内修改不影响外部？

**A**：因为 String 是不可变类（final + private char[]）。方法内 `s = "world"` 实际上让参数引用指向了一个新的 String 对象，外部引用仍然指向原来的 "hello"。所有对 String 的"修改"操作都是创建新对象。
