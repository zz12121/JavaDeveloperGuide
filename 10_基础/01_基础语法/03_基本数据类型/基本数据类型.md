# 基本数据类型（8种：byte/short/int/long/float/double/boolean/char）

## 先说结论

- Java 有 **8 种基本数据类型**，分为四大类：整型（byte/short/int/long）、浮点型（float/double）、字符型（char）、布尔型（boolean）。基本类型不是对象，存于栈上（局部变量时），效率高；对应的包装类存于堆上。
- Java 基本类型的默认值只在**成员变量**（类字段）时自动赋值；**局部变量**不会自动初始化，使用前必须赋值，否则编译报错。8 种基本类型的字节大小是固定的，不随平台变化（区别于 C/C++）。

## 深度解析

### 8 种基本类型一览

| 类型        | 大小               | 默认值       | 范围                   | 包装类       |
| --------- | ---------------- | --------- | -------------------- | --------- |
| `byte`    | 1 字节             | 0         | -128 ~ 127           | Byte      |
| `short`   | 2 字节             | 0         | -32768 ~ 32767       | Short     |
| `int`     | 4 字节             | 0         | -2³¹ ~ 2³¹-1（约 ±21亿） | Integer   |
| `long`    | 8 字节             | 0L        | -2⁶³ ~ 2⁶³-1         | Long      |
| `float`   | 4 字节             | 0.0f      | 约 ±3.4E38（6~7位有效数字）  | Float     |
| `double`  | 8 字节             | 0.0d      | 约 ±1.7E308（15位有效数字）  | Double    |
| `char`    | 2 字节             | '\u0000'  | 0 ~ 65535（Unicode）   | Character |
| `boolean` | 1 bit（JVM按int实现） | false     | true / false         | Boolean   |
| 引用类型      | null             | 4/8（指针大小） |                      |           |

### 关键细节
- **整型默认是 int**：`long l = 2147483648;` 编译报错，必须加 `L` 后缀：`long l = 2147483648L;`
- **浮点默认是 double**：`float f = 3.14;` 编译报错，必须加 `F` 后缀：`float f = 3.14F;`
- **char 是无符号 16 位整数**：可以参与算术运算，`char c = 65;` 等价于 `char c = 'A';`
- **boolean 不能参与数值运算**：不能与 int 互转（不同于 C/C++）

### 基本类型 vs 包装类

| 维度   | 基本类型               | 包装类              |
| :--- | :----------------- | :--------------- |
| 存储位置 | 栈(局部变量)<br>堆(成员变量) | 堆                |
| 默认值  | 各类型零值              | null             |
| 比较方式 | == 比值              | == 比地址 equals 比值 |
| 性能   | 高(直接存值)            | 低(对象开销)          |
| 泛型支持 | 不支持                | 支持               |

### 成员变量 vs 局部变量
- 成员变量在对象创建时由 JVM 自动初始化为零值（堆内存默认清零）
- 局部变量在方法栈帧中，JVM 不会自动清零，要求程序员显式初始化

### 为什么 Java 基本类型大小固定？
- Java 设计目标：**跨平台**（Write Once, Run Anywhere）
- C/C++ 的 `int` 大小随平台变化（16/32 位），Java 固定 int 为 4 字节
- 固定大小使得字节码行为一致，JVM 不需要针对不同平台做类型适配

## 易错点/踩坑

- ❌ `float f = 3.14;` 编译报错：浮点字面量默认是 double，必须写 `3.14F`
- ❌ `char` 不是 C 的 char：Java char 是 2 字节 Unicode，不是 1 字节 ASCII
- ❌ `boolean` 不能转 int：Java 不允许 `boolean x = true; int y = x;`（C/C++ 可以）
- ❌ 基本类型数组不是"基本类型"：`int[]` 是对象，存在堆上
- ❌ 局部变量使用前不赋值：编译直接报错，不是运行时才出问题
- ❌ char 默认值 '\u0000' 不是空格也不是 null：是 Unicode 编码 0，即空字符
- ❌ boolean 的默认值是 false，不是 0（Java 中 boolean 不能和 int 互转）
- ❌ 引用类型默认值是 null，不是空对象：调用 null 的方法会抛 NullPointerException

## 代码示例

```java
// 后缀陷阱
long l = 2147483648;    // ❌ 编译错误：超出 int 范围
long l2 = 2147483648L;  // ✅ 加 L 后缀
float f = 3.14;         // ❌ 编译错误：double → float 需要强转
float f2 = 3.14F;       // ✅ 加 F 后缀

// char 参与运算
char c = 'A';           // 65
int n = c + 1;          // 66
char c2 = (char) n;     // 'B'
```

```java
public class Demo {
    int a;          // 成员变量，默认值 0 ✅
    boolean flag;   // 成员变量，默认值 false ✅
    String name;    // 成员变量，默认值 null ✅

    public void test() {
        int b;      // 局部变量，未初始化 ❌ 编译报错
        System.out.println(b); // Variable 'b' might not have been initialized
    }
}
```

```java
// 引用类型默认值是 null
public class Demo2 {
    String name;    // 默认 null
    int[] arr;      // 默认 null（不是空数组）

    public void test() {
        // System.out.println(name.length()); // ❌ NullPointerException
        System.out.println(name); // null
    }
}
```
## 图解/流程

```
Java 数据类型
├── 基本类型（8种，栈上）
│   ├── 整型：byte(1) → short(2) → int(4) → long(8)
│   ├── 浮点型：float(4) → double(8)
│   ├── 字符型：char(2, Unicode)
│   └── 布尔型：boolean(~1bit)
│
└── 引用类型（堆上）
    ├── 类（class）
    ├── 接口（interface）
    └── 数组（array[]）
```

```
变量初始化规则
├── 成员变量（类字段）
│   ├── 基本类型 → JVM 自动赋零值（0 / 0.0 / false / '\u0000'）
│   └── 引用类型 → JVM 自动赋 null
│   时机：对象创建时，堆内存分配后自动完成
│
└── 局部变量（方法内）
    ├── 基本类型 → 不自动初始化 ❌
    └── 引用类型 → 不自动初始化 ❌
    要求：使用前必须显式赋值，否则编译报错
```
