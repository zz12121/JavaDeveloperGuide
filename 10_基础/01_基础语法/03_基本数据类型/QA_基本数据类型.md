# 基本数据类型（8种：byte/short/int/long/float/double/boolean/char）

## Q1：8 种基本类型一览？

**A**：

| 类型 | 大小 | 默认值 | 范围 | 包装类 |
| :--- | :--- | :--- |
| `byte` | 1 字节 | 0 | -128 ~ 127 | Byte |
| `short` | 2 字节 | 0 | -32768 ~ 32767 | Short |
| `int` | 4 字节 | 0 | -2³¹ ~ 2³¹-1（约 ±21亿） | Integer |
| `long` | 8 字节 | 0L | -2⁶³ ~ 2⁶³-1 | Long |
| `float` | 4 字节 | 0.0f | 约 ±3.4E38（6~7位有效数字） | Float |
| `double` | 8 字节 | 0.0d | 约 ±1.7E308（15位有效数字） | Double |
| `char` | 2 字节 | '\u0000' | 0 ~ 65535（Unicode） | Character |
| `boolean` | 1 bit（JVM按int实现） | false | true / false | Boolean |

---

## Q2：关键细节？

**A**： - **整型默认是 int**：`long l = 2147483648;` 编译报错，必须加 `L` 后缀
- **浮点默认是 double**：`float f = 3.14;` 编译报错，必须加 `F` 后缀
- **char 是无符号 16 位整数**：可以参与算术运算
- **boolean 不能参与数值运算**：不能与 int 互转（不同于 C/C++）

---

## Q3：基本类型 vs 包装类

**A**：

| 维度 | 基本类型 | 包装类 |
| :--- | :--- | :--- |
| 存储位置 | 栈 或 堆 | 堆 |
| 默认值 | 零值 | null |
| 比较方式 | == 比值 | == 比地址 equals 比值 |
| 性能 | 高 | 低 |
| 泛型支持 | 不支持 | 支持 |

---

## Q4：为什么 Java 保留基本类型而不全用包装类？

**A**：性能原因。基本类型存在栈上，没有对象头和 GC 开销；包装类存在堆上，有 16 字节对象头开销。在大量数值计算场景下，基本类型的性能优势显著。

---

## Q5：boolean 在 JVM 中到底占多大空间？

**A**：JVM 规范没有明确规定。HotSpot 中，boolean 数组每个元素占 1 byte；单个 boolean 变量在栈上按 int（4字节）处理。原因：JVM 没有专用的 boolean 字节码指令，用 int 的 0/1 表示。

---

## Q6：为什么 float 精度只有 6~7 位有效数字？

**A**：float 的 32 位中，1 位符号位 + 8 位指数位 + 23 位尾数位，23 位尾数约对应 log₁₀(2²⁴) ≈ 7.22 位十进制精度。

---

## Q7：为什么 Java 基本类型大小固定？

**A**： - Java 设计目标：**跨平台**（Write Once, Run Anywhere）
- C/C++ 的 `int` 大小随平台变化（16/32 位），Java 固定 int 为 4 字节
- 固定大小使得字节码行为一致，JVM 不需要针对不同平台做类型适配

---

## Q8：为什么局部变量不自动初始化？

**A**：
1）性能：方法调用频繁，自动清零有开销；
2）安全：强制程序员显式赋值，减少因使用随机值导致的 bug。

---

## Q9：char 的默认值 '\u0000' 代表什么？

**A**：Unicode 编码 0，即空字符（null character），不是空格也不是 null。打印时通常不可见。

---

## Q10：引用类型在 32 位和 64 位 JVM 中占多大？

**A**：32 位 JVM 中引用占 4 字节；64 位 JVM 中默认占 8 字节，开启指针压缩（-XX:+UseCompressedOops，JDK 6u23+ 默认开启）后堆内存 ≤ 32G 时仍为 4 字节。
