# 自动类型转换与强制类型转换

## 先说结论

Java 中小范围类型可以**自动**转为大范围类型（自动提升），大范围转小范围必须**显式强转**，且可能丢失精度；
表达式运算时 byte/short/char 会自动提升为 int。

## 深度解析

### 自动类型转换（隐式 widen）
- 也称**拓宽转换（widening conversion）**，由编译器自动完成
- 转换方向从小到大，数据完全兼容，**不会丢失精度**（float/double 例外见下文）
- 转换链路：
  - `byte → short → int → long → float → double`
  - `char → int → long → float → double`

### 强制类型转换（显式 narrow）
- 也称**窄化转换（narrowing conversion）**，必须手动书写转换目标类型
- 语法：`(目标类型) 值`
- 大范围 → 小范围，**可能丢失精度或溢出**

### 表达式中的自动提升
- `byte`、`short`、`char` 三者之间运算，结果至少为 `int`
- `int` 与 `float` 运算，`int` 提升为 `float`
- 任何操作数中只要有一个是 `double`，结果就是 `double`

### 隐式精度丢失的特殊情况
- `int → float`：float 只有 23 位尾数，int 有 32 位，大整数可能丢失低位
- `long → float`：long 64 位 → float 23 位尾数，丢失更多
- `long → double`：double 只有 52 位尾数，long 超过 2^53 的值会丢失精度
- **编译器不会报错，但数据可能已经不精确**

## 易错点/踩坑

- ❌ 误区1：认为自动类型转换永远不会丢失精度 → `int` 转 `float`、`long` 转 `float/double` 可能丢失精度
- ❌ 误区2：`byte b1 = 1; byte b2 = 2; byte b3 = b1 + b2;` → 编译错误！`b1 + b2` 结果是 `int`，需强转 `(byte)(b1 + b2)`
- ❌ 误区3：`short s = s + 1;` → 编译错误！`s + 1` 结果是 `int`；但 `s += 1` 是合法的（`+=` 隐含了强转）
- ❌ 误区4：强转浮点数时只截断，不四舍五入 → `(int)3.9` 结果是 `3`，不是 `4`
- ❌ 误区5：字符参与运算时忘记 char 会提升为 int → `'A' + 1` 结果是 `66`（int），不是字符 `'B'`

## 代码示例

```java
// ========== 自动类型转换 ==========
byte b = 10;
short s = b;       // byte → short，自动
int i = s;         // short → int，自动
long l = i;        // int → long，自动
float f = l;       // long → float，自动（可能丢失精度！）
double d = f;      // float → double，自动

char c = 'A';
int num = c;       // char → int，自动，num = 65

// ========== 强制类型转换 ==========
double pi = 3.14159;
int intPi = (int) pi;         // intPi = 3，截断小数部分
int big = 300;
byte small = (byte) big;      // small = 44，溢出（300 % 256 = 44）

// ========== 表达式自动提升 ==========
byte b1 = 10;
byte b2 = 20;
// byte b3 = b1 + b2;         // 编译错误！结果是 int
byte b3 = (byte) (b1 + b2);   // 正确，显式强转

short s1 = 1;
// s1 = s1 + 1;               // 编译错误！s1 + 1 结果是 int
s1 += 1;                      // 正确，+= 隐含强转（等价于 s1 = (short)(s1 + 1)）

// ========== 隐式精度丢失 ==========
int bigInt = 123456789;
float fromInt = bigInt;
System.out.println(fromInt);  // 1.23456792E8（末尾精度丢失）
System.out.println(bigInt == (int) fromInt); // false!
```

## 图解/流程

```
自动类型转换方向（小 → 大，编译器自动完成）：

  ┌──────┐    ┌──────┐    ┌─────┐    ┌──────┐    ┌──────┐    ┌───────┐
  │ byte │───>│short │───>│ int │───>│ long │───>│float │───>│double │
  │  8位  │    │ 16位  │    │ 32位│    │ 64位 │    │ 32位 │    │  64位 │
  └──┬───┘    └──────┘    └─────┘    └──────┘    └──┬───┘    └───────┘
     │                                                 │
     │            ┌─────┐                              │
     └───────────>│char │──────────────────────────────>│
                  │ 16位 │                              │
                  └─────┘                              │
                                                       │
  ⚠️ float/double 虽然范围更大，但尾数位有限，
     int→float / long→float / long→double 可能丢失精度

强制类型转换方向（大 → 小，需显式强转，可能丢失精度/溢出）：

  double ──(显式强转)──> float ──(显式强转)──> long ──(显式强转)──> int ──> short ──> byte
```
