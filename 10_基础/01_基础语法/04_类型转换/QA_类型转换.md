# 自动类型转换与强制类型转换

## Q1：自动类型转换（隐式拓宽）？

**A**： 
- 小范围类型 → 大范围类型，编译器自动完成
- 转换链路：`byte → short → int → long → float → double`，`char → int`
- 不会丢失精度（float/double 例外）

---

## Q2：强制类型转换（显式窄化）？

**A**： 
- 大范围 → 小范围，必须手动强转，可能丢失精度或溢出
- `(int) 3.9` → `3`（截断，不是四舍五入）
- `(byte) 300` → `44`（溢出：300 % 256）

---

## Q3：表达式中的自动提升？

**A**：
- `byte/short/char` 运算时自动提升为 `int`
- `s = s + 1` 编译错误，`s += 1` 合法（`+=` 隐含强转）
- `int → float`、`long → float/double` 可能丢失精度但编译不报错

---

## Q4：byte b = 1; byte b2 = b + 1; 为什么编译报错？

**A**：`b + 1` 中 `b` 是 byte，`1` 是 int，运算时 byte 自动提升为 int，结果是 int。赋值给 byte 需要显式强转 `(byte)(b + 1)`。但 `b += 1` 是合法的，因为 `+=` 运算符隐含了强转。

---

## Q5：int 转 float 为什么可能丢失精度？float 范围不是比 int 大吗？

**A**：float 范围确实更大（约 ±3.4E38），但精度由尾数位决定。float 只有 23 位尾数，而 int 有 32 位。所以大整数（如 123456789）转 float 后，低位会被截断导致精度丢失。范围大不等于精度高。

---

## Q6：short s = 1; s = s + 1 和 s += 1 的区别是什么？

**A**：`s = s + 1` 中 `s + 1` 结果是 int，赋值给 short 编译报错。`s += 1` 是复合赋值运算符，Java 语言规范规定 `E1 op= E2` 等价于 `E1 = (T) ((E1) op (E2))`，其中 T 是 E1 的类型，所以隐含了 `(short)` 强转。
