# 方法（函数）定义与调用

## 先说结论

方法是 Java 中代码复用的基本单位，由**修饰符、返回值类型、方法名、参数列表、方法体**组成。
方法重载（Overload）允许同名方法参数不同，方法调用时会创建**栈帧（Stack Frame）**。

## 深度解析

### 方法组成

```java
修饰符  返回值类型  方法名(参数列表) throws 异常列表 {
    // 方法体
}
```

### 方法重载（Overload）
- **同名不同参**：方法名相同，参数类型/个数/顺序不同
- 返回值不同不算重载（编译器无法根据返回值区分）
- 参数顺序不同算重载（如 `f(int, String)` 和 `f(String, int)`）
- 发生在**编译期**（静态绑定）

### 方法签名
- 方法名 + 参数列表（参数类型和顺序）
- 返回值不是签名的一部分
- 例如：`add(int, int)` 和 `add(double, double)` 是两个不同的签名

### 栈帧（Stack Frame）
每次方法调用都会在栈上创建一个栈帧，包含：
- **局部变量表**：存放方法参数和局部变量
- **操作数栈**：方法执行的工作区
- **动态链接**：指向运行时常量池的方法引用
- **方法返回地址**：方法正常或异常退出后返回到调用者

### 递归
- 方法调用自身
- 必须有终止条件，否则 `StackOverflowError`
- 递归深度受限于栈大小（`-Xss` 参数调整）

## 易错点/踩坑

- ❌ 返回值不同不算重载：`int f()` 和 `long f()` 编译报错
- ❌ 递归没有终止条件导致 `StackOverflowError`
- ❌ 方法参数是基本类型时，修改不影响外部变量（值传递）
- ❌ `varargs`（可变参数）与重载冲突：精确匹配优先于可变参数

## 代码示例

```java
// ========== 方法重载 ==========
public class Calculator {
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }
    public int add(int a, int b, int c) { return a + b + c; }
    // public long add(int a, int b) {} // 编译错误！只有返回值不同
}

// ========== 递归 ==========
public int factorial(int n) {
    if (n <= 1) return 1;        // 终止条件
    return n * factorial(n - 1);  // 递归调用
}

// ========== 栈帧示意 ==========
// 调用 factorial(3) 的栈帧变化：
// main() → factorial(3) → factorial(2) → factorial(1)
// 返回：1 → 2 → 6 → 6（层层返回）
```

## 图解/流程

```
方法调用的栈帧变化：

  调用 factorial(3)：
  ┌──────────┐
  │ main()   │
  └────┬─────┘
       │ 调用 factorial(3)
       ▼
  ┌──────────┐
  │ f(3)     │ → 调用 f(2)
  └────┬─────┘
       ▼
  ┌──────────┐
  │ f(2)     │ → 调用 f(1)
  └────┬─────┘
       ▼
  ┌──────────┐
  │ f(1)     │ → return 1
  └──────────┘
  返回过程：f(1)=1 → f(2)=2 → f(3)=6 → main() 得到 6
```
