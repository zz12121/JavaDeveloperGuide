# Java 程序执行流程

## Q1：编译阶段（.java → .class）？

**A**： - `javac` 编译器将源码编译为字节码（Bytecode）
- 字节码是平台无关的中间代码，存储在 `.class` 文件中
- 编译期只做语法检查和类型检查，不分配内存

---

## Q2：类加载阶段（.class → JVM 内存）？

**A**： - **类加载器**（ClassLoader）将 `.class` 文件加载到 JVM
- 加载过程：**加载 → 链接（验证→准备→解析）→ 初始化**
- 双亲委派模型：Bootstrap → Extension → Application ClassLoader

---

## Q3：执行阶段（字节码 → 机器码）？

**A**： - **解释执行**：字节码由解释器逐条翻译为机器码执行（启动快，运行慢）
- **JIT 编译**：热点代码被 JIT 编译器（C1/C2）编译为本地机器码缓存（启动慢，运行快）
- **AOT 编译**（JDK 9+）：提前编译为本地代码，减少启动时间

---

## Q4：为什么 Java 是"编译+解释"型语言？

**A**：Java 先编译为字节码（编译），再由 JVM 解释执行或 JIT 编译执行（解释），兼具编译型语言的性能和解释型语言的跨平台性。

---

## Q5：JIT 编译器和解释器为什么需要配合使用？

**A**：解释器用于快速启动，JIT 用于优化热点代码。两者配合实现了启动速度和运行性能的平衡，这叫"混合模式"（Mixed Mode）。

---

## Q6：什么是双亲委派模型？为什么需要它？

**A**：类加载器收到加载请求时，先委派父加载器尝试加载，父加载器无法加载时才自己加载。
保证核心类（如 `java.lang.Object`）只被加载一次，避免安全问题和类冲突。
