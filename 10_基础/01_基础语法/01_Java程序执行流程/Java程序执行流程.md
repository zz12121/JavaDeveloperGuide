# Java 程序执行流程（.java → .class → JVM）

## 先说结论

Java 程序从源码到运行经历**编译 → 加载 → 验证 → 执行**四个阶段，核心是 `.java` 源文件经 `javac` 编译为 `.class` 字节码，再由 JVM 的类加载器加载、验证后，由执行引擎解释或 JIT 编译执行。

## 深度解析

### 编译阶段（.java → .class）
- `javac` 编译器将源码编译为字节码（Bytecode）
- 字节码是平台无关的中间代码，存储在 `.class` 文件中
- 编译期只做语法检查和类型检查，不分配内存

### 类加载阶段（.class → JVM 内存）
- **类加载器**（ClassLoader）将 `.class` 文件加载到 JVM
- 加载过程：**加载 → 链接（验证→准备→解析）→ 初始化**
- 双亲委派模型：Bootstrap → Extension → Application ClassLoader

### 执行阶段（字节码 → 机器码）
- **解释执行**：字节码由解释器逐条翻译为机器码执行（启动快，运行慢）
- **JIT 编译**：热点代码被 JIT 编译器（C1/C2）编译为本地机器码缓存（启动慢，运行快）
- **AOT 编译**（JDK 9+）：提前编译为本地代码，减少启动时间

## 易错点/踩坑

- ❌ 字节码 ≠ 机器码：字节码是 JVM 能理解的指令，还需要解释或 JIT 编译才能变成 CPU 执行的机器码
- ❌ AOT ≠ JIT：AOT 是提前编译（编译时完成），JIT 是运行时编译，两者互补不是替代
- ❌ 双亲委派不是"父加载器加载全部"：父加载器加载失败后，子加载器才会自己尝试加载

## 代码示例

```java
// 编译：javac Hello.java → 生成 Hello.class
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello JVM");
    }
}
```

```bash
# 运行时可通过 -verbose:class 观察类加载过程
java -verbose:class Hello
```
