# 循环语句（for、while、do-while、foreach）(break、continue、return)

## 先说结论

Java 提供四种循环结构：`for`、`while`、`do-while` 和 `foreach`（增强 for）。
选择的关键在于**是否已知循环次数**，而 `foreach` 只适合只读遍历，遍历中修改集合会抛异常。

`break` 跳出当前**整个循环**，`continue` 跳过**本次迭代**进入下一次，`return` 结束**整个方法**。
三者作用范围不同：break/continue 作用于循环，return 作用于方法。

## 深度解析

### 四种循环对比

| 循环类型 | 语法特点 | 适用场景 |
|---------|---------|---------|
| `for` | 初始化、条件、迭代三部分一体 | 已知循环次数 |
| `while` | 先判断后执行 | 未知循环次数 |
| `do-while` | 先执行后判断（至少执行一次） | 至少执行一次的场景 |
| `foreach` | 语法简洁，无需索引 | 只读遍历数组/集合 |

### for vs while
- 已知循环次数 → `for`
- 未知循环次数 → `while`
- `for` 循环变量作用域仅限于循环体，`while` 需要在外部声明

### foreach 底层原理
- `foreach` 是 Java 5 引入的语法糖，编译后调用 **Iterator**
- 遍历中通过集合 API 修改会触发 **fail-fast**，抛 `ConcurrentModificationException`
- 正确做法：用 `Iterator.remove()` 或 `removeIf()`（Java 8+）

### do-while
- 至少执行一次循环体
- 注意末尾的分号：`do { } while (条件);`

### 三者对比

|关键字|作用范围|效果|后续代码|
|---|---|---|---|
|`break`|switch / 循环|跳出当前整个循环（或 switch）|循环后代码继续执行|
|`continue`|循环|跳过本次循环体剩余部分|进入下一次迭代|
|`return`|方法|结束当前方法，返回值（void 可省略）|方法后代码不执行|

### 带标签的 break/continue

- 可以跳出/跳过指定外层循环
- 语法：`label: for(...)` + `break label;` / `continue label;`
- 实际开发中不推荐，影响可读性

### finally 中的 return 覆盖问题

- try 块中的 return 返回值会被 finally 中的 return **覆盖**
- 如果 finally 中没有 return，只是修改了变量，不影响 try 中的 return 值（基本类型）
- 引用类型如果在 finally 中修改了对象属性，try 的 return 会受影响

## 易错点/踩坑

- ❌ foreach 中修改集合抛 `ConcurrentModificationException`（fail-fast 机制）
- ❌ `do-while` 忘记末尾分号：`while(条件)` 后需要 `;`
- ❌ `for(;;)` 等价于 `while(true)`，忘记条件导致死循环
- ❌ foreach 中无法获取当前索引，需要索引时用普通 for
- ❌ `continue` 在 `while` 中可能导致死循环（忘记迭代变量 `i++`）
- ❌ `return` 在 `finally` 中会覆盖 try/catch 中的 return，且会抑制未捕获的异常
- ❌ 带标签的 break 不是万能的，不能跨方法跳出
- ❌ `continue` 只能用在循环中，`break` 用在循环或 switch 中

## 代码示例

```java
// ========== foreach 陷阱 ==========
List<String> list = new ArrayList<>(Arrays.asList("A", "B", "C"));

// ❌ 错误：foreach 中修改集合
for (String s : list) {
    if (s.equals("B")) {
        list.remove(s); // ConcurrentModificationException
    }
}

// ✅ 正确：使用 Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (s.equals("B")) {
        it.remove();
    }
}

// ✅ 正确：使用 removeIf（Java 8+）
list.removeIf(s -> s.equals("B"));

// ========== do-while 至少执行一次 ==========
int count = 5;
do {
    System.out.println("执行了，count=" + count);
} while (count < 3);
// 输出：执行了，count=5（虽然条件不满足，但执行了一次）

// ========== for 循环变量作用域 ==========
for (int i = 0; i < 10; i++) {
    System.out.println(i);
}
// System.out.println(i); // 编译错误！i 超出作用域
```

```java
// ========== break ==========
for (int i = 0; i < 10; i++) {
    if (i == 5) break;
    System.out.print(i + " "); // 0 1 2 3 4
}

// ========== continue ==========
for (int i = 0; i < 5; i++) {
    if (i == 2) continue;
    System.out.print(i + " "); // 0 1 3 4
}

// ========== return ==========
public int test() {
    for (int i = 0; i < 10; i++) {
        if (i == 3) return i;
    }
    return -1; // 方法结束
}

// ========== 带标签的 break ==========
outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == 1 && j == 1) break outer;
        System.out.print(i + "," + j + " "); // 0,0 0,1 0,2 1,0
    }
}

// ========== finally 覆盖 return ==========
public int tricky() {
    int x = 1;
    try {
        return x;  // 返回值先保存为 1
    } finally {
        x = 2;     // 修改 x，但 return 的值已经是 1 的副本
        // return 2; // 如果取消注释，会覆盖为 2
    }
}
```

## 图解/流程

```
for 循环：                while 循环：           do-while 循环：
┌──────────┐            ┌──────────┐          ┌──────────┐
│ 初始化    │            │ 条件判断  │─false──→结束│ 循环体    │← 先执行！
└────┬─────┘            └────┬─────┘          └────┬─────┘
     ▼                       │true                 ▼
┌──────────┐            ┌──────────┐          ┌──────────┐
│ 条件判断  │─false──→结束│ 循环体    │          │ 条件判断  │─false──→结束
└────┬─────┘            └────┬─────┘          └────┬─────┘
     │true                   │                     │true
     ▼                       └──────→ 回到条件       └──────→ 回到循环体
┌──────────┐
│ 循环体    │
└────┬─────┘
     ▼
┌──────────┐
│ 迭代      │
└────┬─────┘
     └──────→ 回到条件判断
```

```
循环中三者的控制流：

  ┌─────────────────────────────────────┐
  │             循环开始                  │
  │  ┌───────────────────────────────┐  │
  │  │         条件判断               │  │
  │  │  false → break 跳出整个循环    │  │
  │  │  true  → 进入循环体            │  │
  │  │         │                     │  │
  │  │    continue ──→ 跳回条件判断    │  │
  │  │         │                     │  │
  │  │    return ──→ 结束整个方法      │  │
  │  │         │                     │  │
  │  │    正常执行完 → 迭代 → 条件判断  │  │
  │  └───────────────────────────────┘  │
  │  break 跳到这里继续执行后续代码       │
  └─────────────────────────────────────┘
```
