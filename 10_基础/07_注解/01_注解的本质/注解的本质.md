# 注解的本质

## 核心结论

注解（Annotation）是 Java 5 引入的一种**元数据机制**，本质上是一个继承了 `java.lang.annotation.Annotation` 接口的**特殊接口**。注解本身不直接执行任何逻辑，而是通过**反射机制**在运行时读取，或通过**注解处理器**在编译时处理。

## 深度解析

### 注解的本质是什么

```java
// 自定义一个注解
public @interface MyAnnotation {
    String value() default "";
}
```

编译后，`@interface MyAnnotation` 会被编译器翻译为：

```java
public interface MyAnnotation extends java.lang.annotation.Annotation {
    String value();
}
```

关键结论：
- **注解本质是接口**，继承自 `java.lang.annotation.Annotation`
- 注解中的属性本质是**抽象方法**
- 属性的默认值通过 `default` 关键字指定

### 元数据的含义

"元数据"即"描述数据的数据"：
- 注解不改变程序本身的逻辑
- 注解为程序元素（类、方法、字段等）附加**声明信息**
- 这些信息可以被编译器、框架或运行时读取

### 注解 vs 普通注释

| 对比项 | 普通注释 `//` `/* */` | 注解 `@Xxx` |
|--------|----------------------|-------------|
| 存活阶段 | 只存在于源码，编译后丢失 | 可保留到类文件甚至运行时 |
| 能否被程序读取 | ❌ 不能 | ✅ 可以通过反射读取 |
| 能否影响程序行为 | ❌ 不能 | ✅ 可以（配合框架/处理器） |

### 注解的三个生命周期

通过 `@Retention` 指定：

1. **SOURCE**：仅存在于源码，编译后丢弃（如 `@Override`、`@SuppressWarnings`）
2. **CLASS**：保留到 class 文件，但 JVM 运行时不可读取（默认值，较少使用）
3. **RUNTIME**：运行时可通过反射读取（如 `@Deprecated`、自定义注解）

## 代码示例

```java
// 编译后查看字节码：javap -v MyAnnotation.class
// 输出：public interface MyAnnotation extends java.lang.annotation.Annotation

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface Table {
    String name() default "";
    int version() default 1;
}

@Table(name = "user", version = 2)
public class User {
    // 通过反射读取注解
}
```
