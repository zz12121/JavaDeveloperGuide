# 注解的本质

## Q：注解的本质是什么？

**A：** 注解本质上是一个继承了 `java.lang.annotation.Annotation` 接口的特殊接口。

- `@interface` 语法是 Java 提供的语法糖，编译后自动转换为继承 `Annotation` 的接口
- 注解中的属性本质是**抽象方法**，返回值类型决定了属性的类型
- 注解不直接执行逻辑，而是作为元数据存在，需要通过**反射**或**注解处理器**来读取和处理

```java
// 定义注解
public @interface MyAnno { String value() default ""; }

// 编译后等价于
public interface MyAnno extends Annotation { String value(); }
```

## Q：注解和注释有什么区别？

**A：** 注释（`//`、`/* */`）仅存在于源码阶段，编译后丢失，不能被程序读取。注解（`@Xxx`）可以保留到 class 文件甚至运行时，能通过反射被程序读取和处理。注解本质上是"可以被程序理解的注释"。

## Q：注解中的属性为什么有返回值类型？

**A：** 因为注解本质是接口，注解属性本质是接口中的抽象方法。所以属性必须声明返回值类型（如 `String`、`int`、`Class`、枚举、注解数组等），使用时以键值对方式赋值，编译器会生成一个实现了该接口的**动态代理类**来持有这些值。
