# 注解的应用场景

## 核心结论

注解广泛应用于框架开发中，主要用于**声明式编程**：通过注解标记配置信息，由框架在运行时通过反射读取并执行相应逻辑。典型场景包括 Spring IoC/DI、AOP、参数校验、ORM 映射等。

## 深度解析

### Spring 框架中的注解

```java
// 组件注册
@Component @Service @Repository @Controller
@Configuration @Bean

// 依赖注入
@Autowired @Resource @Value @Qualifier

// Web 层
@RestController @RequestMapping @GetMapping @PostMapping
@RequestParam @PathVariable @RequestBody

// AOP
@Aspect @Before @After @Around @Pointcut

// 事务
@Transactional
```

### 参数校验（JSR 303 / Bean Validation）

```java
public class UserDTO {
    @NotBlank(message = "用户名不能为空")
    private String username;

    @Size(min = 6, max = 20, message = "密码长度6-20位")
    private String password;

    @Email(message = "邮箱格式不正确")
    private String email;

    @Min(value = 0, message = "年龄不能为负数")
    private Integer age;
}
```

### JUnit 测试框架

```java
@Test                              // 标记测试方法
@BeforeEach                       // 每个测试前执行
@DisplayName("测试用户创建")         // 测试用例显示名称
@Timeout(value = 1, unit = SECONDS) // 超时设置
@Disabled("功能开发中")             // 跳过测试
```

### ORM 映射（MyBatis / JPA）

```java
// JPA 实体映射
@Entity
@Table(name = "t_user")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "user_name", nullable = false)
    private String username;
}

// MyBatis
@Select("SELECT * FROM user WHERE id = #{id}")
User findById(Long id);
```

### Lombok 工具库

```java
@Data                              // 自动生成 getter/setter/toString/equals/hashCode
@Builder                           // 生成建造者模式代码
@Slf4j                            // 生成日志对象
@NoArgsConstructor                 // 无参构造
@AllArgsConstructor                // 全参构造
```
