# 86 自定义注解

## 核心结论

自定义注解使用 `@interface` 关键字定义，配合元注解（`@Target`、`@Retention`）指定使用范围和保留策略。注解属性支持基本类型、String、Class、枚举、注解及其一维数组。

## 深度解析

### 定义注解的语法

```java
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
    // 1. 基本类型属性
    int priority() default 0;

    // 2. String 属性
    String value() default "";

    // 3. Class 属性
    Class<?> handler() default Object.class;

    // 4. 枚举属性
    Level level() default Level.INFO;

    // 5. 注解属性（注解嵌套）
    Author author() default @Author(name = "unknown");

    // 6. 数组属性
    String[] tags() default {};
}
```

### 注解属性的限制

- 属性的返回值类型只能是：8 种基本类型、`String`、`Class`、枚举、注解、以上类型的一维数组
- 不能使用 `Object`、`集合类` 等复杂类型
- 如果只有一个属性且名为 `value()`，使用时可以省略属性名

### 使用自定义注解

```java
@MyAnnotation(priority = 1, value = "test", tags = {"java", "annotation"})
public class TestClass {

    @MyAnnotation("simple") // 省略 value 属性名
    public void testMethod() { }
}
```

### 完整实战示例

```java
// 1. 定义注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Log {
    String value() default "";
    boolean enable() default true;
}

// 2. 使用注解
public class UserService {
    @Log("查询用户信息")
    public User getUser(int id) { return null; }

    @Log(value = "删除用户", enable = false)
    public void deleteUser(int id) { }
}

// 3. 通过反射读取（见 Card_87）
```
