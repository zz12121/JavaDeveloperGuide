# 元注解

## 核心结论

元注解（Meta-Annotation）是**用于修饰注解的注解**，定义在 `java.lang.annotation` 包中。Java 提供了 5 个标准元注解：`@Target`、`@Retention`、`@Documented`、`@Inherited`、`@Repeatable`（JDK 8+）。

## 深度解析

### @Target — 注解可以用在哪里

指定注解的适用目标，使用 `ElementType` 枚举：

| ElementType | 适用位置 |
|-------------|---------|
| `TYPE` | 类、接口、枚举 |
| `FIELD` | 字段（包括枚举常量） |
| `METHOD` | 方法 |
| `PARAMETER` | 方法参数 |
| `CONSTRUCTOR` | 构造方法 |
| `LOCAL_VARIABLE` | 局部变量 |
| `ANNOTATION_TYPE` | 注解类型 |
| `PACKAGE` | 包 |
| `TYPE_PARAMETER` | 类型参数（JDK 8+） |
| `TYPE_USE` | 任何类型使用处（JDK 8+） |

```java
@Target({ElementType.TYPE, ElementType.METHOD})
public @interface MyAnnotation { }
```

### @Retention — 注解的保留策略

指定注解的生命周期：

| RetentionPolicy | 说明 | 场景 |
|----------------|------|------|
| `SOURCE` | 仅源码，编译后丢弃 | 编译器检查（如 `@Override`） |
| `CLASS` | 保留到 class 文件，运行时不可读 | 字节码增强工具（默认值） |
| `RUNTIME` | 运行时可通过反射读取 | Spring 框架、自定义注解 |

### @Documented — 注解是否出现在 JavaDoc 中

```java
@Documented
public @interface ApiDoc { }
// 使用 @ApiDoc 的类会在 javadoc 中显示该注解
```

### @Inherited — 注解是否可以被继承

```java
@Inherited
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface MyAnnotation { }

@MyAnnotation
public class Parent { }
public class Child extends Parent { }

// Child 类也会继承 @MyAnnotation 注解
// Child.class.isAnnotationPresent(MyAnnotation.class) → true
```

注意：
- `@Inherited` 只对**类**生效，对方法和字段不生效
- 子类如果自行标注了注解，会覆盖父类的继承

### @Repeatable — 注解是否可重复（JDK 8+）

```java
@Repeatable(Schedules.class)
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface Schedule {
    String cron();
}

@Schedule(cron = "0 0 8 * * ?")   // 每天早8点
@Schedule(cron = "0 0 20 * * ?")  // 每天晚8点
public void doTask() { }

// 容器注解（编译器自动生成）
public @interface Schedules {
    Schedule[] value();
}
```

## 代码示例

```java
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
public @interface Log {
    String value() default "";
}
```
