
# 线程池工作流程

## Q1：描述线程池处理任务的完整流程。

**A**：

```
提交任务
  1. 当前线程数 < corePoolSize → 创建核心线程直接执行
  2. 核心线程满 → 任务入 workQueue 等待
  3. 队列满 + 线程数 < maximumPoolSize → 创建非核心线程执行
  4. 队列满 + 线程数 = maximumPoolSize → 执行拒绝策略
```

关键：任务先尝试核心线程，再入队列，队列满了才创建非核心线程。

---

## Q2：为什么入队后需要二次检查？

**A**：入队（workQueue.offer）和重新检查（ctl.get）之间不是原子的：

1. 入队后线程池可能已被 shutdown()
2. 入队后所有核心线程可能已死亡（队列为空，无线程处理）
3. 需要确保：如果线程池关闭则移除任务并拒绝；如果没有线程则至少创建一个

---

## Q3：核心线程和非核心线程的区别是什么？

**A**：本质上没有区别，都是 Worker 对象。区别在于：

- **核心线程**：getTask() 使用 `take()` 阻塞等待，不会因空闲超时退出
- **非核心线程**：getTask() 使用 `poll(keepAliveTime)`，超时后返回 null，线程退出

可以通过 `allowCoreThreadTimeOut(true)` 让核心线程也有超时。
