
# 线程池核心参数

## 核心结论

ThreadPoolExecutor 有 7 个核心参数，理解每个参数的含义和相互关系是掌握线程池的基础。

## 深度解析

### 7 个参数

```java
public ThreadPoolExecutor(
    int corePoolSize,        // 核心线程数
    int maximumPoolSize,     // 最大线程数
    long keepAliveTime,      // 非核心线程空闲存活时间
    TimeUnit unit,           // 时间单位
    BlockingQueue<Runnable> workQueue,     // 任务队列
    ThreadFactory threadFactory,           // 线程工厂
    RejectedExecutionHandler handler        // 拒绝策略
)
```

### 参数详解

| 参数 | 说明 | 默认值 |
|------|------|--------|
| corePoolSize | 核心线程数，即使空闲也不回收 | 1（Executors） |
| maximumPoolSize | 最大线程数 = 核心 + 非核心 | Integer.MAX_VALUE |
| keepAliveTime | 非核心线程空闲超时时间 | 60s |
| unit | keepAliveTime 的时间单位 | SECONDS |
| workQueue | 任务等待队列 | — |
| threadFactory | 创建线程的工厂 | Executors.defaultThreadFactory |
| handler | 队列满且线程满时的拒绝策略 | AbortPolicy |

### 线程数变化

```
任务提交后：

线程数 < corePoolSize     → 创建核心线程
线程数 = corePoolSize     → 任务入 workQueue
队列满 + 线程数 < maxPool  → 创建非核心线程
线程数 = maxPool + 队列满  → 执行拒绝策略

空闲超时：
  非核心线程空闲 > keepAliveTime → 回收
  核心线程空闲 → 不回收（除非 allowCoreThreadTimeOut(true)）
```

### ThreadFactory

```java
// 自定义线程工厂
ThreadFactory factory = r -> {
    Thread t = new Thread(r);
    t.setName("my-pool-" + t.getId());
    t.setDaemon(false);      // 非守护线程
    t.setPriority(Thread.NORM_PRIORITY);
    return t;
};
```

### RejectedExecutionHandler

```java
// 4 种内置拒绝策略
new ThreadPoolExecutor.AbortPolicy()      // 抛 RejectedExecutionException
new ThreadPoolExecutor.CallerRunsPolicy()  // 调用者线程执行
new ThreadPoolExecutor.DiscardPolicy()     // 静默丢弃
new ThreadPoolExecutor.DiscardOldestPolicy() // 丢弃队列头部最旧任务
```

## 易错点与踩坑

### 1. corePoolSize 设置为 0 的隐患
```java
// CachedThreadPool 本质就是 core=0，任务来时才创建线程
// 如果任务是突发性的，可能导致频繁创建/销毁线程，开销大
new ThreadPoolExecutor(0, 100, 60, SECONDS, new SynchronousQueue<>());
```
- 空闲时线程全部回收，新任务有冷启动延迟
- 如果任务量突然增大，线程创建速度跟不上 → 大量任务触发拒绝策略

### 2. maximumPoolSize 被无界队列"架空"
- 使用 `LinkedBlockingQueue()`（无界）时，maximumPoolSize 形同虚设
- 因为队列永远不满，永远不会创建非核心线程
- **面试金句**："有界队列才能让 maximumPoolSize 生效"

### 3. workQueue 容量与拒绝策略的关系
- **无界队列**：永远不会触发拒绝策略 → 可能 OOM
- **有界队列 + 合理容量**：可以触发拒绝策略，但需要选择合适的策略
- **SynchronousQueue**：不存储任务，直接交线程 → 拒绝策略更早触发

### 4. keepAliveTime 单位混淆
```java
// 错误：想设 60 秒但实际设了 60 纳秒
new ThreadPoolExecutor(10, 20, 60, NANOSECONDS, queue);
// 正确
new ThreadPoolExecutor(10, 20, 60, SECONDS, queue);
```
