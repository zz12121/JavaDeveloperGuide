
# 线程池关闭

## 核心结论

`shutdown()` 温和关闭（等任务完成），`shutdownNow()` 暴力关闭（中断任务）。推荐 shutdown + awaitTermination 组合使用。

## 深度解析

### shutdown()

```java
pool.shutdown();
// 状态：RUNNING → SHUTDOWN
// 不接受新任务
// 已提交的任务（包括队列中的）继续执行
// 等待所有任务完成后线程池终止
```

### shutdownNow()

```java
List<Runnable> notExecuted = pool.shutdownNow();
// 状态：RUNNING → STOP
// 不接受新任务
// 队列中的任务不再执行，返回未执行任务列表
// 中断所有正在执行的任务（Thread.interrupt()）
```

### 推荐关闭方式

```java
// 优雅关闭
public void shutdownGracefully(ExecutorService pool) {
    // 1. 停止接受新任务
    pool.shutdown();
    
    try {
        // 2. 等待已提交任务完成
        if (!pool.awaitTermination(60, TimeUnit.SECONDS)) {
            // 3. 超时后强制关闭
            pool.shutdownNow();
            // 4. 再等一会
            if (!pool.awaitTermination(60, TimeUnit.SECONDS)) {
                log.error("线程池未完全终止");
            }
        }
    } catch (InterruptedException e) {
        pool.shutdownNow();
        Thread.currentThread().interrupt();
    }
}
```

### awaitTermination

```java
// 阻塞等待线程池终止，超时返回 false
boolean terminated = pool.awaitTermination(60, TimeUnit.SECONDS);
```

### 对比

| 维度 | shutdown | shutdownNow |
|------|---------|-------------|
| 新任务 | 拒绝 | 拒绝 |
| 队列任务 | 执行完 | 丢弃 |
| 执行中任务 | 继续执行 | 尝试中断 |
| 阻塞 | 不阻塞 | 不阻塞 |
| 返回值 | void | List<Runnable> |
| 推荐度 | ⭐⭐⭐⭐⭐ | ⭐⭐（兜底用） |

## 易错点与踩坑

### 1. shutdown 不等待任务完成
```java
// shutdown() 立即返回，不阻塞
// 只是设置了状态为 SHUTDOWN，不接受新任务
// 已提交的任务会继续执行，但 shutdown() 不会等它们执行完
// 需要 awaitTermination() 来等待
```

### 2. awaitTermination 被中断
```java
// awaitTermination() 响应中断
// 如果等待期间线程被中断 → 抛 InterruptedException
// 此时线程池可能还没完全关闭
// 正确做法：catch 后重新调用 shutdownNow()
try {
    pool.shutdown();
    if (!pool.awaitTermination(60, SECONDS)) {
        pool.shutdownNow();
    }
} catch (InterruptedException e) {
    pool.shutdownNow();
    Thread.currentThread().interrupt(); // 恢复中断标志
}
```

### 3. shutdownNow 的未执行任务需要处理
```java
// shutdownNow() 返回未执行的任务列表
List<Runnable> notExecuted = pool.shutdownNow();
// 这些任务可能需要持久化或重新提交
notExecuted.forEach(task -> {
    // 保存到数据库 / 发送到 MQ
});
```

### 4. Spring 容器中忘记关闭线程池
```java
// ❌ 线程池作为局部变量，方法结束对象被 GC，但线程不会自动停止
public void process() {
    ExecutorService pool = Executors.newFixedThreadPool(10);
    pool.submit(task);
    // 忘记 shutdown → 线程泄漏
}

// ✅ 实现 DisposableBean 或 @PreDestroy
@PreDestroy
public void destroy() {
    pool.shutdown();
    pool.awaitTermination(30, SECONDS);
}
```
