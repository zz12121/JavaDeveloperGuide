
# 线程池监控

## 核心结论

ThreadPoolExecutor 提供了丰富的监控方法，可以实时获取线程池运行状态，用于告警和调优。

## 深度解析

### 核心监控方法

| 方法 | 说明 | 告警阈值 |
|------|------|---------|
| `getActiveCount()` | 正在执行任务的线程数 | > corePoolSize |
| `getPoolSize()` | 当前线程池线程数 | > corePoolSize |
| `getQueue().size()` | 等待队列中的任务数 | > 容量×80% |
| `getQueue().remainingCapacity()` | 队列剩余容量 | < 容量×20% |
| `getTaskCount()` | 总任务数（已完成+执行中+等待中） | — |
| `getCompletedTaskCount()` | 已完成任务数 | — |
| `getLargestPoolSize()` | 历史最大线程数 | 接近 maximumPoolSize |
| `isShutdown()` | 是否已关闭 | true |
| `isTerminated()` | 是否已终止 | true |
| `getRejectedExecutionHandler()` | 当前拒绝策略 | — |

### 监控示例

```java
// 定时监控线程池
ScheduledExecutorService monitor = Executors.newSingleThreadScheduledExecutor();
monitor.scheduleAtFixedRate(() -> {
    int active = pool.getActiveCount();
    int poolSize = pool.getPoolSize();
    int queueSize = pool.getQueue().size();
    long completed = pool.getCompletedTaskCount();
    long total = pool.getTaskCount();
    
    log.info("pool={}, active={}, queue={}, completed={}, total={}",
             poolSize, active, queueSize, completed, total);
    
    // 告警
    if (active >= pool.getCorePoolSize() && queueSize > 100) {
        log.warn("线程池过载！active={}, queue={}", active, queueSize);
    }
}, 0, 10, TimeUnit.SECONDS);
```

### 钩子方法监控

```java
ThreadPoolExecutor pool = new ThreadPoolExecutor(...) {
    @Override
    protected void beforeExecute(Thread t, Runnable r) {
        super.beforeExecute(t, r);
        // 记录任务开始时间
    }
    
    @Override
    protected void afterExecute(Runnable r, Throwable t) {
        super.afterExecute(r, t);
        // 记录任务完成时间、异常
        if (t != null) {
            log.error("任务异常", t);
        }
    }
    
    @Override
    protected void terminated() {
        super.terminated();
        log.info("线程池已终止");
    }
};
```

## 易错点与踩坑

### 1. getTaskCount 不等于"待处理任务数"
```java
// getTaskCount() = 已完成 + 正在执行 + 队列中等待
// 不是"还需要处理的任务数"
// 如果想看"待处理" = getTaskCount() - getCompletedTaskCount() - getActiveCount()
```

### 2. getQueue().size() 的延迟
```java
// 监控数据是某一时刻的快照，不是实时精确值
// 高并发下，读取时数据可能已经变化
// 不要基于单次监控数据做关键决策
```

### 3. 监控线程池本身也需要线程池
```java
// ❌ 用 Executors.newSingleThreadScheduledExecutor() 创建监控线程
// 这是另一个线程池，也需要管理生命周期
// 如果主线程池都没关，监控线程池可能泄漏

// ✅ 复用现有调度框架（如 Spring @Scheduled）
// 或使用 JDK 管理的监控方案（JMX）
```

### 4. afterExecute 无法获取 Runnable 的原始信息
```java
// afterExecute(Runnable r, Throwable t) 中的 r 可能被包装
// submit() 提交的任务会被包装为 FutureTask
// 无法直接获取原始的 Callable/Runnable 对象
// 建议：用 ThreadLocal 或 MDC 传递任务上下文
```
## 生产环境监控方案

### JMX 监控

```java
// 注册线程池到 JMX
ThreadPoolExecutor executor = new ThreadPoolExecutor(...);
MBeanServer mBeanServer = ManagementFactory.getPlatformMBeanServer();
ObjectName objectName = new ObjectName("com.example:type=ThreadPool,name=BusinessPool");
mBeanServer.registerMBean(new StandardMBean(executor, ThreadPoolExecutorMXBean.class), objectName);
```

JConsole/VisualVM 可直接查看：
- ActiveCount、PoolSize、QueueSize
- CompletedTaskCount、TaskCount
- CorePoolSize、MaximumPoolSize

### Micrometer + Prometheus + Grafana

```java
// Spring Boot 集成
@Bean
public MeterRegistryCustomizer<MeterRegistry> metricsCommonTags() {
    return registry -> registry.config().commonTags("application", "myapp");
}

// 自定义线程池指标
@Component
public class ThreadPoolMetrics {
    
    @Autowired
    private MeterRegistry meterRegistry;
    
    @PostConstruct
    public void init() {
        ThreadPoolExecutor executor = ...;
        
        Gauge.builder("threadpool.active", executor, ThreadPoolExecutor::getActiveCount)
            .tag("name", "business")
            .register(meterRegistry);
            
        Gauge.builder("threadpool.queue.size", executor, e -> e.getQueue().size())
            .tag("name", "business")
            .register(meterRegistry);
            
        Gauge.builder("threadpool.completed", executor, ThreadPoolExecutor::getCompletedTaskCount)
            .tag("name", "business")
            .register(meterRegistry);
    }
}
```

**PromQL 告警规则示例**：
```yaml
groups:
  - name: threadpool_alerts
    rules:
      - alert: ThreadPoolQueueFull
        expr: threadpool_queue_size / threadpool_queue_capacity > 0.8
        for: 1m
        labels:
          severity: warning
        annotations:
          summary: "线程池队列积压"
          
      - alert: ThreadPoolActiveHigh
        expr: threadpool_active / threadpool_max > 0.9
        for: 2m
        labels:
          severity: critical
```

### 监控指标采集最佳实践

| 指标 | 采集频率 | 用途 |
|------|---------|------|
| active/poolSize | 10s | 实时负载 |
| queueSize | 10s | 积压检测 |
| completedTaskCount | 60s | 吞吐量计算 |
| taskCount | 60s | 任务总量 |
| largestPoolSize | 300s | 峰值分析 |

**吞吐量计算**：
```java
// 每分钟任务完成数
long lastCompleted = ...;
long currentCompleted = pool.getCompletedTaskCount();
int throughput = (int) ((currentCompleted - lastCompleted) / 60);
```
