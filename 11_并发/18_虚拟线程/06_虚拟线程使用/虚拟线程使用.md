# 虚拟线程使用

## 核心结论

Java 21+ 提供了多种创建虚拟线程的方式，推荐使用 `Executors.newVirtualThreadPerTaskExecutor()` 创建虚拟线程执行器。虚拟线程的使用方式与平台线程几乎一致，但可以轻松创建**百万级别**的并发任务。

## 深度解析

### 创建虚拟线程的四种方式

#### 方式一：Thread.startVirtualThread（最简单）

```java
// Java 21+
Thread vt = Thread.startVirtualThread(() -> {
    System.out.println("Hello from " + Thread.currentThread());
    System.out.println("Is virtual: " + Thread.currentThread().isVirtual());
});
vt.join();
```

#### 方式二：Thread.ofVirtual() Builder（可配置）

```java
Thread vt = Thread.ofVirtual()
    .name("my-virtual-thread", 0)  // 线程名前缀 + 计数器
    .start(() -> {
        System.out.println(Thread.currentThread().getName());
    });
vt.join();
```

Builder 支持的方法：
- `name(String prefix)` / `name(String prefix, long start)` — 设置线程名
- `uncaughtExceptionHandler(Thread.UncaughtExceptionHandler)` — 未捕获异常处理
- `start(Runnable task)` — 创建并立即启动
- `unstarted(Runnable task)` — 创建但不启动

#### 方式三：Executors.newVirtualThreadPerTaskExecutor()（推荐）

```java
// 每提交一个任务创建一个新的虚拟线程，无复用
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 100_000; i++) {
        executor.submit(() -> {
            // 每个任务使用一个独立虚拟线程
            var result = httpClient.send(request, BodyHandlers.ofString());
            return result.body();
        });
    }
}
// try-with-resources 确保所有任务完成后关闭
```

**为什么推荐**：
- 自动管理虚拟线程生命周期
- 返回 `ExecutorService`，与现有线程池 API 完全兼容
- `shutdown()` / `awaitTermination()` 语义清晰

#### 方式四：自定义 ThreadFactory

```java
ThreadFactory factory = Thread.ofVirtual()
    .name("vt-pool-", 0)
    .factory();

try (var executor = new ThreadPoolExecutor(
    0, Integer.MAX_VALUE, 60L, TimeUnit.SECONDS,
    new SynchronousQueue<>(), factory)) {
    executor.submit(task);
}
```

> ⚠️ 实际上虚拟线程**不需要线程池**复用，方式三已经是最优解。此方式仅用于兼容现有框架。

### 判断当前是否为虚拟线程

```java
Thread.currentThread().isVirtual();  // true = 虚拟线程
Thread.currentThread().isVirtual();  // false = 平台线程
```

### 与 CompletableFuture 配合

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
        // 在虚拟线程中执行
        return fetchData();
    }, executor);

    future.thenApply(data -> process(data))
          .thenAccept(result -> System.out.println(result))
          .join();
}
```

### 与 Spring Boot 3.2+ 集成

```yaml
# application.yml
spring:
  threads:
    virtual:
      enabled: true  # Spring Boot 3.2+ 内置支持
```

开启后，Tomcat 的每个请求处理线程自动切换为虚拟线程。
