
# 虚拟线程概念

## 核心结论

虚拟线程（Virtual Threads）是 **JDK 21 正式发布**（JEP 444）的轻量级线程实现，由 JVM 而非操作系统管理。虚拟线程的创建成本极低，适合创建**百万级**并发任务，彻底改变 Java 的并发编程模型。

## 深度解析

### 什么是虚拟线程

```
传统平台线程：
  Java Thread → 1:1 → OS Thread（内核线程）
  创建成本高（~1MB 栈内存），数量受限于系统

虚拟线程：
  Java VirtualThread → M:N → Carrier Thread（平台线程）
  创建成本低（~几 KB），可创建百万级
  JVM 负责调度，不依赖 OS 调度
```

### 发展历程

| 版本 | 里程碑 |
|------|--------|
| JDK 18 (2022) | 虚拟线程预览版（JEP 425） |
| JDK 19 (2022) | 二次预览（JEP 436） |
| JDK 20 (2023) | 三次预览（JEP 438） |
| JDK 21 (2023) | **正式发布**（JEP 444） |

### 核心特性

| 特性 | 说明 |
|------|------|
| **轻量级** | 栈空间仅几 KB，平台线程约 1MB |
| **低成本** | 创建/销毁几乎无开销 |
| **高数量** | 单机可创建百万级虚拟线程 |
| **透明挂起** | 阻塞操作自动挂起，不占用载体线程 |
| **兼容性** | 与现有 Thread API 完全兼容 |

### 与其他语言对比

| 语言/框架 | 轻量级线程方案 |
|-----------|--------------|
| Go | Goroutine |
| Kotlin | Coroutine |
| Node.js | Event Loop + async/await |
| Java | Virtual Thread（JDK 21+） |

### 纤程（Fiber）与绿色线程的发展历史

**绿色线程（Green Threads）**：由用户态运行时（而非 OS）调度的线程。

```
Java 的绿色线程历史：
  JDK 1.1（1997）：使用绿色线程，所有 Java 线程映射到单个 OS 线程
    问题：无法利用多核 CPU，被放弃
  JDK 1.2+：改为 1:1 模型（Java 线程 ↔ OS 线程）
    优势：利用多核，成熟稳定
    问题：创建成本高，数量受限

其他语言的绿色线程：
  Go（2009）：Goroutine（M:N 模型），从语言层面设计
  Erlang（1986）：Actor 模型 + 轻量级进程
  Kotlin（2017）：Coroutine（编译器实现，非 JVM 修改）
```

**纤程（Fiber）**：比绿色线程更轻量的概念。

- Erlang 的 "process" ≈ 纤程
- Kotlin 的 Coroutine ≈ 纤程
- Java 的 Loom 项目最初命名为 "Project Loom: Fibers"，后改为 "Virtual Thread"

```
名词辨析：
  绿色线程（Green Thread）  → OS 不可见，用户态调度
  纤程（Fiber）            → 比线程更轻，可暂停/恢复
  协程（Coroutine）        → 语言层面支持，Kotlin 使用此名
  虚拟线程（Virtual Thread） → Java 21 正式名，JVM 层面实现
```

### Java 绿色线程的历史教训

JDK 1.1 的绿色线程被放弃的原因：

| 原因 | 说明 |
|------|------|
| 无法利用多核 | 所有 Java 线程映射到单个 OS 线程 |
| 无法与 Native 库配合 | JNI 调用会阻塞整个进程 |
| 调试困难 | OS 层面看不到 Java 线程 |

**虚拟线程吸取教训**：保留与 OS 线程的 1:1 映射语义（Thread API 兼容），但在虚拟线程不运行时卸载载体线程，实现 M:N 效果。

### 设计理念

虚拟线程的核心思想：**让开发者用简单的同步编程模型，获得异步编程的性能**。

```java
// 同步代码风格，但实际上是非阻塞的
Thread.startVirtualThread(() -> {
    String data = httpClient.send(request); // IO 阻塞 → 自动挂起
    String result = db.query(sql);          // DB 阻塞 → 自动挂起
    return data + result;                   // 不占用载体线程
});
```
