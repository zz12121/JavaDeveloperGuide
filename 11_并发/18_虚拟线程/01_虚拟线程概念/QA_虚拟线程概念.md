
# 虚拟线程概念

## Q1：什么是虚拟线程？

**A**：

虚拟线程（Virtual Threads）是 JDK 21 正式发布的轻量级线程，由 JVM 管理而非操作系统。

与平台线程的区别：
- **平台线程**：Java Thread 与 OS Thread 一一对应，创建成本高（~1MB 栈），数量受限
- **虚拟线程**：多个虚拟线程映射到少量平台线程（载体线程），创建成本极低（~几 KB），可创建百万级

## Q2：虚拟线程是哪个版本引入的？

**A**：

- JDK 18：首次预览（JEP 425）
- JDK 19/20：后续预览
- **JDK 21（2023年9月）**：正式发布（JEP 444）

## Q3：虚拟线程和 Go 的 Goroutine 有什么类似之处？

**A**：

两者都是**用户态轻量级线程**，核心思路相同：
- 由运行时管理，不依赖 OS 调度
- 阻塞时自动挂起，不占用底层线程
- 创建成本极低，支持大规模并发

区别：Go 的 Goroutine 从语言诞生就内置，Java 的虚拟线程是后期引入的兼容方案。

## Q4：Java 为什么在 JDK 1.2 中放弃了绿色线程？

**A**：

JDK 1.1 使用的是**绿色线程**（所有 Java 线程映射到单个 OS 线程），JDK 1.2 改为 **1:1 模型**（每个 Java 线程对应一个 OS 线程）。

**放弃绿色线程的原因**：

| 原因 | 说明 |
|------|------|
| **无法利用多核 CPU** | 单个 OS 线程只能跑在一个 CPU 核上 |
| **无法与 Native 库配合** | JNI 调用阻塞时，整个进程卡死 |
| **调试困难** | OS 层面看不到 Java 线程，无法用系统工具调试 |

**虚拟线程如何解决这些问题？**

- 保留 1:1 模型的 API 兼容性（Thread API 不变）
- 在虚拟线程不运行时卸载载体线程（M:N 效果）
- Native 调用时载体线程被钉住，但其他虚拟线程不受影响

## Q5：虚拟线程、纤程、协程有什么区别？

**A**：

这几个名词经常被混用，但含义有区别：

| 名词 | 含义 | Java 对应 |
|------|------|-----------|
| **绿色线程**（Green Thread） | OS 不可见的用户态线程 | JDK 1.1 |
| **纤程**（Fiber） | 比线程更轻，可暂停/恢复 | Loom 项目最初命名 |
| **协程**（Coroutine） | 语言层协程，可暂停/恢复 | Kotlin Coroutine |
| **虚拟线程**（Virtual Thread） | JVM 层实现，与 Thread API 兼容 | JDK 21+ |

**发展脉络**：

```
Java 绿色线程（JDK 1.1）  →  被放弃
       ↓
Java 平台线程（JDK 1.2+） →  稳定，但重量级
       ↓
Project Loom 启动（2017）  →  最初命名为 "Fibers"
       ↓
JEP 425（JDK 18）         →  预览，命名为 "Virtual Threads"
       ↓
JEP 444（JDK 21）         →  正式发布 ✅
```

**一句话总结**：虚拟线程 = Java 版的轻量级线程，和 Go 的 Goroutine、Kotlin 的 Coroutine 思路相同，但实现层面（JVM vs 编译器）不同。



> **代码示例：平台线程 vs 虚拟线程的数量对比**

```java
// 平台线程：受限于 OS 资源，创建 10 万个会 OOM
// for (int i = 0; i < 100_000; i++) new Thread(() -> sleep(1000)).start(); // OOM!

// 虚拟线程：轻松创建百万级（JDK 21+）
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 1_000_000; i++) {
        executor.submit(() -> {
            Thread.sleep(1000); // 阻塞时自动让出载体线程，不浪费 OS 资源
            return "done";
        });
    }
}
// 虚拟线程由 JVM 调度，阻塞时挂起、就绪时恢复，底层载体线程数远小于虚拟线程数
```
