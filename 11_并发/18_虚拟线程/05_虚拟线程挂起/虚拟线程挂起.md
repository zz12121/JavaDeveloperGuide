
# 虚拟线程挂起

## 核心结论

虚拟线程遇到阻塞操作时会**自动挂起**（Unmount），释放载体线程给其他虚拟线程使用。IO 操作完成后**自动恢复**（Mount），继续执行。这个过程对开发者完全透明。

## 深度解析

### 会触发挂起的操作

| 操作类型 | 示例 | 挂起行为 |
|---------|------|---------|
| 网络 IO | `Socket.read()` / `HttpClient.send()` | ✅ 自动挂起 |
| 文件 IO | `FileChannel.read()`（部分） | ✅ 大部分场景挂起 |
| 睡眠 | `Thread.sleep()` | ✅ 自动挂起 |
| 等待 | `LockSupport.park()` | ✅ 自动挂起 |
| 阻塞队列 | `BlockingQueue.put()/take()` | ✅ 自动挂起 |
| synchronized | `synchronized` 块内阻塞 | ⚠️ JDK 21+ 大部分优化 |
| Lock | `ReentrantLock.lock()` | ✅ 自动挂起 |
| Native 方法 | JNI 调用中的阻塞 | ⚠️ 可能 pinning |

### park/unpark 语义变化（重要）

`LockSupport.park()` / `unpark()` 在虚拟线程中的行为发生了**根本性变化**：

**平台线程中**：`park()` 直接阻塞当前 OS 线程，`unpark()` 唤醒指定线程 → 涉及内核态切换。

**虚拟线程中**：`park()` 被 JVM 拦截 → 虚拟线程挂起（unmount），**不阻塞载体线程**；`unpark()` 将虚拟线程标记为可运行，重新挂载到某个载体线程。

```java
// 平台线程：park() 阻塞 OS 线程
Thread t = new Thread(() -> {
    LockSupport.park();  // OS 线程被阻塞，100% 内核态切换
});
t.start();
LockSupport.unpark(t);  // 唤醒 OS 线程

// 虚拟线程：park() 挂起虚拟线程，不阻塞载体线程
Thread vt = Thread.startVirtualThread(() -> {
    LockSupport.park();  // JVM 拦截 → VT 挂起 → 载体线程释放
    // 恢复后继续执行
});
LockSupport.unpark(vt);  // 标记 VT 可运行 → 重新挂载到载体线程
```

**语义变化的影响**：

| 场景 | 平台线程 | 虚拟线程 |
|------|---------|---------|
| `park()` 后线程状态 | `WAITING`（OS 线程阻塞） | `PARKED`（JVM 管理，不阻塞 OS） |
| `unpark()` 提前调用 | 下次的 `park()` 立即返回 | 行为相同（许可证机制不变） |
| 中断响应 | `park()` 响应 `interrupt()` | **行为改变** → 见下文 |
| 性能 | 内核态切换，约 1-10μs | 用户态操作，约 0.1μs |

**中断响应的变化**（重要）：

在平台线程中，`LockSupport.park()` 响应中断（`Thread.interrupt()`），但不会抛出 `InterruptedException`，只是设置中断标志位。

在虚拟线程中，`LockSupport.park()` 被中断时行为**有所不同**：JVM 会在虚拟线程层面处理中断，具体行为取决于 JDK 版本，JDK 21+ 已统一为与平台线程一致的中断语义。

### 挂起/恢复流程

```
虚拟线程执行 HTTP 请求：
                                    载体线程
VT ──→ Continuation.run() ──────→ Carrier-1
     │  执行到 httpClient.send()
     │  ↓ 检测到阻塞操作
     │  保存栈帧到 Continuation
     │  unmount (VT 与 Carrier-1 解绑)
     │
     │    Carrier-1 被释放 ──→ 执行其他虚拟线程
     │
     │  （IO 操作在底层完成...）
     │
     │  IO 完成
     │  mount (VT 挂载到 Carrier-2)
     │  恢复 Continuation 栈帧
     ↓
继续执行 response 处理逻辑
```

### 不需要修改的代码

```java
// 传统 IO 代码，虚拟线程直接运行，无需任何修改
Thread.startVirtualThread(() -> {
    var conn = DriverManager.getConnection(url);  // 挂起
    var stmt = conn.prepareStatement(sql);         // 挂起
    var rs = stmt.executeQuery();                  // 挂起
    // ... 处理结果
    rs.close();  // 挂起
});
```
