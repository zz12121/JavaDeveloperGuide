# 线程优先级（Priority）

## 先说结论

Java 线程优先级范围是 1~10，**优先级高不代表一定先执行**，只是被调度器选中的概率更高。底层实现依赖操作系统，跨平台时优先级行为可能不一致。

## 深度解析

### 优先级常量

```java
public class Thread implements Runnable {
    // 最小优先级
    public static final int MIN_PRIORITY = 1;
    // 默认优先级
    public static final int NORM_PRIORITY = 5;
    // 最大优先级
    public static final int MAX_PRIORITY = 10;
}
```

### 相关方法

```java
// 设置优先级
public final void setPriority(int newPriority) {
    if (newPriority > MAX_PRIORITY || newPriority < MIN_PRIORITY) {
        throw new IllegalArgumentException();
    }
    // ...
    setPriority0(priority);
}

// 获取优先级
public final int getPriority() {
    return priority;
}
```

### 优先级调度策略

线程调度器根据优先级决定 CPU 时间片分配：

- **抢占式调度**：高优先级线程可以抢占低优先级线程
- **概率性**：同优先级线程按时间片轮转，优先级只是影响被选中概率
- **协作式**：某些系统可能不会强制抢占低优先级线程

### 操作系统差异

| 操作系统 | 优先级行为 |
|----------|------------|
| Windows | 支持 1~10 级，较可靠 |
| Linux | 由内核决定，优先级概念较弱 |
| macOS | 基于 BSD，优先级调度不同 |

## 易错点/踩坑

- ❌ 认为高优先级线程一定先执行 —— 错误！优先级只影响概率，不保证顺序
- ❌ 依赖优先级实现业务逻辑 —— 跨平台行为不一致
- ❌ 将优先级设置过低期望让线程"排队" —— 可能导致线程饿死
- ✅ 优先级用于性能调优，不用于业务排序
- ✅ 结合 `Thread.yield()` 可改善低优先级线程的执行机会

## 代码示例

```java
public class PriorityDemo {
    public static void main(String[] args) {
        Thread low = new Thread(() -> {
            int count = 0;
            while (!Thread.currentThread().isInterrupted()) {
                count++;
                if (count % 10000 == 0) {
                    System.out.println("低优先级: " + count);
                }
            }
        }, "LOW");

        Thread high = new Thread(() -> {
            int count = 0;
            while (!Thread.currentThread().isInterrupted()) {
                count++;
                if (count % 10000 == 0) {
                    System.out.println("高优先级: " + count);
                }
            }
        }, "HIGH");

        low.setPriority(Thread.MIN_PRIORITY);   // 1
        high.setPriority(Thread.MAX_PRIORITY);  // 10

        low.start();
        high.start();

        // 运行多次会发现高优先级线程执行机会更多
        // 但不是绝对的、确定的
    }
}
```
