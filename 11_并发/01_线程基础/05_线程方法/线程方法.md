# start vs run

## 先说结论

`start()` 用于启动新线程，由 JVM 调用 `run()` 方法在新线程中执行；直接调用 `run()` 只是普通方法调用，不会创建新线程，代码仍在当前线程同步执行。

## 深度解析

### 核心区别对比

| 特性 | start() | run() |
|------|---------|-------|
| **线程创建** | ✅ 创建新线程 | ❌ 不创建新线程 |
| **执行方式** | 异步执行 | 同步执行 |
| **调用次数** | 只能调用一次 | 可以多次调用 |
| **调用者** | 用户代码调用 | 用户代码直接调用或 JVM 回调 |
| **状态变化** | NEW → RUNNABLE | 无状态变化 |

### start() 方法详解

```java
public synchronized void start() {
    // 检查线程状态，如果不是 NEW 则抛出异常
    if (threadStatus != 0)
        throw new IllegalThreadStateException();
    
    // 加入线程组
    group.add(this);
    
    boolean started = false;
    try {
        // 调用 native 方法启动线程
        start0();
        started = true;
    } finally {
        // ...
    }
}

private native void start0();  // 由 JVM 实现，调用操作系统创建线程
```

**执行流程：**
1. 检查线程状态是否为 NEW（防止重复启动）
2. 将线程加入线程组
3. 调用 native 方法 `start0()`
4. JVM 向操作系统申请创建新线程
5. 新线程启动后，JVM 回调 `run()` 方法

### run() 方法详解

```java
// Thread 类的默认实现
@Override
public void run() {
    if (target != null) {
        target.run();  // 调用传入的 Runnable 对象
    }
}
```

**两种使用方式：**

1. **继承 Thread 类，重写 run()**
```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("执行线程任务");
    }
}
```

2. **实现 Runnable 接口，传入 Thread**
```java
Runnable task = () -> System.out.println("执行线程任务");
Thread t = new Thread(task);
```

### 调用方式的本质差异

```java
Thread t = new Thread(() -> {
    System.out.println("当前线程: " + Thread.currentThread().getName());
});

// 方式1：调用 start()
t.start();  
// 输出: 当前线程: Thread-0 (新线程)

// 方式2：直接调用 run()
t.run();    
// 输出: 当前线程: main (主线程)
```

## 易错点/踩坑

- ❌ **直接调用 run() 期望异步执行** —— 直接调用 run() 只是普通方法调用，不会创建新线程
- ❌ **多次调用 start()** —— 会抛出 `IllegalThreadStateException`，线程只能启动一次
- ❌ **认为 start() 立即执行** —— start() 只是让线程进入 RUNNABLE 状态，具体执行时机由操作系统调度决定
- ❌ **在 run() 中捕获所有异常就万事大吉** —— 未捕获异常会导致线程终止，但不会通知其他线程
- ❌ **混淆 Thread 和 Runnable** —— Thread 是线程本身，Runnable 是任务逻辑，推荐优先使用 Runnable

## 代码示例

```java
public class StartVsRunDemo {
    
    public static void main(String[] args) {
        System.out.println("主线程: " + Thread.currentThread().getName());
        
        Thread thread = new Thread(() -> {
            System.out.println("任务线程: " + Thread.currentThread().getName());
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("任务执行完毕");
        });
        
        // 演示1：调用 start() - 异步执行
        System.out.println("\n=== 调用 start() ===");
        thread.start();
        System.out.println("start() 已调用，主线程继续执行...");
        
        // 等待线程结束
        try {
            thread.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        // 演示2：直接调用 run() - 同步执行
        System.out.println("\n=== 直接调用 run() ===");
        thread.run();  // 注意：这里不会创建新线程！
        System.out.println("run() 执行完毕，主线程继续...");
        
        // 演示3：多次调用 start() 会报错
        System.out.println("\n=== 再次调用 start() ===");
        try {
            thread.start();  // 抛出 IllegalThreadStateException
        } catch (IllegalThreadStateException e) {
            System.out.println("错误: " + e.getMessage());
        }
    }
}
```

# sleep vs wait

## 先说结论

`sleep()` 是 Thread 类的静态方法，不释放锁，时间到自动唤醒；`wait()` 是 Object 类的方法，必须配合 synchronized 使用，会释放锁并进入等待队列，需要被 notify()/notifyAll() 唤醒或超时唤醒。

## 深度解析

### 核心区别对比

|特性|sleep()|wait()|
|---|---|---|
|**所属类**|Thread (静态方法)|Object (实例方法)|
|**锁释放**|❌ 不释放锁|✅ 释放锁|
|**调用位置**|任意位置|必须在 synchronized 块内|
|**唤醒方式**|时间到自动唤醒|需 notify/notifyAll 或超时|
|**使用场景**|暂停执行一段时间|线程间协作/条件等待|
|**异常**|会抛出 InterruptedException|会抛出 InterruptedException|

### sleep() 详解

```java
// 两种重载形式
public static native void sleep(long millis) throws InterruptedException;
public static void sleep(long millis, int nanos) throws InterruptedException;
```

**特点：**

- 让当前线程暂停执行指定时间
- **不释放任何锁**（包括 synchronized 锁和显式锁）
- 时间到后自动恢复 RUNNABLE 状态
- 可被 interrupt() 提前唤醒，抛出 InterruptedException

```java
public class SleepDemo {
    private static final Object lock = new Object();
    
    public static void main(String[] args) {
        Thread t = new Thread(() -> {
            synchronized (lock) {
                System.out.println("获取锁，准备 sleep");
                try {
                    Thread.sleep(5000);  // 不释放锁！
                } catch (InterruptedException e) {
                    System.out.println("被中断唤醒");
                }
                System.out.println("sleep 结束，释放锁");
            }
        });
        
        t.start();
        
        // 主线程尝试获取锁，会被阻塞 5 秒
        synchronized (lock) {
            System.out.println("主线程获取到锁");
        }
    }
}
```

### wait() 详解

```java
// 三种重载形式
public final void wait() throws InterruptedException;           // 无限等待
public final native void wait(long timeout) throws InterruptedException;  // 限时等待
public final void wait(long timeout, int nanos) throws InterruptedException;
```

**特点：**

- 必须在 synchronized 块内调用，否则会抛出 `IllegalMonitorStateException`
- **会释放对象的 monitor 锁**
- 线程进入 WAITING/TIMED_WAITING 状态
- 需要被 notify()/notifyAll() 唤醒，或超时自动唤醒
- 被唤醒后需要重新竞争获取锁

```java
public class WaitDemo {
    private static final Object lock = new Object();
    private static boolean flag = false;
    
    public static void main(String[] args) throws InterruptedException {
        // 消费者线程
        Thread consumer = new Thread(() -> {
            synchronized (lock) {
                while (!flag) {
                    System.out.println("条件不满足，进入 wait");
                    try {
                        lock.wait();  // 释放锁，进入等待
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
                System.out.println("条件满足，继续执行");
            }
        });
        
        // 生产者线程
        Thread producer = new Thread(() -> {
            synchronized (lock) {
                try {
                    Thread.sleep(2000);  // 模拟处理时间
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                flag = true;
                System.out.println("设置条件，唤醒消费者");
                lock.notify();  // 唤醒等待的线程
            }
        });
        
        consumer.start();
        Thread.sleep(100);  // 确保消费者先执行
        producer.start();
    }
}
```

### 执行流程对比

```
sleep() 执行流程：
┌─────────┐    ┌──────────┐    ┌──────────────┐    ┌──────────┐
│ 获取锁   │───►│ sleep()  │───►│ TIMED_WAITING │───►│ 自动唤醒  │
│         │    │ (不释放锁)│    │ (持有锁等待)   │    │ 继续执行  │
└─────────┘    └──────────┘    └──────────────┘    └──────────┘

wait() 执行流程：
┌─────────┐    ┌──────────┐    ┌──────────────┐    ┌──────────┐
│ 获取锁   │───►│ wait()   │───►│ WAITING      │───►│ 被 notify │
│         │    │ (释放锁)  │    │ (释放锁等待)  │    │ 重新竞争锁 │
└─────────┘    └──────────┘    └──────────────┘    └──────────┘
```

## 易错点/踩坑

- ❌ **认为 sleep() 会释放锁** —— sleep() 不释放任何锁，其他线程无法进入同步代码块
- ❌ **在 synchronized 外调用 wait()** —— 会抛出 `IllegalMonitorStateException`
- ❌ **wait() 被唤醒后立即执行** —— 被唤醒后需要重新竞争锁，获取到锁后才能继续
- ❌ **用 if 判断条件后 wait** —— 应该用 while，防止虚假唤醒（spurious wakeup）
- ❌ **忘记 notify/notifyAll** —— 调用 wait() 的线程如果没有被唤醒，会一直等待
- ❌ **混淆两者使用场景** —— sleep 用于暂停，wait 用于线程协作

## 代码示例

```java
public class SleepVsWaitDemo {
    
    private static final Object lock = new Object();
    
    public static void main(String[] args) throws InterruptedException {
        System.out.println("=== 测试 sleep() 不释放锁 ===");
        testSleep();
        
        Thread.sleep(1000);
        
        System.out.println("\n=== 测试 wait() 释放锁 ===");
        testWait();
    }
    
    // 测试 sleep() 不释放锁
    static void testSleep() throws InterruptedException {
        Thread t1 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("[sleep] t1 获取锁");
                try {
                    Thread.sleep(3000);  // 持有锁睡眠
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("[sleep] t1 释放锁");
            }
        });
        
        Thread t2 = new Thread(() -> {
            System.out.println("[sleep] t2 尝试获取锁...");
            synchronized (lock) {
                System.out.println("[sleep] t2 获取到锁");
            }
        });
        
        long start = System.currentTimeMillis();
        t1.start();
        Thread.sleep(100);  // 确保 t1 先执行
        t2.start();
        t2.join();
        System.out.println("[sleep] 总耗时: " + (System.currentTimeMillis() - start) + "ms");
        // 结果：约 3000ms，说明 t2 等待了 t1 的 sleep
    }
    
    // 测试 wait() 释放锁
    static void testWait() throws InterruptedException {
        Thread t1 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("[wait] t1 获取锁");
                try {
                    lock.wait(3000);  // 释放锁等待
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("[wait] t1 被唤醒，继续执行");
            }
        });
        
        Thread t2 = new Thread(() -> {
            System.out.println("[wait] t2 尝试获取锁...");
            synchronized (lock) {
                System.out.println("[wait] t2 获取到锁");
                lock.notify();  // 唤醒 t1
            }
        });
        
        long start = System.currentTimeMillis();
        t1.start();
        Thread.sleep(100);  // 确保 t1 先执行并进入 wait
        t2.start();
        t1.join();
        System.out.println("[wait] 总耗时: " + (System.currentTimeMillis() - start) + "ms");
        // 结果：约 100ms，说明 t2 立即获取了锁
    }
}
```

# yield vs join

## 先说结论

`yield()` 只是建议操作系统将 CPU 时间片让给其他就绪状态的线程，**不能保证真正让出**；`join()` 则强制当前线程等待目标线程执行完毕才会继续执行，是一种**阻塞式等待**。

## 深度解析

### yield() 方法

`yield()` 是 `Thread` 类的静态方法，调用它意味着告诉调度器："我愿意让出 CPU，但如果有需要我也可以继续执行"。

```java
public static void yield() {
    Thread.yield0();
}
```

**特点**：

- 只是一个**提示(hint)**，对 JVM 和操作系统没有强制约束
- 实际效果取决于底层线程调度实现
- 常用于**测试和调试**，观察线程调度行为
- 不会释放任何锁

### join() 方法

`join()` 是 `Thread` 实例方法，表示"等待这个线程执行完毕"。

```java
public final void join() throws InterruptedException {
    join(0);
}

public final synchronized void join(long millis) throws InterruptedException {
    // 无限等待或超时等待
    while (isAlive()) {
        wait(0);  // 释放锁，等待 notify
    }
}
```

**特点**：

- 底层调用 `wait()`，会**释放对象锁**
- 支持超时参数：`join(ms)` 最多等待指定毫秒
- 线程终止时会自动调用 `notifyAll()`

## 易错点/踩坑

- ❌ 误以为 `yield()` 一定能切换线程 —— 实际上调度器可以忽略这个提示
- ❌ 在持有锁时调用 `yield()` —— 它不会释放锁，可能导致其他线程饿死
- ❌ 使用 `yield()` 实现同步 —— 完全没有同步保证
- ✅ `join()` 可以被中断，抛出 `InterruptedException`
- ✅ `join()` 保证线程执行顺序的因果关系

## 代码示例

```java
public class YieldJoinDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            for (int i = 0; i < 3; i++) {
                System.out.println("子线程 - " + i);
            }
        }, "Worker");

        t.start();

        // join: 等待子线程执行完毕
        t.join();

        System.out.println("主线程 - 执行完成");
        // 此时确保子线程已经执行完毕
    }
}
```

```java
public class YieldDemo {
```

## 其他常用方法

### 线程标识与命名

```java
// 获取当前正在执行的线程对象
Thread current = Thread.currentThread();

// 获取/设置线程名称
System.out.println(current.getName());   // 默认 "Thread-0"、"main" 等
current.setName("my-worker-thread");

// 获取线程 ID（唯一，不可重复）
System.out.println(current.getId());

// 判断线程是否存活（已 start 但未 TERMINATED）
Thread t = new Thread(() -> {});
System.out.println(t.isAlive()); // false（未 start）
t.start();
System.out.println(t.isAlive()); // true（执行中）
```

> **最佳实践**：创建线程时通过构造函数设置有意义的名称，方便通过 `jstack` 排查问题。

### UncaughtExceptionHandler 完整示例

当线程的 `run()` 抛出未捕获异常时，线程终止，但异常默认只打印到控制台。通过 `UncaughtExceptionHandler` 可以统一处理：

```java
public class UncaughtExceptionDemo {
    public static void main(String[] args) {
        // 1. 为单个线程设置异常处理器
        Thread t = new Thread(() -> {
            throw new RuntimeException("线程内部异常");
        }, "worker-1");
        
        t.setUncaughtExceptionHandler((thread, ex) -> {
            System.err.println("线程 [" + thread.getName() + "] 发生未捕获异常：" + ex.getMessage());
            // 可在这里：记录日志、告警、上报监控等
        });
        t.start();
        
        // 2. 为所有线程设置全局默认处理器
        Thread.setDefaultUncaughtExceptionHandler((thread, ex) -> {
            System.err.println("全局处理：线程 " + thread.getName() + " 异常: " + ex);
        });
        
        // 3. 配合线程工厂在线程池中使用
        ThreadFactory factory = r -> {
            Thread thread = new Thread(r);
            thread.setUncaughtExceptionHandler((th, ex) ->
                System.err.println("线程池线程异常: " + ex.getMessage()));
            return thread;
        };
        ExecutorService pool = Executors.newFixedThreadPool(2, factory);
    }
}
```
