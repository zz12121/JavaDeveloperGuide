# 线程生命周期（NEW → RUNNABLE → BLOCKED → WAITING → TIMED_WAITING → TERMINATED）

## 先说结论

Java线程有6种状态：**NEW（新建）→ RUNNABLE（可运行）→ BLOCKED（阻塞）→ WAITING（等待）→ TIMED_WAITING（超时等待）→ TERMINATED（终止）**。
线程状态转换由JVM管理，状态反映的是线程在JVM层面的执行情况，而非操作系统的真实调度状态。

## 深度解析

### 六种状态详解

| 状态 | 含义 | 转换条件 |
|------|------|----------|
| NEW | 线程创建但未启动 | new Thread() |
| RUNNABLE | 可运行或正在运行 | start() |
| BLOCKED | 等待获取监视器锁 | 进入synchronized块但锁被占用 |
| WAITING | 无限期等待 | Object.wait()/Thread.join()/LockSupport.park() |
| TIMED_WAITING | 限时等待 | Thread.sleep()/Object.wait(timeout)/Thread.join(timeout) |
| TERMINATED | 线程执行完毕 | run()正常返回或抛出未捕获异常 |

### Java 线程状态与 OS 线程状态映射

| Java 线程状态 | 对应 OS 线程状态 | 说明 |
|---------------|-----------------|------|
| NEW | 未创建 | Java 对象创建，OS 线程还未分配 |
| RUNNABLE | Ready + Running | Java 将就绪和运行合并为一个状态 |
| BLOCKED | Blocked/Waiting | 等待 synchronized 锁，OS 层面挂起 |
| WAITING | Waiting/Sleeping | 主动等待，OS 层面挂起 |
| TIMED_WAITING | Timed Waiting | 限时等待，OS 层面挂起 |
| TERMINATED | Terminated | 线程执行完毕，OS 线程已销毁 |

> **关键点**：Java 的 **RUNNABLE** 包含了 OS 层面的"就绪（Ready）"和"运行（Running）"两种状态，因为 JVM 无法区分线程是在等待 CPU 还是已经获得 CPU 正在运行。

### BLOCKED vs WAITING

- **BLOCKED**：等待获取 synchronized 锁，被动等待，锁释放后自动唤醒
- **WAITING**：调用 `Object.wait()` / `LockSupport.park()` 等方法主动放弃 CPU，主动等待，需其他线程显式唤醒

## 易错点/踩坑

- ❌ 混淆操作系统线程状态与Java线程状态 → Java线程状态是JVM层面的，RUNNABLE包含就绪和运行两种情况
- ❌ 认为BLOCKED和WAITING是同一种状态 → BLOCKED是等待锁，WAITING是主动等待
- ❌ sleep()会释放锁 → sleep()不释放锁，而wait()会释放锁

## 代码示例

```java
public class ThreadStateDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            synchronized (ThreadStateDemo.class) {
                System.out.println("线程执行中，状态: RUNNABLE");
            }
        }, "测试线程");
        
        System.out.println("创建后: " + thread.getState()); // NEW
        
        thread.start();
        System.out.println("启动后: " + thread.getState()); // RUNNABLE
        
        Thread.sleep(100);
        System.out.println("等待锁时: " + thread.getState()); // BLOCKED
        
        thread.join();
        System.out.println("执行完毕后: " + thread.getState()); // TERMINATED
    }
}
```
