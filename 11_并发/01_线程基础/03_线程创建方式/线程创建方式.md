# 线程创建方式（Thread / Runnable / Callable + Future / 线程池）

## 先说结论

Java中有4种创建线程的方式：1）继承Thread类；2）实现Runnable接口；3）实现Callable接口配合FutureTask；4）使用线程池。推荐使用Runnable或线程池方式，避免单继承限制和资源浪费。

## 深度解析

### 四种创建方式对比

| 方式 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| Thread类 | 简单直接 | 单继承限制 | 简单场景 |
| Runnable | 解耦、可多实现 | 无返回值 | 通用场景 |
| Callable+Future | 有返回值、可抛异常 | 需配合FutureTask | 需要结果 |
| 线程池 | 复用线程、控制并发 | 需理解线程池参数 | 生产环境 |

### 线程池核心参数

```java
public ThreadPoolExecutor(
    int corePoolSize,      // 核心线程数
    int maximumPoolSize,   // 最大线程数
    long keepAliveTime,    // 空闲线程存活时间
    TimeUnit unit,         // 时间单位
    BlockingQueue<Runnable> workQueue,  // 任务队列
    ThreadFactory threadFactory,         // 线程工厂
    RejectedExecutionHandler handler     // 拒绝策略
)
```

### 常见线程池类型

- **newFixedThreadPool**：固定线程数，无界队列
- **newCachedThreadPool**：按需创建，60秒回收
- **newSingleThreadExecutor**：单线程串行执行
- **newScheduledThreadPool**：定时/周期任务

## 易错点/踩坑

- ❌ 直接new Thread()创建大量线程 → 线程创建销毁开销大，应使用线程池
- ❌ Thread.start()可多次调用 → 只会执行一次，多次调用抛出IllegalThreadStateException
- ❌ 线程池核心线程不会回收 → allowCoreThreadTimeOut(true)后核心线程也会回收
- ❌ Callable直接传Thread构造方法 → Callable需包装成FutureTask才能用于Thread

## 代码示例

```java
public class ThreadCreationDemo {
    public static void main(String[] args) throws Exception {
        // 方式1：继承Thread
        Thread t1 = new Thread("T1") {
            @Override
            public void run() {
                System.out.println("方式1：继承Thread");
            }
        };
        
        // 方式2：实现Runnable
        Thread t2 = new Thread(() -> System.out.println("方式2：Runnable"), "T2");
        
        // 方式3：Callable + FutureTask
        FutureTask<Integer> future = new FutureTask<>(() -> {
            System.out.println("方式3：Callable");
            return 42;
        });
        new Thread(future, "T3").start();
        System.out.println("Callable结果: " + future.get());
        
        // 方式4：线程池
        ExecutorService pool = Executors.newFixedThreadPool(2);
        pool.submit(() -> System.out.println("方式4：线程池"));
        pool.shutdown();
    }
}
```


### 四种拒绝策略（RejectedExecutionHandler）

当线程池已满（队列满 + 线程数达到最大值）时，触发拒绝策略：

| 策略类 | 行为 | 适用场景 |
|--------|------|----------|
| `AbortPolicy`（默认） | 抛出 `RejectedExecutionException` | 快速发现问题，需调用方处理 |
| `CallerRunsPolicy` | 由调用者所在线程执行任务 | 降速效果，避免丢任务 |
| `DiscardPolicy` | 直接丢弃任务，不抛异常 | 允许丢失的非关键任务 |
| `DiscardOldestPolicy` | 丢弃队列头部最老的任务，然后重试 | 更愿意保留新任务的场景 |

```java
// 自定义线程池 + 拒绝策略示例
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    2,                              // 核心线程数
    4,                              // 最大线程数
    60, TimeUnit.SECONDS,           // 空闲线程存活时间
    new LinkedBlockingQueue<>(10),  // 任务队列容量
    Executors.defaultThreadFactory(),
    new ThreadPoolExecutor.CallerRunsPolicy()  // 调用者执行策略
);
```

### 优雅关闭线程池

| 方法 | 行为 | 是否等待任务完成 |
|------|------|-----------------|
| `shutdown()` | 不再接受新任务，等待已提交的任务全部执行完毕 | ✅ 等待 |
| `shutdownNow()` | 不再接受新任务，尝试中断正在执行的任务，返回未执行的任务列表 | ❌ 不等待 |
| `awaitTermination(timeout)` | 阻塞等待线程池关闭，超时返回 false | 需配合上面两个使用 |

```java
// 推荐的优雅关闭方式
ExecutorService pool = Executors.newFixedThreadPool(4);
pool.submit(() -> System.out.println("任务执行"));

// 1. 停止接受新任务
pool.shutdown();
try {
    // 2. 等待最多 60 秒让任务完成
    if (!pool.awaitTermination(60, TimeUnit.SECONDS)) {
        // 3. 超时则强制关闭
        pool.shutdownNow();
        // 4. 再等一下，确保强制关闭完成
        if (!pool.awaitTermination(60, TimeUnit.SECONDS)) {
            System.err.println("线程池未能正常关闭");
        }
    }
} catch (InterruptedException e) {
    pool.shutdownNow();
    Thread.currentThread().interrupt();
}
```
