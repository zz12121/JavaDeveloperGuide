# 守护线程（Daemon Thread）

## 先说结论

守护线程（Daemon Thread）是 JVM 的"后台服务线程"，当 JVM 中所有用户线程（非守护线程）都结束时，**JVM 会立即退出**，不会等待守护线程执行完毕。典型的守护线程如垃圾回收（GC）线程。

## 深度解析

### 守护线程特性

```java
// 设置为守护线程，必须在 start() 之前调用
thread.setDaemon(true);
thread.start();
```

**核心规则**：
1. 守护线程不能执行关键业务逻辑
2. 守护线程创建的子线程默认也是守护线程
3. 必须在 `start()` 之前设置，之后不可更改
4. 所有用户线程结束后，JVM 直接退出，守护线程被强制终止

### 用户线程 vs 守护线程

| 特性 | 用户线程 | 守护线程 |
|------|----------|----------|
| 生命周期 | 必须执行完毕 | JVM退出时强制终止 |
| 典型例子 | 业务逻辑线程 | GC线程、监控线程 |
| 设置方式 | 默认（isDaemon=false） | setDaemon(true) |
| finally 块 | 正常执行 | **不保证执行** |

### 危险场景

```java
public class DaemonDanger {
    public static void main(String[] args) {
        Thread daemon = new Thread(() -> {
            try {
                // 关键业务操作
                saveToDatabase();
            } finally {
                // ⚠️ 这里可能不执行！
                closeConnection();
            }
        });
        daemon.setDaemon(true);
        daemon.start();

        // 主线程结束
        System.out.println("主线程结束");
    }
}
```

### 常见守护线程

- **GC 线程**：垃圾回收器的工作线程
- **Finalizer 线程**：执行对象的 finalize() 方法
- **Signal Dispatcher**：处理操作系统信号
- **Reference Handler**：管理 Reference 对象

## 易错点/踩坑

- ❌ 在守护线程中执行关键业务逻辑 —— JVM 退出时数据可能丢失
- ❌ 在 start() 之后调用 setDaemon() —— 抛出 IllegalThreadStateException
- ❌ 依赖守护线程的 finally 块做清理 —— 不保证执行
- ❌ 混淆守护线程和优先级 —— 两者是完全独立的概念
- ✅ 守护线程只用于"后台服务"类任务
- ✅ 使用守护线程时要考虑优雅退出的问题

## 代码示例

```java
public class DaemonThreadDemo {
    public static void main(String[] args) {
        // 用户线程
        Thread userThread = new Thread(() -> {
            System.out.println("用户线程开始");
            try {
                Thread.sleep(1000);
                System.out.println("用户线程执行完毕");
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }, "UserThread");

        // 守护线程
        Thread daemonThread = new Thread(() -> {
            System.out.println("守护线程开始");
            int count = 0;
            while (true) {
                count++;
                System.out.println("守护线程运行: " + count);
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    break;
                }
            }
        }, "DaemonThread");
        daemonThread.setDaemon(true);

        userThread.start();
        daemonThread.start();

        // 用户线程sleep 2秒后结束
        // 此时JVM会直接退出，不管守护线程是否在运行
    }
}
```

```java
public class DaemonInherit {
    public static void main(String[] args) {
        Thread parent = new Thread(() -> {
            System.out.println("父线程是守护线程吗？ " +
                Thread.currentThread().isDaemon());

            // 在守护线程中创建子线程
            Thread child = new Thread(() -> {
                System.out.println("子线程是守护线程吗？ " +
                    Thread.currentThread().isDaemon());
                // 子线程会继承父线程的守护属性
            });
            child.start();
        });
        parent.setDaemon(true);
        parent.start();
    }
}
```

### 优雅退出：ShutdownHook

`Runtime.addShutdownHook()` 是 JVM 提供的优雅退出机制，允许在 JVM 退出前执行清理逻辑，是解决守护线程资源无法释放问题的官方方案：

```java
public class ShutdownHookDemo {
    public static void main(String[] args) {
        // 注册 JVM 关闭钩子（本质是一个用户线程）
        Runtime.getRuntime().addShutdownHook(new Thread(() -> {
            System.out.println("JVM 即将退出，执行清理...");
            // 可在这里：关闭连接池、刷新日志缓冲、保存状态等
            closeResources();
        }, "shutdown-hook-thread"));

        // 守护线程（JVM退出时无法保证finally执行）
        Thread daemon = new Thread(() -> {
            try {
                while (true) {
                    processTask();
                    Thread.sleep(1000);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            } finally {
                // ⚠️ 守护线程的 finally 不保证执行，不应在这里做关键清理
            }
        });
        daemon.setDaemon(true);
        daemon.start();

        // 主线程执行完毕 → 触发 ShutdownHook → JVM 退出
        System.out.println("主线程结束");
    }

    private static void closeResources() {
        System.out.println("资源清理完成");
    }
    
    private static void processTask() { /* 任务处理 */ }
}
```

**ShutdownHook 关键规则**：
1. ShutdownHook 本质是**用户线程**，JVM 退出时会等待所有 ShutdownHook 执行完毕
2. 可注册多个 ShutdownHook，它们会**并发执行**（顺序不保证）
3. ShutdownHook 中不能调用 `System.exit()`，会导致死锁
4. JVM 被强制 kill（`kill -9`）时 ShutdownHook **不会执行**

| 场景 | 守护线程 finally | ShutdownHook |
|------|-----------------|--------------|
| 正常退出（`System.exit()`） | ❌ 不保证 | ✅ 执行 |
| `kill -15`（SIGTERM） | ❌ 不保证 | ✅ 执行 |
| `kill -9`（SIGKILL） | ❌ 不执行 | ❌ 不执行 |
