# 线程状态转换（各种状态之间的转换条件和触发方式）

## 先说结论

Java 线程有 6 种状态：NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED。状态转换由 JVM 和操作系统共同管理，核心触发方式包括：start() 启动、sleep()/wait() 等待、notify()/notifyAll() 唤醒、锁竞争阻塞、run() 执行完毕等。

## 深度解析

### Java 线程的 6 种状态

```
NEW ───────────► RUNNABLE ◄──────────────────┐
                 │    │                      │
                 │    ▼                      │
                 │  RUNNING (操作系统层面)    │
                 │    │                      │
                 ▼    ▼                      │
              BLOCKED (等待锁)               │
                 │                           │
                 ▼                           │
             WAITING (无限等待) ◄─────────────┤
                 │                           │
                 ▼                           │
          TIMED_WAITING (限时等待) ◄──────────┘
                 │
                 ▼
            TERMINATED
```

### 各状态详解

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| **NEW** | 新建状态 | 线程被创建但未调用 start() |
| **RUNNABLE** | 可运行状态 | 调用 start() 后，等待 CPU 调度 |
| **BLOCKED** | 阻塞状态 | 等待获取 monitor 锁 |
| **WAITING** | 等待状态 | 调用 wait()、join() 等，无限期等待 |
| **TIMED_WAITING** | 限时等待 | 调用 sleep(time)、wait(time)、join(time) |
| **TERMINATED** | 终止状态 | run() 执行完毕或抛出异常 |

### 状态转换触发方式

#### 1. NEW → RUNNABLE
```java
Thread t = new Thread(() -> {});
t.start();  // 调用 start() 后进入 RUNNABLE
```

#### 2. RUNNABLE → BLOCKED
- 线程试图进入 synchronized 代码块/方法，但锁被其他线程持有
- 等待获取 ReentrantLock 等显式锁

#### 3. RUNNABLE → WAITING
```java
// 方式1：wait()
synchronized(obj) {
    obj.wait();  // 释放锁，进入 WAITING
}

// 方式2：join()
thread.join();  // 等待 thread 执行完毕

// 方式3：LockSupport.park()
LockSupport.park();
```

#### 4. RUNNABLE → TIMED_WAITING
```java
Thread.sleep(1000);           // 不释放锁，限时等待
obj.wait(1000);               // 释放锁，限时等待
thread.join(1000);            // 限时等待 thread 执行完毕
LockSupport.parkNanos(1000);  // 纳秒级限时等待
```

#### 5. WAITING/TIMED_WAITING → RUNNABLE
```java
// 被其他线程唤醒
obj.notify();      // 唤醒一个等待线程
obj.notifyAll();   // 唤醒所有等待线程

// 或超时自动唤醒
```

#### 6. BLOCKED → RUNNABLE
- 获取到 monitor 锁后自动恢复

#### 7. RUNNABLE → TERMINATED
- run() 方法正常执行完毕
- run() 方法抛出未捕获异常

## 易错点/踩坑

- ❌ **认为调用 start() 后线程立即执行** —— 实际是进入 RUNNABLE 等待 CPU 调度，何时执行由操作系统决定
- ❌ **认为 sleep() 会释放锁** —— sleep() 不释放任何锁，只是让出 CPU
- ❌ **认为 wait() 不释放锁** —— wait() 必须配合 synchronized 使用，会释放 monitor 锁
- ❌ **混淆 BLOCKED 和 WAITING** —— BLOCKED 是等锁，WAITING 是等通知
- ❌ **线程状态只能获取不能设置** —— Thread.getState() 是只读的，无法手动改变线程状态

## 代码示例

```java
public class ThreadStateDemo {
    
    private static final Object lock = new Object();
    
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            // RUNNABLE → TIMED_WAITING
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            
            // RUNNABLE → BLOCKED (尝试获取锁)
            synchronized (lock) {
                System.out.println("获取到锁");
            }
        });
        
        // NEW
        System.out.println("1. 新建状态: " + t.getState());  // NEW
        
        t.start();
        // RUNNABLE (可能还在等待调度)
        System.out.println("2. 启动后: " + t.getState());   // RUNNABLE
        
        // 主线程先占用锁，让 t 进入 BLOCKED
        synchronized (lock) {
            Thread.sleep(200);  // 确保 t 已经启动并尝试获取锁
            System.out.println("3. 等待锁时: " + t.getState());  // BLOCKED
        }
        
        t.join();
        // TERMINATED
        System.out.println("4. 结束后: " + t.getState());  // TERMINATED
    }
}
```
