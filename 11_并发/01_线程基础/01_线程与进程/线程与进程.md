# 线程与进程（进程是资源分配单位，线程是CPU调度单位，共享进程资源）

## 先说结论

**进程**是资源分配的基本单位，拥有独立的地址空间；**线程**是CPU调度的基本单位，同一进程内的线程共享进程的堆和方法区资源。线程是轻量级的进程，创建、切换和销毁的开销远小于进程。

## 深度解析

### 进程与线程的核心区别

| 对比维度 | 进程 | 线程 |
|---------|------|------|
| 资源分配 | 操作系统分配资源的基本单位 | 操作系统调度CPU的基本单位 |
| 地址空间 | 独立的虚拟地址空间 | 共享进程的地址空间 |
| 通信方式 | IPC（管道、消息队列、共享内存等） | 直接通过共享内存通信 |
| 开销 | 创建/切换开销大（需要复制资源） | 创建/切换开销小（共享资源） |
| 独立性 | 进程间相互隔离，一个崩溃不影响其他 | 线程共享资源，一个崩溃可能导致整个进程崩溃 |

### JVM内存结构中的进程与线程

```
JVM 进程内存结构
├── 堆（Heap）        ← 所有线程共享
├── 方法区（Method Area）← 所有线程共享
├── 虚拟机栈（VM Stack）← 每个线程私有
├── 本地方法栈（Native Stack）← 每个线程私有
└── 程序计数器（PC Register）← 每个线程私有
```

### 线程的组成

每个线程包含：
- **程序计数器**：记录当前线程执行的字节码行号
- **虚拟机栈**：存储局部变量、操作数栈、方法调用等
- **本地方法栈**：执行native方法
- **共享资源**：堆内存、方法区

## 易错点/踩坑

- ❌ 认为线程创建开销与进程相同 → 线程创建只需分配少量私有资源，开销远小于进程
- ❌ 认为线程间通信需要IPC机制 → 同一进程的线程可直接通过共享变量通信（需注意同步）
- ❌ 认为堆内存是线程私有的 → 堆是进程内所有线程共享的

## 代码示例

```java
public class ProcessVsThreadDemo {
    // 共享变量（所有线程可见）
    private static int sharedCounter = 0;
    
    public static void main(String[] args) {
        // 演示多线程共享进程资源
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                sharedCounter++; // 存在线程安全问题，需要同步
            }
        }, "线程1");
        
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                sharedCounter++;
            }
        }, "线程2");
        
        t1.start();
        t2.start();
        
        try {
            t1.join();
            t2.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        System.out.println("最终计数: " + sharedCounter); // 可能小于2000
    }
}
```

### 并发与并行

| 概念 | 定义 | 关键词 | 比喻 |
|------|------|--------|------|
| **并发（Concurrency）** | 多个任务在同一时间段内交替执行 | 交替、时间段 | 一个厨师同时炒多道菜（来回切换） |
| **并行（Parallelism）** | 多个任务在同一时刻同时执行 | 同时、多核 | 多个厨师同时各炒一道菜 |

> **面试一句话总结**：并发是"逻辑上同时"，并行是"物理上同时"。单核 CPU 只能并发，多核 CPU 可以做到真正的并行。

```
并发（单核 CPU）：
CPU: ──┬── 任务A ──┬── 任务B ──┬── 任务A ──▶ 时间轴
       └──切换──┘           └──切换──┘

并行（多核 CPU）：
CPU1: ─── 任务A ─────────────────────────▶ 时间轴
CPU2: ─── 任务B ─────────────────────────▶ 时间轴
```

### 用户态与内核态线程切换

线程创建和切换开销小的底层原因：
- **用户态**：程序代码运行的空间，不能直接访问硬件
- **内核态**：操作系统内核运行的空间，可以直接访问硬件和系统资源
- **切换开销来源**：每次从用户态切换到内核态（如系统调用）需要保存/恢复寄存器上下文，约耗费数百纳秒
- **线程 vs 进程开销差异**：创建进程需要复制内存空间、文件描述符等资源；创建线程只需分配栈空间和少量私有数据，开销约小 10~100 倍

### 拓展：Java 21 虚拟线程（Virtual Thread）

Java 21 正式引入了虚拟线程（Virtual Thread），是对传统平台线程的重要补充：

| 对比项 | 平台线程（Platform Thread） | 虚拟线程（Virtual Thread） |
|--------|----------------------------|--------------------------|
| 底层映射 | 1:1 映射到 OS 线程 | N:M 映射到少量 OS 线程（由 JVM 调度） |
| 创建成本 | 较高（约 1MB 栈空间） | 极低（初始几百字节） |
| 数量限制 | 通常数千个 | 可创建数百万个 |
| 适用场景 | CPU 密集型任务 | I/O 密集型、高并发任务 |
| 阻塞行为 | 阻塞操作会占用 OS 线程 | 阻塞时自动卸载，不占用 OS 线程 |

```java
// Java 21 创建虚拟线程
Thread vt = Thread.ofVirtual().start(() -> {
    System.out.println("虚拟线程：" + Thread.currentThread().isVirtual()); // true
});

// 使用虚拟线程池（推荐用于大量 I/O 任务）
try (ExecutorService exec = Executors.newVirtualThreadPerTaskExecutor()) {
    exec.submit(() -> System.out.println("虚拟线程执行任务"));
}
```
