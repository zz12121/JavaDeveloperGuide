# 原子类分类（基本类型/数组类型/引用类型/字段更新器）

## 先说结论

JUC 原子类按操作数据类型分为四大类：**基本类型**（AtomicInteger/AtomicLong/AtomicBoolean）、**数组类型**（AtomicIntegerArray/AtomicLongArray/AtomicReferenceArray）、**引用类型**（AtomicReference/AtomicStampedReference/AtomicMarkableReference）、**字段更新器**（AtomicIntegerFieldUpdater/AtomicLongFieldUpdater/AtomicReferenceFieldUpdater）。底层全部基于 `Unsafe` + `volatile` + CAS 实现。

## 深度解析

### 四大分类

```
java.util.concurrent.atomic
│
├── 基本类型原子类
│   ├── AtomicInteger      — int 原子操作
│   ├── AtomicLong         — long 原子操作
│   └── AtomicBoolean      — boolean 原子操作
│
├── 数组类型原子类
│   ├── AtomicIntegerArray  — int[] 数组元素原子更新
│   ├── AtomicLongArray     — long[] 数组元素原子更新
│   └── AtomicReferenceArray — E[] 引用数组元素原子更新
│
├── 引用类型原子类
│   ├── AtomicReference<V>            — 对象引用原子更新
│   ├── AtomicStampedReference<V>     — 带版本号的引用（解决ABA）
│   └── AtomicMarkableReference<V>    — 带标记位的引用（解决ABA）
│
└── 字段更新器
    ├── AtomicIntegerFieldUpdater<T>  — 原子更新 volatile int 字段
    ├── AtomicLongFieldUpdater<T>     — 原子更新 volatile long 字段
    └── AtomicReferenceFieldUpdater<T,V> — 原子更新 volatile 引用字段
```

### 设计原则

| 原则 | 说明 |
|------|------|
| **volatile 语义** | 所有原子类的值字段都用 `volatile` 修饰，保证可见性 |
| **CAS 原子性** | 通过 `Unsafe.compareAndSetXxx()` 保证更新原子性 |
| **无锁并发** | 不使用 synchronized，避免线程阻塞 |
| **自旋重试** | CAS 失败后在循环中重试 |

### 累加器（JDK8+）

```
java.util.concurrent.atomic
├── LongAdder          — 高并发 long 累加（分段CAS）
├── LongAccumulator    — 通用版 LongAdder（自定义运算）
├── DoubleAdder        — 高并发 double 累加
└── DoubleAccumulator  — 通用版 DoubleAdder
```

## 易错点/踩坑

- ❌ 原子类的复合操作不是原子的——`i++` 和 `if(get() == 0) set(1)` 不是原子操作
- ❌ AtomicIntegerFieldUpdater 要求字段是 `volatile int`——非 volatile 字段会抛异常
- ✅ AtomicStampedReference 和 AtomicMarkableReference 只解决引用的 ABA，不解决基本类型的 ABA

## 代码示例

```java
// 基本类型
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();       // ++i
count.getAndIncrement();       // i++
count.addAndGet(5);            // i += 5
count.compareAndSet(10, 20);   // CAS

// 数组类型
AtomicIntegerArray array = new AtomicIntegerArray(10);
array.set(0, 100);             // array[0] = 100
array.incrementAndGet(0);      // ++array[0]
array.compareAndSet(0, 100, 200); // CAS array[0]

// 引用类型
AtomicReference<String> ref = new AtomicReference<>("hello");
ref.compareAndSet("hello", "world");

// 字段更新器
AtomicIntegerFieldUpdater<User> AGE_UPDATER =
    AtomicIntegerFieldUpdater.newUpdater(User.class, "age");

class User {
    volatile int age;  // 必须是 volatile
}

AGE_UPDATER.incrementAndGet(user);  // 原子 age++
```

### 完整对比表

#### 基本类型原子类

| 类 | 操作类型 | 主要 API | 适用场景 |
|----|----------|----------|----------|
| `AtomicInteger` | int | incrementAndGet / getAndAdd / compareAndSet | 计数器、序号生成 |
| `AtomicLong` | long | 同上 | 超过 int 范围的值 |
| `AtomicBoolean` | boolean | set / compareAndSet | 标志位、状态切换 |

> **注意**：`AtomicLong` 在高并发下性能远差于 `LongAdder`，若无 `get()` 返回精确值的需求，优先使用 `LongAdder`。

#### 数组类型原子类

| 类 | 操作类型 | 索引方式 | 主要 API |
|----|----------|----------|----------|
| `AtomicIntegerArray` | int[] | 整数索引 | getAndIncrement / compareAndSet |
| `AtomicLongArray` | long[] | 整数索引 | getAndAdd / accumulateAndGet |
| `AtomicReferenceArray<E>` | E[] | 整数索引 | getAndSet / compareAndSet |

> **注意**：数组类型原子类的索引是**整数**，不是线程本地哈希值，与 `LongAdder` 的分段机制完全不同。

#### 引用类型原子类

| 类 | 解决的问题 | 版本机制 | 适用场景 |
|----|------------|----------|----------|
| `AtomicReference<V>` | 引用原子更新 | 无 | 乐观锁、对象替换 |
| `AtomicStampedReference<V>` | ABA + 版本号 | int stamp | 乐观重试、栈/队列 |
| `AtomicMarkableReference<V>` | ABA + 布尔标记 | boolean mark | 节点删除标记 |

> `AtomicMarkableReference` 的标记位只能存 `boolean`，适合\"是否已删除\"等二元状态；`AtomicStampedReference` 存 `int`，适合版本号。

#### 字段更新器

| 类 | 字段类型 | 创建方式 | 限制 |
|----|----------|----------|------|
| `AtomicIntegerFieldUpdater<T>` | volatile int | newUpdater(clazz, "field") | 字段必须是 volatile int |
| `AtomicLongFieldUpdater<T>` | volatile long | newUpdater(clazz, "field") | 字段必须是 volatile long |
| `AtomicReferenceFieldUpdater<T,V>` | volatile V | newUpdater(clazz, vClazz, "field") | 字段必须是 volatile 引用 |

> 字段更新器适合**减少对象数量**的场景，避免为每个字段创建独立的 AtomicXxx 对象。

#### 累加器（JDK8+）

| 类 | 操作类型 | 是否分段 | 适用场景 |
|----|----------|----------|----------|
| `LongAdder` | long 加法 | ✅ 分段 CAS | 高并发计数器、QPS |
| `LongAccumulator` | long 自定义 | ✅ 分段 CAS | max/min/乘积等聚合 |
| `DoubleAdder` | double 加法 | ✅ 分段 CAS | 高精度统计 |
| `DoubleAccumulator` | double 自定义 | ✅ 分段 CAS | 浮点聚合 |

### 选型决策树

```
需要原子操作？
│
├─ 基本类型（int）?
│   └─ AtomicInteger ✅
│
├─ 基本类型（long）?
│   ├─ 高并发计数器？
│   │   └─ LongAdder ✅（性能 ~AtomicLong 的 10 倍）
│   └─ 需要精确返回值？
│       └─ AtomicLong ✅
│
├─ double 类型?
│   ├─ 高并发累加？
│   │   └─ DoubleAdder ✅
│   └─ 自定义运算？
│       └─ DoubleAccumulator ✅
│
├─ 数组元素?
│   ├─ int[] → AtomicIntegerArray
│   ├─ long[] → AtomicLongArray
│   └─ 引用[] → AtomicReferenceArray
│
├─ 对象引用?
│   ├─ 有 ABA 问题？
│   │   ├─ 需要版本号？
│   │   │   └─ AtomicStampedReference ✅
│   │   └─ 只需标记位？
│   │       └─ AtomicMarkableReference ✅
│   └─ 无 ABA 问题？
│       └─ AtomicReference ✅
│
├─ 不想创建新对象？
│   ├─ int/long 字段？
│   │   └─ AtomicInteger/LongFieldUpdater ✅
│   └─ 引用字段？
│       └─ AtomicReferenceFieldUpdater ✅
│
└─ 自定义运算（max/min/乘积）?
    ├─ long → LongAccumulator ✅
    └─ double → DoubleAccumulator ✅
```
