# 生产者-消费者模式

## 核心结论

生产者-消费者模式通过**阻塞队列**解耦生产者和消费者。生产者 put 数据到队列，消费者 take 数据处理。BlockingQueue 自动平衡生产消费速度。

## 深度解析

### 基本实现

```java
BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);

// 生产者
Runnable producer = () -> {
    try {
        for (int i = 0; i < 100; i++) {
            queue.put("item-" + i); // 队列满时自动阻塞
            System.out.println("Produced: item-" + i);
        }
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
};

// 消费者
Runnable consumer = () -> {
    try {
        while (true) {
            String item = queue.take(); // 队列空时自动阻塞
            System.out.println("Consumed: " + item);
            Thread.sleep(100); // 模拟处理耗时
        }
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
};

new Thread(producer).start();
new Thread(consumer).start();
```

### 优势

1. **解耦**：生产者和消费者通过队列交互，不直接依赖
2. **削峰填谷**：队列缓冲突发流量
3. **异步处理**：生产者不需要等待消费者处理完
4. **线程安全**：BlockingQueue 保证线程安全

### 不同队列的适用场景

| 队列 | 适用场景 |
|------|---------|
| ArrayBlockingQueue | 固定缓冲，内存可控 |
| LinkedBlockingQueue | 高吞吐，可配置容量 |
| PriorityBlockingQueue | 按优先级处理任务 |
| DelayQueue | 延迟处理任务 |
| SynchronousQueue | 直接传递，不缓冲 |

### 线程池队列选型深度分析

Java 线程池（ThreadPoolExecutor）的队列选择直接影响系统行为：

```java
// 1. FixedThreadPool - LinkedBlockingQueue（默认无界，OOM风险）
Executors.newFixedThreadPool(10);
// → new ThreadPoolExecutor(10, 10, 0L, MILLISECONDS, new LinkedBlockingQueue<>())
// 风险：任务无限堆积，最终OOM

// 2. CachedThreadPool - SynchronousQueue（零缓冲）
Executors.newCachedThreadPool();
// → new ThreadPoolExecutor(0, Integer.MAX_VALUE, 60L, SECONDS, new SynchronousQueue<>())
// 特点：无任务时0线程，有任务时创建新线程，60秒空闲回收

// 3. SingleThreadExecutor - LinkedBlockingQueue（单线程+无界队列）
Executors.newSingleThreadExecutor();
// → 同FixedThreadPool，但core=max=1

// 4. 自定义：有界队列 + 拒绝策略（推荐生产环境）
new ThreadPoolExecutor(
    4, 8, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100),           // 有界队列，内存可控
    new ThreadPoolExecutor.CallerRunsPolicy() // 队列满时由调用线程执行
);
```

**队列选型决策树**：

```
任务特性？
├── 需要延迟执行 → DelayQueue / ScheduledThreadPoolExecutor
├── 需要优先级 → PriorityBlockingQueue
└── 普通任务
    ├── 内存敏感？
    │   ├── 是 → ArrayBlockingQueue（固定大小）
    │   └── 否 → LinkedBlockingQueue（可配置容量）
    ├── 需要快速响应？
    │   ├── 是 → SynchronousQueue（直接传递，无缓冲）
    │   └── 否 → 有界队列
    └── 生产速度 vs 消费速度？
        ├── 生产 >> 消费 → 有界队列 + 拒绝策略（防止OOM）
        └── 生产 ≈ 消费 → 根据延迟要求选择
```

**各队列在线程池中的表现**：

| 队列 | 队列满时行为 | 适用线程池 | 风险 |
|------|-------------|-----------|------|
| **LinkedBlockingQueue**（无界） | 永不满，无限堆积 | Fixed/Single | OOM |
| **ArrayBlockingQueue**（有界） | 触发拒绝策略 | 自定义 | 任务被拒绝 |
| **SynchronousQueue** | 直接传递，无缓冲 | Cached | 线程数暴涨 |
| **PriorityBlockingQueue** | 无界，按优先级 | 自定义 | OOM |

**生产环境建议**：
- 始终使用**有界队列**（ArrayBlockingQueue 或指定容量的 LinkedBlockingQueue）
- 配置合适的**拒绝策略**（CallerRunsPolicy 或自定义）
- 监控队列大小，设置告警阈值

### 优雅停止

```java
// 使用毒丸（Poison Pill）模式停止消费者
final String POISON = "STOP";
queue.put(POISON); // 生产者结束后放入

// 消费者
String item = queue.take();
if (POISON.equals(item)) {
    return; // 停止消费
}
```

### 多生产者多消费者

```java
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);
ExecutorService producers = Executors.newFixedThreadPool(3);
ExecutorService consumers = Executors.newFixedThreadPool(5);

producers.submit(() -> {
    while (hasWork()) queue.put(produce());
});
consumers.submit(() -> {
    while (true) {
        Task task = queue.take();
        task.execute();
    }
});
```
