# 阻塞队列公平性

## 核心结论

公平性指线程获取锁的顺序是否与等待顺序一致。**公平队列**按 FIFO 顺序唤醒线程，**非公平队列**允许"插队"，吞吐量更高但可能导致线程饥饿。

## 深度解析

### 支持公平性的队列

| 队列 | 公平参数 | 默认 | 说明 |
|------|---------|------|------|
| ReentrantLock | new ReentrantLock(true) | 非公平 | 独占锁 |
| ReentrantReadWriteLock | new ReentrantReadWriteLock(true) | 非公平 | 读写锁 |
| ArrayBlockingQueue | new ArrayBlockingQueue<>(n, true) | 非公平 | 阻塞队列 |
| SynchronousQueue | new SynchronousQueue<>(true) | 非公平 | 同步队列 |

### 公平 vs 非公平原理

```
公平模式（FIFO）：
┌────────────────────────────────────────┐
│  线程A ─┐                              │
│  线程B ─┼──→ 按等待顺序排队 ──→ 依次唤醒 │
│  线程C ─┘                              │
└────────────────────────────────────────┘

非公平模式（LIFO）：
┌────────────────────────────────────────┐
│  线程A 正在持有锁                       │
│  线程B ─┐                              │
│  线程C ─┼──→ 新线程可能"插队"获取锁     │
│  线程D ─┘                              │
└────────────────────────────────────────┘
```

### ReentrantLock 公平实现

```java
// 公平锁 tryAcquire
protected final boolean tryAcquire(int acquires) {
    Thread current = Thread.currentThread();
    int c = getState();
    if (c == 0) {
        // 关键：hasQueuedPredecessors() 检查是否有更早等待的线程
        if (hasQueuedPredecessors()) {
            return false;  // 有更早等待的，让出
        }
        setExclusiveOwnerThread(current);
        setState(acquires);
        return true;
    }
    // ...
}

// 非公平锁 tryAcquire
final boolean nonfairTryAcquire(int acquires) {
    // 没有 hasQueuedPredecessors() 检查
    // 直接 CAS 抢锁，可能"插队"成功
    if (compareAndSetState(0, acquires)) {
        setExclusiveOwnerThread(current);
        return true;
    }
    return false;
}
```

### ArrayBlockingQueue 公平实现

```java
// 公平模式：等待时间最长的线程优先获取锁
public ArrayBlockingQueue(int capacity, boolean fair) {
    // fair=true 时，使用公平锁
    this.lock = new ReentrantLock(fair);
    this.notEmpty = lock.newCondition();
    this.notFull = lock.newCondition();
}

// 公平模式下：
// 1. put 线程等待 notFull
// 2. take 线程等待 notEmpty
// 3. signal() 唤醒时，按等待顺序（FIFO）
```

### SynchronousQueue 公平 vs 非公平

```java
// 公平模式（TransferQueue - FIFO）
new SynchronousQueue<>(true);
// 等待最久的 put/take 优先匹配

// 非公平模式（TransferStack - LIFO，默认）
new SynchronousQueue<>(false);
// 后到的线程可能先匹配（栈顶优先）
```

## 易错点与踩坑

### 1. 公平锁吞吐量低，非公平锁可能导致饥饿
```java
// 公平锁：每次都要检查队列，高并发下性能差 3~5 倍
ReentrantLock fairLock = new ReentrantLock(true);

// 非公平锁：可能有线程永远抢不到锁
ReentrantLock unfairLock = new ReentrantLock(false);

// ✅ 经验：
// - 低并发、要求公平 → 公平锁
// - 高并发、追求吞吐 → 非公平锁（默认）
// - Java 并发包默认都是非公平
```

### 2. ArrayBlockingQueue 公平模式的影响
```java
ArrayBlockingQueue<String> fairQueue = new ArrayBlockingQueue<>(10, true);
ArrayBlockingQueue<String> unfairQueue = new ArrayBlockingQueue<>(10, false);

// 公平队列：线程A put → 线程B put → 线程C take
//           A 先等 → A 先被唤醒 → A put 成功
//           然后 B 才被唤醒

// 非公平队列：线程A put → 线程B put → 线程C take
//             C 可能直接抢锁，先执行 take
//             减少了上下文切换，吞吐量更高
```

### 3. signal vs signalAll 的选择
```java
// signal()：只唤醒一个等待线程
// signalAll()：唤醒所有等待线程

// 大多数情况用 signal()：
// - 只放入一个元素 → 只唤醒一个消费者
// - 只取出一个元素 → 只唤醒一个生产者

// 用 signalAll() 的场景：
// - 批量操作后，可能有多个线程需要被唤醒
// - 条件复杂，一个 signal 可能唤醒错误的线程
```

### 4. 虚假唤醒（Spurious Wakeup）
```java
// JDK Condition.await() 文档说明：可能出现虚假唤醒
// 即：没有调用 signal()，线程也可能被唤醒

// ❌ 错误写法
while (count == 0) {  // 应该用 while，不是 if
    notEmpty.await();
}

// ✅ 正确写法
if (count == 0) {     // 错误：虚假唤醒时会继续执行
    notEmpty.await();
}

while (count == 0) {   // 正确：醒来后重新检查条件
    notEmpty.await();
}
```

### 5. 饥饿（Starvation）问题
```java
// 非公平锁 + 高并发 → 可能饥饿
ReentrantLock lock = new ReentrantLock();

// 线程A 持续持有锁（执行耗时操作）
// 线程B、C、D 依次等待
// 新线程E 不断"插队"
// B、C、D 可能永远得不到锁

// ✅ 解决方案：
// 1. 换成公平锁
// 2. 控制并发度，减少新线程抢锁
// 3. 使用锁超时 tryLock()
if (lock.tryLock(1, TimeUnit.SECONDS)) {
    try {
        // 业务逻辑
    } finally {
        lock.unlock();
    }
}
```

### 6. 公平锁在生产者-消费者中的表现
```java
// 公平模式：按提交顺序处理
ArrayBlockingQueue<Task> queue = new ArrayBlockingQueue<>(100, true);

// 线程A 提交任务1 → 线程B 提交任务2 → 线程C 提交任务3
// 任务1 一定先被消费（即使任务3先完成）

// 非公平模式：不保证顺序
// 可能任务3 先被消费，任务1 后消费
// 但吞吐量更高
```

## 选型建议

| 场景 | 推荐 | 理由 |
|------|------|------|
| 高并发 API | 非公平 | 吞吐量优先 |
| 金融交易 | 公平 | 保证顺序，防止套利 |
| 任务调度 | 公平 | 按优先级处理 |
| 生产者-消费者 | 非公平 | 吞吐量优先 |
| 短临界区 | 非公平 | 持有时间短，饥饿不严重 |
| 长临界区 | 公平 | 避免饥饿 |
