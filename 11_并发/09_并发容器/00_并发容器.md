## 四、容器对比总表

### Map族

| 容器 | 底层 | 并发度 | 有序 | null支持 | 适用场景 |
|------|------|--------|------|---------|---------|
| **ConcurrentHashMap** | Node数组+红黑树 | 高（桶锁） | ❌ | key/value都不允许 | 高并发Map（最高频） |
| **ConcurrentSkipListMap** | 跳表 | 高（CAS） | ✅ | key不允许，value不允许 | 高并发有序Map |
| Collections.synchronizedMap | HashMap+锁 | 低（单锁） | ❌ | key/value都允许 | 低并发过渡期 |

### List族

| 容器 | 底层 | 读性能 | 写性能 | 迭代器 | 适用场景 |
|------|------|--------|--------|--------|---------|
| **CopyOnWriteArrayList** | 数组 | O(1)无锁 | O(n)复制 | 快照（弱一致） | 读多写少 |
| **ConcurrentLinkedQueue** | 链表 | O(1)无锁 | O(1)CAS | 无锁遍历 | 高并发队列 |
| Collections.synchronizedList | 数组+锁 | O(1)+锁 | O(1)+锁 | 需手动同步 | 低并发过渡期 |

### Set族

| 容器 | 底层 | add | contains | 有序 | 适用场景 |
|------|------|-----|----------|------|---------|
| **ConcurrentSkipListSet** | 跳表 | O(log n) | O(log n) | ✅ | 大数据量有序Set |
| **CopyOnWriteArraySet** | 数组 | O(n) | O(n) | ❌ | 小数据量读多写少 |
| Collections.synchronizedSet | HashSet+锁 | O(1)+锁 | O(1)+锁 | ❌ | 低并发过渡期 |

## 六、易错点章节覆盖

| 模块 | 易错点数量 | 核心踩坑点 |
|------|:----------:|-----------|
| ConcurrentHashMap | 5个 | computeIfAbsent死锁陷阱、size vs mappingCount、null禁止 |
| CopyOnWriteArrayList | 4个 | 写操作O(n)、迭代器remove抛异常、内存翻倍OOM |
| CopyOnWriteArraySet | 3个 | addIfAbsent双重检查O(n)、contains是O(n)、无序误以为有序 |
| ConcurrentLinkedQueue | 4个 | offer永远true但OOM、poll null二义性、size非精确 |
| ConcurrentLinkedDeque | 3个 | 双指针CAS竞争、工作窃取陷阱、栈操作非原子 |
| ConcurrentSkipListMap | 3个 | 跳表层数随机性、逻辑删除不释放内存、子视图弱一致 |
| ConcurrentSkipListSet | 3个 | vs COWAS性能选择、视图影响原集合、迭代器弱一致 |
| Collections.synchronizedXXX | 4个 | 复合操作需外部同步、迭代必须加锁、mutex死锁风险 |

## 八、选型决策树

```
需要并发容器？
│
├─ 需要Map？
│   │
│   ├─ 高并发 + 无需有序 → ConcurrentHashMap ✅
│   ├─ 高并发 + 需要有序 → ConcurrentSkipListMap
│   └─ 低并发 + 过渡期 → Collections.synchronizedMap
│
├─ 需要List？
│   │
│   ├─ 读多写少 + 小数据量 → CopyOnWriteArrayList ✅
│   ├─ 高并发队列 → ConcurrentLinkedQueue
│   └─ 低并发 + 过渡期 → Collections.synchronizedList
│
├─ 需要Set？
│   │
│   ├─ 大数据量 + 有序 → ConcurrentSkipListSet ✅
│   ├─ 小数据量 + 读多写少 → CopyOnWriteArraySet
│   └─ 低并发 + 过渡期 → Collections.synchronizedSet
│
└─ 需要Deque？
    │
    ├─ 非阻塞 + 双端 → ConcurrentLinkedDeque
    ├─ 阻塞 + 双端 → LinkedBlockingDeque
    └─ 低并发 + 单端 → Collections.synchronizedList
```
