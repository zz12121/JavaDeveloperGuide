
# 并发任务编排

## 核心结论

并发任务编排是协调多个异步任务的执行顺序和依赖关系。常用工具：`CompletableFuture`（异步链式编排）、`CountDownLatch`（等待多个任务完成）、`ForkJoinPool`（分治并行）。

## 深度解析

### CompletableFuture 编排

```java
// 场景：查用户信息 + 查订单信息 + 查推荐，三个独立任务并行
CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(() -> userService.get(id));
CompletableFuture<List<Order>> orderFuture = CompletableFuture.supplyAsync(() -> orderService.list(id));
CompletableFuture<List<Product>> recommendFuture = CompletableFuture.supplyAsync(() -> recommendService.get(id));

// 三个都完成后合并
CompletableFuture<Result> result = CompletableFuture.allOf(userFuture, orderFuture, recommendFuture)
    .thenApply(v -> new Result(userFuture.join(), orderFuture.join(), recommendFuture.join()));

Result r = result.get(3, TimeUnit.SECONDS);
```

### CountDownLatch 等待多个任务

```java
CountDownLatch latch = new CountDownLatch(3);

// 并行加载三个模块
executor.submit(() -> { loadConfig(); latch.countDown(); });
executor.submit(() -> { loadCache(); latch.countDown(); });
executor.submit(() -> { loadRoute(); latch.countDown(); });

latch.await(10, TimeUnit.SECONDS); // 等待全部完成
// 开始提供服务
```

### 任务依赖关系

```
串行：A → B → C
  thenApply / thenCompose

并行：A + B + C → 合并
  allOf / thenCombine

条件：A 成功 → B，A 失败 → C
  exceptionally / handle

任一完成：A | B | C → 结果
  anyOf
```

### thenApply vs thenCompose（高频考点）

| 方法 | 函数签名 | 作用 | 适用场景 |
|------|---------|------|---------|
| `thenApply` | `T → U` | 同步转换结果，返回 `CompletableFuture<U>` | 结果变换，不涉及异步调用 |
| `thenCompose` | `T → CompletableFuture<U>` | 扁平化，避免嵌套 `CF<CF<U>>` | 下一步本身也是异步操作 |

```java
// thenApply：结果变换（非异步调用）
CompletableFuture<String> name = CompletableFuture
    .supplyAsync(() -> 1L)          // CF<Long>
    .thenApply(id -> "user_" + id); // CF<String>（直接变换）

// thenCompose：异步调用链（避免嵌套）
CompletableFuture<Order> order = CompletableFuture
    .supplyAsync(() -> getUserId())         // CF<Long>
    .thenCompose(id -> getOrderAsync(id));  // 扁平成 CF<Order>，而非 CF<CF<Order>>
```

### thenCombine：合并两个独立 Future 的结果

```java
// 场景：同时查询商品价格和库存，合并计算
CompletableFuture<Double> priceFuture = CompletableFuture.supplyAsync(() -> getPrice(skuId));
CompletableFuture<Integer> stockFuture = CompletableFuture.supplyAsync(() -> getStock(skuId));

CompletableFuture<String> result = priceFuture.thenCombine(
    stockFuture,
    (price, stock) -> String.format("价格:%.2f 库存:%d", price, stock)
);
// priceFuture 和 stockFuture 并行执行，都完成后合并

System.out.println(result.get()); // 价格:99.00 库存:500
```

**thenCombine vs allOf 区别**：
- `thenCombine`：合并**两个** Future 并能直接获取各自结果
- `allOf`：等待**N个** Future，但不直接提供结果，需要手动 `.join()`

### 异常处理三件套

```java
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    if (Math.random() > 0.5) throw new RuntimeException("查询失败");
    return "OK";
});

// 方式1：exceptionally — 只处理异常，正常值直接透传
future.exceptionally(ex -> {
    log.error("异常", ex);
    return "默认值";   // 异常时返回兜底值
});

// 方式2：handle — 无论正常/异常都执行，可同时处理两种情况
future.handle((result, ex) -> {
    if (ex != null) return "默认值";
    return result.toUpperCase();  // 可对正常结果做变换
});

// 方式3：whenComplete — 副作用，不改变结果
future.whenComplete((result, ex) -> {
    if (ex != null) log.error("异常", ex);
    else log.info("结果: {}", result);
    // 返回值无效，结果还是原来的 result 或 ex
});
```

| 方法 | 接收正常值 | 接收异常 | 能改变返回值 | 典型用途 |
|------|-----------|---------|------------|---------|
| `exceptionally` | ❌（透传） | ✅ | ✅（兜底值） | 异常降级 |
| `handle` | ✅ | ✅ | ✅ | 统一处理+变换 |
| `whenComplete` | ✅ | ✅ | ❌ | 日志/监控副作用 |

### 超时控制

```java
// orTimeout（Java 9+）：超时后抛出 TimeoutException
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> slowQuery())
    .orTimeout(3, TimeUnit.SECONDS);   // 3秒超时直接失败

// completeOnTimeout（Java 9+）：超时后以默认值完成，不抛异常
CompletableFuture<String> future2 = CompletableFuture
    .supplyAsync(() -> slowQuery())
    .completeOnTimeout("超时默认值", 3, TimeUnit.SECONDS);

// Java 8 方案：get(timeout)
try {
    String result = future.get(3, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    future.cancel(true);
}
```

### thenApply vs thenApplyAsync（线程切换）

```java
// thenApply：在完成上一个任务的线程中执行（或调用线程）
future.thenApply(r -> transform(r));    // 可能用 ForkJoin 公共池线程

// thenApplyAsync：提交到新线程池执行
future.thenApplyAsync(r -> transform(r), customExecutor);  // 指定线程池

// 规律：带 Async 后缀的方法都是异步提交到新线程
// 不带 Async：在触发线程或上游线程中直接执行（可能阻塞）
```

### 自定义线程池（生产必备）

```java
// ❌ 危险：默认使用 ForkJoinPool.commonPool()，可能被耗尽
CompletableFuture.supplyAsync(() -> callRemoteService());

// ✅ 生产推荐：指定独立线程池，隔离资源
private static final ExecutorService IO_POOL = new ThreadPoolExecutor(
    10, 50, 60, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(500),
    new ThreadPoolExecutor.CallerRunsPolicy()
);

CompletableFuture.supplyAsync(() -> callRemoteService(), IO_POOL)
    .thenApplyAsync(result -> process(result), IO_POOL)
    .exceptionally(ex -> fallback());
```

### 方案选择

| 需求 | 工具 |
|------|------|
| 链式异步 | CompletableFuture |
| 等待 N 个任务完成 | CountDownLatch / allOf |
| 多线程相互等待 | CyclicBarrier |
| 分治递归 | ForkJoinPool |
| 并行流 | parallelStream |
| A→B 串行，B依赖A结果 | thenCompose |
| A+B 并行，合并结果 | thenCombine |
| 超时降级 | completeOnTimeout |
