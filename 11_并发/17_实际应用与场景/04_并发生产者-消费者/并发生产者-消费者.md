
# 并发生产者-消费者

## 核心结论

生产者-消费者模式通过**阻塞队列**解耦生产者和消费者。生产者放入队列，消费者从队列取出，队列满时生产者阻塞，队列空时消费者阻塞。

## 深度解析

### 标准实现

```java
public class ProducerConsumerDemo {
    private final BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);

    // 生产者
    public void startProducer() {
        new Thread(() -> {
            try {
                for (int i = 0; i < 100; i++) {
                    queue.put("item-" + i);   // 队列满时阻塞
                    System.out.println("生产: item-" + i);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }).start();
    }

    // 消费者
    public void startConsumer() {
        new Thread(() -> {
            try {
                while (true) {
                    String item = queue.take(); // 队列空时阻塞
                    System.out.println("消费: " + item);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }).start();
    }
}
```

### 多生产者-多消费者

```java
ExecutorService producers = Executors.newFixedThreadPool(3);
ExecutorService consumers = Executors.newFixedThreadPool(2);
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);

// 多个生产者
for (int i = 0; i < 3; i++) {
    producers.submit(() -> {
        while (running) {
            Task task = generateTask();
            queue.put(task); // 满时阻塞
        }
    });
}

// 多个消费者
for (int i = 0; i < 2; i++) {
    consumers.submit(() -> {
        while (running) {
            Task task = queue.take(); // 空时阻塞
            process(task);
        }
    });
}
```

### 队列选择

| 队列 | 特点 | 适用场景 |
|------|------|---------|
| `ArrayBlockingQueue` | 有界、数组、FIFO | 内存有限 |
| `LinkedBlockingQueue` | 可选有界、链表 | 吞吐量优先 |
| `PriorityBlockingQueue` | 优先级排序 | 任务有优先级 |
| `SynchronousQueue` | 零容量、直接传递 | 生产消费速度匹配 |
| `DelayQueue` | 延迟取出 | 定时任务、超时处理 |

### 优雅停止

```java
// 使用毒丸（Poison Pill）模式停止消费者
queue.put(POISON_PILL); // 生产者结束前放入

// 消费者判断
while (true) {
    Task task = queue.take();
    if (task == POISON_PILL) break; // 收到毒丸，退出
    process(task);
}
```

### 线程池内置的生产者-消费者

`ThreadPoolExecutor` 本身就是生产者-消费者模式：调用者 = 生产者，工作线程 = 消费者，`workQueue` = 阻塞队列。

### TransferQueue：直接移交模式

`LinkedTransferQueue` 实现了 `TransferQueue` 接口，是 `SynchronousQueue` 的增强版：

```java
LinkedTransferQueue<Task> queue = new LinkedTransferQueue<>();

// transfer：直接移交给等待的消费者线程（无消费者时阻塞）
queue.transfer(task);    // 等价于 SynchronousQueue.put

// tryTransfer：立即尝试移交（无消费者立即返回 false）
boolean handed = queue.tryTransfer(task);

// tryTransfer(timeout)：等待指定时间
boolean handed = queue.tryTransfer(task, 100, TimeUnit.MILLISECONDS);

// put：也可以当普通队列用（无消费者时入队等待）
queue.put(task);         // 比 SynchronousQueue 更灵活
```

**TransferQueue vs SynchronousQueue**：
- `SynchronousQueue`：必须生产消费同步，容量 0，只能直接移交
- `LinkedTransferQueue`：既支持直接移交（transfer），也支持缓冲（put），更灵活

**适用场景**：生产者想确认任务被消费者接收时（如异步请求-响应模式）。

### 高性能场景：Disruptor

当 BlockingQueue 的吞吐量无法满足需求时，考虑 Disruptor（详见 10_Disruptor章节）：

| 维度 | BlockingQueue | Disruptor |
|------|--------------|-----------|
| 并发控制 | Lock/CAS | 无锁（序号机制） |
| 内存分配 | 运行时创建 | 预分配 Ring Buffer |
| GC 压力 | 频繁创建对象 | 几乎无 GC |
| 适用场景 | 通用场景 | 金融交易/日志框架 |
