
# 高并发初始化

## 核心结论

高并发初始化的核心是**等待多个资源加载完成后再启动服务**。常用方案：`CountDownLatch`（一次性等待）、`CyclicBarrier`（多阶段同步）、`CompletableFuture`（异步编排）。

## 深度解析

### 方案一：CountDownLatch（推荐）

```java
public class ApplicationInitializer {
    private final CountDownLatch initLatch = new CountDownLatch(4);

    public void start() {
        ExecutorService executor = Executors.newFixedThreadPool(4);
        
        // 并行加载四个模块
        executor.submit(() -> { loadDatabase(); initLatch.countDown(); });
        executor.submit(() -> { loadCache(); initLatch.countDown(); });
        executor.submit(() -> { loadConfig(); initLatch.countDown(); });
        executor.submit(() -> { loadRoute(); initLatch.countDown(); });

        // 等待全部完成
        try {
            initLatch.await(30, TimeUnit.SECONDS);
            System.out.println("所有模块初始化完成，服务启动");
        } catch (InterruptedException e) {
            System.err.println("初始化超时");
        }
    }
}
```

### 方案二：CompletableFuture（更灵活）

```java
public void start() {
    CompletableFuture<Void> db = CompletableFuture.runAsync(this::loadDatabase);
    CompletableFuture<Void> cache = CompletableFuture.runAsync(this::loadCache);
    CompletableFuture<Void> config = CompletableFuture.runAsync(this::loadConfig);
    CompletableFuture<Void> route = CompletableFuture.runAsync(this::loadRoute);

    CompletableFuture.allOf(db, cache, config, route)
        .orTimeout(30, TimeUnit.SECONDS)
        .whenComplete((v, ex) -> {
            if (ex != null) {
                System.err.println("初始化失败: " + ex.getMessage());
            } else {
                System.out.println("初始化完成");
            }
        });
}
```

### 方案三：CyclicBarrier（多阶段初始化）

```java
// 分阶段初始化：先加载基础资源 → 再初始化业务模块
CyclicBarrier phase1 = new CyclicBarrier(3, () -> System.out.println("阶段1完成"));
CyclicBarrier phase2 = new CyclicBarrier(3, () -> System.out.println("阶段2完成"));

executor.submit(() -> {
    loadDB();
    phase1.await(); // 等待其他基础资源
    initUserService();
    phase2.await(); // 等待其他业务模块
});
```

### 方案对比

| 方案 | 可复用 | 异常处理 | 链式组合 | 超时 |
|------|--------|---------|---------|------|
| CountDownLatch | ❌ 一次性 | 子线程内处理 | ❌ | await(timeout) |
| CyclicBarrier | ✅ 可复用 | 可选 barrierAction | ❌ | await(timeout) |
| CompletableFuture | ❌ 一次性 | exceptionally/handle | ✅ | orTimeout |

## 单例对象的高并发安全初始化

> 与上面的"多资源等待加载"不同，这里关注的是**单例对象只初始化一次**的并发安全问题。

### DCL 双重检查锁（Double-Checked Locking）

```java
public class Singleton {
    // 必须加 volatile！！
    private static volatile Singleton instance;

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {                    // 第一次检查（无锁，提升性能）
            synchronized (Singleton.class) {
                if (instance == null) {            // 第二次检查（防止重复初始化）
                    instance = new Singleton();    // 问题所在
                }
            }
        }
        return instance;
    }
}
```

**为什么必须加 `volatile`？**

`instance = new Singleton()` 实际被编译为3步：
```
1. 分配内存空间
2. 初始化对象（调用构造函数）
3. 将引用 instance 指向内存地址
```

没有 `volatile` 时，JIT 编译器/CPU 可能将步骤 2 和 3 重排序为 **1→3→2**：

```
线程A：分配内存 → 赋值引用（instance 非null，但对象未初始化！）
线程B：第一次检查 instance != null → 直接返回 → 使用未初始化的对象 → NPE/崩溃
```

`volatile` 的写屏障（StoreStore + StoreLoad）禁止此重排序，保证 1→2→3 顺序。

### 静态内部类（Holder 模式，推荐）

```java
public class Singleton {
    private Singleton() {}

    // 静态内部类：只在第一次调用 getInstance() 时才加载
    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}
```

**为什么线程安全？**

- JVM 保证类加载过程中 `<clinit>` 方法是线程安全的（由 ClassLoader 加锁）
- `Holder` 类只在 `getInstance()` 第一次被调用时才加载和初始化
- 既实现了**懒加载**，又不需要 `synchronized` 或 `volatile`

**推荐原因**：实现最简洁，无性能损耗，JVM 层面保证安全。

### 枚举单例（最安全）

```java
public enum Singleton {
    INSTANCE;

    public void doSomething() {
        System.out.println("单例方法调用");
    }
}

// 使用
Singleton.INSTANCE.doSomething();
```

**为什么最安全？**
- 枚举实例由 JVM 保证只创建一次（类加载机制）
- **防反射攻击**：`Constructor.newInstance()` 对枚举抛 `IllegalArgumentException`
- **防序列化攻击**：枚举序列化/反序列化自动复用同一实例（JVM 内置保证）

| 方式 | 懒加载 | 线程安全 | 防反射 | 防序列化 | 推荐度 |
|------|--------|---------|--------|---------|--------|
| 饿汉式 | ❌ | ✅ | ❌ | ❌ | 一般 |
| DCL | ✅ | ✅（volatile） | ❌ | ❌ | 良好 |
| 静态内部类 | ✅ | ✅ | ❌ | ❌ | **推荐** |
| 枚举 | ❌ | ✅ | ✅ | ✅ | **最安全** |
