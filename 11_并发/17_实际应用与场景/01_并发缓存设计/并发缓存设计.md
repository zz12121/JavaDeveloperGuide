
# 并发缓存设计

## 核心结论

并发缓存设计的核心是**保证线程安全**的同时实现**缓存一致性**。常用方案：`ConcurrentHashMap` + `computeIfAbsent` 原子计算、`Guava Cache` / `Caffeine` 高级缓存框架、二级缓存 + 过期策略。

## 深度解析

### 方案一：ConcurrentHashMap + computeIfAbsent（推荐）

```java
public class ConcurrentCache<K, V> {
    private final ConcurrentHashMap<K, V> cache = new ConcurrentHashMap<>();

    public V get(K key, Function<K, V> loader) {
        return cache.computeIfAbsent(key, loader);
    }
}

// 使用
ConcurrentCache<String, User> cache = new ConcurrentCache<>();
User user = cache.get("user:1", k -> userDao.findById(1));
```

**优点**：`computeIfAbsent` 原子操作，同一 key 只计算一次，天然防止缓存击穿。

### 方案二：防缓存击穿的细粒度锁

```java
public class FineGrainedCache<K, V> {
    private final ConcurrentHashMap<K, V> cache = new ConcurrentHashMap<>();
    private final ConcurrentHashMap<K, ReentrantLock> locks = new ConcurrentHashMap<>();

    public V get(K key, Function<K, V> loader) {
        V value = cache.get(key);
        if (value != null) return value;

        ReentrantLock lock = locks.computeIfAbsent(key, k -> new ReentrantLock());
        lock.lock();
        try {
            value = cache.get(key); // 双重检查
            if (value == null) {
                value = loader.apply(key);
                cache.put(key, value);
            }
            return value;
        } finally {
            lock.unlock();
            locks.remove(key);
        }
    }
}
```

### 方案三：带过期时间的缓存

```java
public class ExpiringCache<K, V> {
    private final ConcurrentHashMap<K, CacheEntry<V>> cache = new ConcurrentHashMap<>();
    private final long expireAfterMillis;

    record CacheEntry<V>(V value, long createTime) {}

    public V get(K key, Function<K, V> loader) {
        CacheEntry<V> entry = cache.get(key);
        long now = System.currentTimeMillis();
        if (entry != null && now - entry.createTime() < expireAfterMillis) {
            return entry.value();
        }
        return cache.computeIfAbsent(key, k -> 
            new CacheEntry<>(loader.apply(k), now)
        ).value();
    }
}
```

### 常见缓存问题与应对

| 问题 | 定义 | 解决方案 |
|------|------|---------|
| 缓存穿透 | 查询不存在的 key | 布隆过滤器 / 缓存空值 |
| 缓存击穿 | 热点 key 过期瞬间大量请求 | `computeIfAbsent` / 互斥锁 |
| 缓存雪崩 | 大量 key 同时过期 | 随机过期时间 / 多级缓存 |

### 成熟框架推荐

| 框架 | 特点 |
|------|------|
| **Caffeine** | 高性能本地缓存，W-TinyLFU 淘汰算法 |
| **Guava Cache** | 老牌稳定，功能全面 |
| **Redis + 本地缓存** | 二级缓存，一致性用 Canal / 消息队列 |

### 方案四：CAS 方式更新缓存值

```java
// 场景：多线程并发更新同一 key 的缓存值
ConcurrentHashMap<String, AtomicReference<String>> cache = new ConcurrentHashMap<>();

public boolean updateCache(String key, String oldValue, String newValue) {
    AtomicReference<String> ref = cache.computeIfAbsent(key, k -> new AtomicReference<>());
    // CAS 更新：只有当前值等于 oldValue 时才更新
    return ref.compareAndSet(oldValue, newValue);
}

// 或者直接用 ConcurrentHashMap.replace（语义相同）
cache.replace(key, oldValue, newValue); // 返回 true 表示替换成功
```

**场景**：多个线程同时更新配置缓存，避免后写覆盖先写的最新值。

### 方案五：热点 Key 处理

热点 Key 问题：少数 Key 承受极高并发，单点 Redis 实例压力过大或本地缓存失效风暴。

```java
// 方案1：本地缓存 + Redis 二级缓存
public class HotKeyCache<K, V> {
    // 一级：本地 Caffeine 缓存（极快，无网络开销）
    private final Cache<K, V> localCache = Caffeine.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(5, TimeUnit.SECONDS)  // 本地缓存5秒，较短
        .build();
    
    // 二级：Redis（分布式共享）
    private final RedisTemplate<K, V> redis;

    public V get(K key) {
        // 先查本地缓存
        V value = localCache.getIfPresent(key);
        if (value != null) return value;
        
        // 再查 Redis
        value = redis.opsForValue().get(key);
        if (value != null) {
            localCache.put(key, value); // 回填本地
            return value;
        }
        
        // 最后查 DB（加锁防击穿）
        return loadFromDB(key);
    }
}

// 方案2：Key 分片（将热点 key 拆成多个子 key，分散到不同 slot）
String hotKey = "product:hot:1001";
int shardIndex = ThreadLocalRandom.current().nextInt(10);
String shardKey = hotKey + ":shard:" + shardIndex;  // 10个分片分散压力
V value = redis.get(shardKey);
// 写入时同时写 10 个分片，读取时随机读一个
```
