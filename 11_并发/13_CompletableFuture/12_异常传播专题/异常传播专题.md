# CompletableFuture 异常传播专题

## 核心结论

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        CompletableFuture 异常传播机制                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   supplyAsync(抛异常)                                                         │
│          ↓                                                                   │
│   ┌───────┴───────┐                                                          │
│   ↓               ↓                                                          │
│ thenApply      thenAccept                                                    │
│ (跳过)          (跳过)                                                        │
│   ↓               ↓                                                          │
│ thenRun        thenCompose                                                   │
│ (跳过)          (跳过)                                                        │
│   ↓               ↓                                                          │
│   └───────┬───────┘                                                          │
│           ↓                                                                  │
│   exceptionally / handle / whenComplete                                       │
│           ↓                                                                  │
│   ┌───────┴───────┐                                                          │
│   ↓               ↓                                                          │
│ 捕获成功        捕获失败                                                       │
│ 返回值/正常      函数抛异常                                                     │
│           ↓                                                                  │
│     替换原始异常                                                                │
│           ↓                                                                  │
│   后续链继续传播新异常                                                          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 传播路径详解

### 1. supplyAsync → thenApply 异常传播

```java
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> {
        // 抛出异常
        throw new RuntimeException("supplyAsync 异常");
    })
    .thenApply(s -> {
        // ❌ 不执行：异常跳过了 thenApply
        System.out.println("不会执行");
        return s.toUpperCase();
    })
    .thenApply(s -> {
        // ❌ 不执行
        return s + "!";
    });

try {
    cf.join();
} catch (CompletionException e) {
    System.out.println(e.getCause()); // RuntimeException("supplyAsync 异常")
}
```

### 2. exceptionally 捕获后恢复正常

```java
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> { throw new RuntimeException("A"); })
    .thenApply(s -> s + " -> B")  // 跳过
    .exceptionally(ex -> {
        // ✅ 执行：捕获异常
        System.out.println("捕获: " + ex.getMessage()); // "A"
        return "recovered"; // 返回恢复值
    })
    .thenApply(s -> s + " -> C"); // ✅ 执行：恢复正常

cf.join(); // "recovered -> C"
```

### 3. exceptionally 本身抛异常

```java
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> { throw new RuntimeException("原始异常"); })
    .exceptionally(ex -> {
        // ✅ 执行：捕获异常
        throw new RuntimeException("exceptionally 中抛异常");
    })
    .exceptionally(ex -> {
        // ✅ 执行：捕获新的异常
        System.out.println("捕获到: " + ex.getCause().getMessage());
        // 输出：exceptionally 中抛异常
        return "fallback";
    });

cf.join(); // "fallback"
```

### 4. handle 的异常处理

```java
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> { throw new RuntimeException("原始"); })
    .handle((result, ex) -> {
        // ✅ 执行：handle 无论成功失败都触发
        if (ex != null) {
            System.out.println("handle 收到异常: " + ex.getMessage());
            return "handled"; // 返回值替换原结果
        }
        return result;
    })
    .thenApply(s -> s + " processed");

cf.join(); // "handled processed"
```

### 5. handle 本身抛异常

```java
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> { throw new RuntimeException("原始"); })
    .handle((result, ex) -> {
        if (ex != null) {
            // ⚠️ handle 的 function 抛异常 → 替换原异常
            throw new IllegalStateException("handle 中抛异常");
        }
        return result;
    });

try {
    cf.join();
} catch (CompletionException e) {
    System.out.println(e.getCause());
    // IllegalStateException("handle 中抛异常")
    // ⚠️ 原始的 RuntimeException("原始") 丢失了！
}
```

### 6. whenComplete 的异常覆盖

```java
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> { throw new RuntimeException("原始"); })
    .whenComplete((result, ex) -> {
        if (ex != null) {
            // ⚠️ whenComplete 中抛异常 → 覆盖原异常
            throw new RuntimeException("whenComplete 中抛异常");
        }
    });

try {
    cf.join();
} catch (CompletionException e) {
    System.out.println(e.getCause());
    // RuntimeException("whenComplete 中抛异常")
    // ⚠️ 原始异常丢失！
}
```

## 异常包装机制

### CompletionException 包装规则

```java
// 所有 join()/get() 抛出的异常都是 CompletionException
// 原始异常被包装在其 cause 中

supplyAsync(() -> { throw new RuntimeException("原始"); })
    .join();

// 实际抛出
// CompletionException(RuntimeException("原始"))

// 正确获取原始异常
try {
    cf.join();
} catch (CompletionException e) {
    Throwable cause = e.getCause(); // RuntimeException("原始")
}
```

### 多层包装

```java
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> { throw new RuntimeException("level 1"); })
    .thenApply(s -> { throw new RuntimeException("level 2"); })
    .exceptionally(ex -> {
        throw new RuntimeException("level 3");
    });

try {
    cf.join();
} catch (CompletionException e) {
    // e.getCause() → RuntimeException("level 3")
    // e.getCause().getCause() → RuntimeException("level 2")
    // e.getCause().getCause().getCause() → RuntimeException("level 1")

    // 展开所有层级
    Throwable current = e;
    while (current != null) {
        System.out.println(current.getClass().getSimpleName() + ": " +
            current.getMessage());
        current = current.getCause();
    }
    // 输出:
    // CompletionException: null
    // RuntimeException: level 3
    // RuntimeException: level 2
    // RuntimeException: level 1
}
```

## 异常传播状态图

```
┌────────────────────────────────────────────────────────────────────────────┐
│                           异常传播状态机                                      │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  ┌──────────┐      抛出异常        ┌──────────────┐                         │
│  │   NULL   │ ─────────────────▶  │ EXCEPTIONAL  │                         │
│  │ (正常结果) │                    │   (异常状态)   │                         │
│  └──────────┘                     └──────┬───────┘                         │
│       ▲                                  │                                  │
│       │                                  │ exceptionally/ex                 │
│       │                                  ▼                                  │
│       │                           ┌──────────────┐                          │
│       │                           │ RECOVERED   │                          │
│       │                           │ (恢复状态)   │                          │
│       │                           └──────┬───────┘                          │
│       │                                  │                                  │
│       │                                  │ exceptionally 抛异常              │
│       ▼                                  ▼                                  │
│  ┌──────────┐                     ┌──────────────┐                          │
│  │ COMPLETE │                     │ EXCEPTIONAL  │                          │
│  │ (正常完成) │                     │ (新异常替换)  │                          │
│  └──────────┘                     └──────────────┘                          │
│                                                                            │
│  ┌──────────────────────────────────────────────────────────────────────┐   │
│  │ thenApply/thenAccept/thenRun/thenCompose:                             │   │
│  │ 遇到 EXCEPTIONAL 状态 → 跳过当前阶段 → 继续传播异常                     │   │
│  │                                                                       │   │
│  │ exceptionally/handle/whenComplete:                                    │   │
│  │ 捕获 EXCEPTIONAL → 执行处理函数 →                                        │   │
│  │   • 返回正常值 → 转换为 RECOVERED → 后续链正常                           │   │
│  │   • 抛异常 → 转换为新 EXCEPTIONAL → 后续链传播新异常                     │   │
│  └──────────────────────────────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────────────────────────────┘
```

## 实际应用示例

### 分层异常处理

```java
CompletableFuture<Order> orderFuture = orderService.getOrderAsync(orderId)
    // 第1层：数据查询异常
    .exceptionally(ex -> {
        if (ex instanceof DataAccessException) {
            return getOrderFromBackup(orderId); // 查备份
        }
        throw new CompletionException(ex); // 继续传播
    })
    // 第2层：业务逻辑异常
    .handle((order, ex) -> {
        if (ex != null) {
            log.error("获取订单失败", ex);
            return getDefaultOrder(orderId); // 返回默认订单
        }
        return order;
    })
    // 第3层：监控记录
    .whenComplete((order, ex) -> {
        if (ex != null) {
            metrics.increment("order.fetch.error",
                Tags.of("type", ex.getClass().getSimpleName()));
        } else {
            metrics.increment("order.fetch.success");
        }
    });
```

### 异常传播控制

```java
// 场景：需要保留原始异常信息
CompletableFuture<String> cf = CompletableFuture
    .supplyAsync(() -> { throw new BusinessException("CODE_001", "业务错误"); })
    .exceptionally(ex -> {
        // ⚠️ 必须保留原始异常
        if (ex instanceof BusinessException) {
            throw (BusinessException) ex; // 强转后重新抛出
        }
        throw new BusinessException("UNKNOWN", ex.getMessage());
    })
    .handle((result, ex) -> {
        if (ex != null) {
            // 确保异常信息完整
            if (ex instanceof BusinessException) {
                BusinessException be = (BusinessException) ex;
                return "错误码: " + be.getCode() + ", 消息: " + be.getMessage();
            }
            return "未知错误: " + ex.getMessage();
        }
        return result;
    });
```
