
# 超时处理

## 核心结论

JDK9+ 为 CompletableFuture 引入了**原生超时机制**：

| 方法 | 版本 | 行为 |
|------|------|------|
| `orTimeout(duration, unit)` | JDK9+ | 超时后**异常完成**（TimeoutException） |
| `completeOnTimeout(value, duration, unit)` | JDK9+ | 超时后**正常完成**（返回默认值） |
| `future.get(timeout, unit)` | JDK8 | 阻塞等待，超时抛受检异常 |

## 深度解析

### orTimeout — 超时抛异常

```java
CompletableFuture<String> future = supplyAsync(() -> {
    sleep(5000);
    return "result";
}).orTimeout(3, TimeUnit.SECONDS);

future.join(); // 抛 TimeoutException
```

**特点**：
- 超时后 future **异常完成**，结果为 `TimeoutException`
- **不会取消**底层任务，异步线程继续执行（浪费资源）
- 超时是相对于**当前调用时刻**计算

### completeOnTimeout — 超时返回默认值

```java
CompletableFuture<String> future = supplyAsync(() -> {
    sleep(5000);
    return "result";
}).completeOnTimeout("默认值", 3, TimeUnit.SECONDS);

future.join(); // "默认值"，不抛异常
```

**特点**：
- 超时后 future **正常完成**，结果为指定默认值
- 更适合**降级场景**

### JDK8 兼容方案

```java
// JDK8: 用 scheduledExecutor + complete 实现超时
public static <T> CompletableFuture<T> withTimeout(
        CompletableFuture<T> future, long timeout, TimeUnit unit,
        T defaultValue) {
    
    ScheduledFuture<?> timeoutFuture = scheduler.schedule(() -> {
        if (!future.isDone()) {
            future.complete(defaultValue);
        }
    }, timeout, unit);
    
    return future.whenComplete((result, ex) -> {
        timeoutFuture.cancel(false); // 完成后取消超时任务
    });
}
```

### 实际使用：超时 + 熔断降级

```java
public CompletableFuture<User> getUserWithTimeout(String userId) {
    return userClient.getAsync(userId)
        .orTimeout(3, TimeUnit.SECONDS)
        .exceptionally(ex -> {
            if (ex.getCause() instanceof TimeoutException) {
                log.warn("获取用户{}超时，执行降级", userId);
                return User.defaultUser(); // 超时降级
            }
            throw new CompletionException(ex); // 其他异常继续抛
        });
}
```

### 超时不能取消底层任务

```java
CompletableFuture<String> f = supplyAsync(() -> {
    sleep(10000);
    return "done";
}).orTimeout(2, TimeUnit.SECONDS);

f.exceptionally(ex -> "timeout");
// sleep(10000) 仍然会继续执行完！
// orTimeout 只影响 CompletableFuture 的完成状态，不中断执行线程
```

**解决方案**：使用 Future.cancel 或线程中断机制。

## 代码示例

### 多服务并行 + 全局超时

```java
public CompletableFuture<AggregateResult> fetchAll(String id) {
    CompletableFuture<A> fa = serviceA.get(id);
    CompletableFuture<B> fb = serviceB.get(id);
    CompletableFuture<C> fc = serviceC.get(id);

    return CompletableFuture.allOf(fa, fb, fc)
        .orTimeout(5, TimeUnit.SECONDS)
        .thenApply(v -> new AggregateResult(
            fallbackIfNotDone(fa, A::default),
            fallbackIfNotDone(fb, B::default),
            fallbackIfNotDone(fc, C::default)
        ));
}

private <T> T fallbackIfNotDone(CompletableFuture<T> f, Supplier<T> defaultSupplier) {
    try {
        return f.isDone() && !f.isCompletedExceptionally() 
            ? f.join() : defaultSupplier.get();
    } catch (Exception e) {
        return defaultSupplier.get();
    }
}
```

## 易错点与踩坑

### 1. JDK9+ orTimeout 是替代方案
```java
// ✅ JDK9+ 推荐写法
supplyAsync(() -> remoteCall())
    .orTimeout(3, TimeUnit.SECONDS)
    .exceptionally(ex -> {
        if (ex instanceof TimeoutException) {
            return "fallback";
        }
        return "error";
    });

// ❌ JDK8 方式（复杂，不推荐）
supplyAsync(() -> remoteCall())
    .thenApplyAsync(r -> r, delayedExecutor(3, SECONDS, scheduler));
```

### 2. orTimeout 不会取消底层任务
```java
// orTimeout 只让 CF 进入超时异常状态
// 底层的异步任务可能还在执行！
supplyAsync(() -> {
    Thread.sleep(5000); // 超时了但仍在执行
    return "result";
}).orTimeout(3, SECONDS)
.exceptionally(ex -> "fallback"); // 3秒后返回 "fallback"
// 但 5秒后的 "result" 仍在计算，浪费资源
```

### 3. completeOnTimeout 的"超时降级"语义
```java
// orTimeout → 超时抛 TimeoutException
// completeOnTimeout → 超时返回默认值（不抛异常）

supplyAsync(() -> remoteCall())
    .completeOnTimeout("默认值", 3, TimeUnit.SECONDS);
// 3秒内完成 → 返回真实结果
// 3秒未完成 → 返回 "默认值"（不进入异常状态）
```

### 4. JDK8 的超时实现方式
```java
// JDK8 没有原生的 CF 超时支持
// 常见方案：使用 ScheduledExecutorService + completeExceptionally
public static <T> CompletableFuture<T> withTimeout(
    CompletableFuture<T> future, long timeout, TimeUnit unit) {
    ScheduledFuture<?> timeoutFuture = scheduler.schedule(
        () -> future.completeExceptionally(new TimeoutException()),
        timeout, unit
    );
    return future.whenComplete((result, ex) -> timeoutFuture.cancel(false));
}
```
