# 读写锁降级

## 先说结论

- **锁降级**：持有写锁 → 获取读锁 → 释放写锁，将写锁降级为读锁
- **锁升级不支持**：读锁 → 写锁会导致死锁，ReentrantReadWriteLock明确禁止
- **核心价值**：保证写线程修改数据后能读取到最新值，避免被其他写线程抢占
- **典型场景**：缓存更新（先写后读确认）、数据一致性要求高的读写操作

---

## 核心结论

```
┌─────────────────────────────────────────────────────────────────┐
│                        锁降级流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1. 持有写锁        2. 获取读锁        3. 释放写锁               │
│   ┌───────┐         ┌───────┐         ┌───────┐                 │
│   │ Write │   →    │Read+  │   →    │ Read  │                 │
│   │ Lock  │        │Write  │        │ Lock  │                 │
│   └───────┘        └───────┘        └───────┘                 │
│                                                                  │
│   目的：写线程修改数据后，能继续读取最新数据                       │
│   好处：避免释放写锁后被其他写线程抢占，保证数据可见性             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                      不支持锁升级                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   读锁 → 写锁 ❌  会导致死锁                                     │
│                                                                  │
│   原因：多个读线程同时尝试升级，互相等待对方释放读锁               │
│                                                                  │
│   Thread A: 读锁 → 尝试获取写锁（等待B释放读锁）                 │
│   Thread B: 读锁 → 尝试获取写锁（等待A释放读锁）                 │
│   ↓                                                              │
│   死锁！                                                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 深度解析

### 1. 什么是锁降级

锁降级（Lock Downgrade）是指在持有写锁的情况下，先获取读锁，然后再释放写锁，从而将写锁降级为读锁的过程。

**标准流程**：
```
writeLock.lock();      // 1. 获取写锁
    ↓
// 修改共享数据
    ↓
readLock.lock();       // 2. 获取读锁（此时持有写锁+读锁）
    ↓
writeLock.unlock();    // 3. 释放写锁（降级完成，只持有读锁）
    ↓
// 读取数据验证
    ↓
readLock.unlock();     // 4. 释放读锁
```

### 2. 为什么需要锁降级

**场景问题**：
假设一个写线程修改了缓存数据，如果直接释放写锁：
1. 写锁释放后，其他写线程可能立即获取写锁并修改数据
2. 原写线程再读取时，可能读到的是被其他线程修改后的数据
3. 无法保证"修改后立即读取最新值"的语义

**锁降级的解决方案**：
```
写线程修改数据后：
  1. 先获取读锁（此时仍持有写锁，其他线程无法进入）
  2. 释放写锁（降级为读锁，其他读线程可以进入）
  3. 读取数据（保证读到的是自己修改的最新值）
  4. 释放读锁
```

**核心价值**：
- 保证数据修改后的可见性
- 避免写锁释放后被其他写线程抢占
- 提高并发度（降级后允许多个读线程并发）

### 3. 为什么不支持锁升级

**锁升级定义**：持有读锁的情况下，尝试获取写锁。

**死锁风险分析**：
```java
// 假设支持锁升级，两个线程同时执行：

// Thread A                    // Thread B
readLock.lock();               readLock.lock();
       ↓                              ↓
// 读取数据                     // 读取数据
       ↓                              ↓
writeLock.lock();  // 等待B释放读锁   writeLock.lock();  // 等待A释放读锁
       ↓                              ↓
// 死锁！互相等待对方释放读锁
```

**ReentrantReadWriteLock的设计**：
- 明确禁止锁升级
- 如果尝试在持有读锁时获取写锁，会导致死锁或无限等待
- 源码中`tryAcquire`方法会检查是否存在读锁占用

### 4. 锁降级的实现原理

**ReentrantReadWriteLock的状态设计**：
```
32位state变量（高16位读锁，低16位写锁）：
┌─────────────────┬─────────────────┐
│   读锁计数(16)   │   写锁计数(16)   │
│   (共享状态)     │   (独占状态)     │
└─────────────────┴─────────────────┘
```

**降级时的状态变化**：
```
状态1: 写锁=1, 读锁=0  (持有写锁)
状态2: 写锁=1, 读锁=1  (获取读锁)
状态3: 写锁=0, 读锁=1  (释放写锁，降级完成)
状态4: 写锁=0, 读锁=0  (释放读锁)
```

## 易错点

### ❌ 错误1：忘记先获取读锁就释放写锁
```java
// 错误示范
writeLock.lock();
try {
    data = newData;
    // 直接释放写锁，没有获取读锁
} finally {
    writeLock.unlock();  // ❌ 其他写线程可能立即抢占
}
// 此时读取可能读到其他线程修改的数据
```

### ❌ 错误2：尝试锁升级
```java
// 错误示范
readLock.lock();
try {
    // ... 读取操作
    writeLock.lock();  // ❌ 会导致死锁！不支持锁升级
    try {
        data = newData;
    } finally {
        writeLock.unlock();
    }
} finally {
    readLock.unlock();
}
```

### ❌ 错误3：降级后忘记释放读锁
```java
writeLock.lock();
try {
    data = newData;
    readLock.lock();  // 获取读锁
} finally {
    writeLock.unlock();  // 释放写锁
}
// ❌ 忘记释放读锁，导致内存泄漏！
```

### ✅ 正确做法
```java
writeLock.lock();
try {
    data = newData;
    readLock.lock();       // 1. 先获取读锁
} finally {
    writeLock.unlock();    // 2. 再释放写锁（降级）
}
try {
    // 3. 读取验证
    validate(data);
} finally {
    readLock.unlock();     // 4. 最后释放读锁
}
```

## 代码示例

### 示例1：缓存数据更新（典型锁降级场景）

```java
import java.util.concurrent.locks.ReentrantReadWriteLock;

public class CacheWithLockDowngrade<K, V> {
    private final ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock();
    private final ReentrantReadWriteLock.ReadLock readLock = rwLock.readLock();
    private final ReentrantReadWriteLock.WriteLock writeLock = rwLock.writeLock();
    
    private volatile V data;
    private volatile long version = 0;
    
    /**
     * 更新缓存数据并使用锁降级保证读取一致性
     */
    public V updateAndGet(V newData) {
        writeLock.lock();
        try {
            // 1. 修改数据
            this.data = newData;
            this.version++;
            System.out.println("[" + Thread.currentThread().getName() + "] 数据已更新，版本: " + version);
            
            // 2. 【关键】获取读锁后再释放写锁（锁降级）
            readLock.lock();
        } finally {
            // 3. 释放写锁，完成降级
            writeLock.unlock();
        }
        
        try {
            // 4. 读取最新数据验证（保证读到的是自己修改的值）
            V currentData = this.data;
            long currentVersion = this.version;
            System.out.println("[" + Thread.currentThread().getName() + "] 验证数据，版本: " + currentVersion);
            
            // 模拟数据验证
            validateData(currentData);
            
            return currentData;
        } finally {
            // 5. 释放读锁
            readLock.unlock();
        }
    }
    
    /**
     * 普通读操作
     */
    public V get() {
        readLock.lock();
        try {
            System.out.println("[" + Thread.currentThread().getName() + "] 读取数据，版本: " + version);
            return data;
        } finally {
            readLock.unlock();
        }
    }
    
    private void validateData(V data) {
        // 数据验证逻辑
        if (data == null) {
            throw new IllegalStateException("数据不能为空");
        }
    }
    
    // 测试
    public static void main(String[] args) throws InterruptedException {
        CacheWithLockDowngrade<String, String> cache = new CacheWithLockDowngrade<>();
        
        // 初始化数据
        cache.updateAndGet("初始数据");
        
        // 多线程测试
        Runnable writer = () -> {
            for (int i = 0; i < 3; i++) {
                cache.updateAndGet("数据-" + Thread.currentThread().getName() + "-" + i);
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        };
        
        Runnable reader = () -> {
            for (int i = 0; i < 5; i++) {
                cache.get();
                try {
                    Thread.sleep(50);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        };
        
        // 启动2个写线程和3个读线程
        for (int i = 0; i < 2; i++) {
            new Thread(writer, "Writer-" + i).start();
        }
        for (int i = 0; i < 3; i++) {
            new Thread(reader, "Reader-" + i).start();
        }
    }
}
```

### 示例2：数据一致性验证场景

```java
import java.util.concurrent.locks.ReentrantReadWriteLock;

public class DataValidator {
    private final ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock();
    private final ReentrantReadWriteLock.ReadLock readLock = rwLock.readLock();
    private final ReentrantReadWriteLock.WriteLock writeLock = rwLock.writeLock();
    
    private volatile Data data;
    
    static class Data {
        private final String content;
        private final long timestamp;
        
        public Data(String content) {
            this.content = content;
            this.timestamp = System.currentTimeMillis();
        }
        
        public boolean isValid() {
            return content != null && !content.isEmpty();
        }
        
        @Override
        public String toString() {
            return "Data{content='" + content + "', timestamp=" + timestamp + "}";
        }
    }
    
    /**
     * 更新数据并立即验证（使用锁降级保证一致性）
     */
    public boolean updateAndValidate(String newContent) {
        Data newData = new Data(newContent);
        
        writeLock.lock();
        try {
            // 写入新数据
            this.data = newData;
            System.out.println("写入数据: " + newData);
            
            // 【锁降级】获取读锁后再释放写锁
            readLock.lock();
        } finally {
            writeLock.unlock();  // 降级完成
        }
        
        try {
            // 此时持有读锁，可以安全读取验证
            Data current = this.data;
            boolean valid = current.isValid();
            System.out.println("验证结果: " + valid + ", 数据: " + current);
            return valid;
        } finally {
            readLock.unlock();
        }
    }
    
    /**
     * 对比：不使用锁降级的风险
     */
    public boolean updateWithoutDowngrade(String newContent) {
        Data newData = new Data(newContent);
        
        writeLock.lock();
        try {
            this.data = newData;
        } finally {
            writeLock.unlock();  // 直接释放，可能被其他写线程抢占
        }
        
        // ⚠️ 风险：此时可能被其他线程修改了data
        Data current = this.data;
        // 读取到的可能不是刚才写入的数据！
        return current.isValid();
    }
    
    public static void main(String[] args) {
        DataValidator validator = new DataValidator();
        
        // 使用锁降级
        System.out.println("=== 使用锁降级 ===");
        validator.updateAndValidate("有效数据1");
        validator.updateAndValidate("有效数据2");
        
        System.out.println("\n=== 不使用锁降级（存在风险） ===");
        validator.updateWithoutDowngrade("测试数据");
    }
}
```

### 示例3：读写锁状态追踪

```java
import java.util.concurrent.locks.ReentrantReadWriteLock;

public class LockStateDemo {
    private final ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock();
    private final ReentrantReadWriteLock.ReadLock readLock = rwLock.readLock();
    private final ReentrantReadWriteLock.WriteLock writeLock = rwLock.writeLock();
    
    public void demonstrateDowngrade() {
        System.out.println("=== 锁降级状态演示 ===\n");
        
        // 初始状态
        printLockState("初始状态");
        
        // 1. 获取写锁
        writeLock.lock();
        printLockState("获取写锁后");
        
        // 2. 获取读锁（此时同时持有写锁和读锁）
        readLock.lock();
        printLockState("获取读锁后（持有写+读锁）");
        
        // 3. 释放写锁（完成降级）
        writeLock.unlock();
        printLockState("释放写锁后（降级完成，只持有读锁）");
        
        // 4. 释放读锁
        readLock.unlock();
        printLockState("释放读锁后");
    }
    
    private void printLockState(String stage) {
        System.out.println(stage + ":");
        System.out.println("  写锁被持有: " + rwLock.isWriteLocked());
        System.out.println("  写锁持有线程数: " + rwLock.getWriteHoldCount());
        System.out.println("  读锁被持有: " + (rwLock.getReadLockCount() > 0));
        System.out.println("  读锁持有计数: " + rwLock.getReadLockCount());
        System.out.println();
    }
    
    public static void main(String[] args) {
        new LockStateDemo().demonstrateDowngrade();
    }
}
```
