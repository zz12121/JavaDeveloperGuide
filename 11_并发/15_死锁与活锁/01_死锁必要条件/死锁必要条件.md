
# 死锁必要条件

## 核心结论

死锁是指两个或多个线程互相持有对方需要的锁，导致所有线程永远阻塞。产生死锁必须**同时满足四个条件**（Coffman 条件）：互斥、占有并等待、不可抢占、循环等待。

## 深度解析

### 四个必要条件

| 条件 | 说明 | 示例 |
|------|------|------|
| **互斥（Mutual Exclusion）** | 资源一次只能被一个线程持有 | synchronized/Lock 同一时间只有一个线程持有 |
| **占有并等待（Hold and Wait）** | 持有锁的同时等待另一个锁 | 线程持有锁 A，同时请求锁 B |
| **不可抢占（No Preemption）** | 锁不能被强制释放，只能由持有者主动释放 | synchronized 不能被其他线程抢占 |
| **循环等待（Circular Wait）** | 存在线程的循环等待链 | T1 等待 T2，T2 等待 T1 |

### 经典死锁示例

```java
Object lockA = new Object();
Object lockB = new Object();

// Thread-1
synchronized (lockA) {
    Thread.sleep(100);
    synchronized (lockB) { // 等待 lockB
        // ...
    }
}

// Thread-2
synchronized (lockB) {
    Thread.sleep(100);
    synchronized (lockA) { // 等待 lockA → 死锁！
        // ...
    }
}
```

### 死锁图解

```
Thread-1: 持有 lockA → 等待 lockB ──┐
                                  │ 循环等待
Thread-2: 持有 lockB → 等待 lockA ──┘
```

### 破坏死锁

只要破坏四个条件中的**任意一个**，就能避免死锁：

| 破坏条件 | 方法 |
|---------|------|
| 互斥 | 无法破坏（锁的本质） |
| 占有并等待 | 一次性获取所有锁 |
| 不可抢占 | tryLock + 超时 |
| 循环等待 | 按固定顺序获取锁 |

## 易错点与踩坑

### 1. 误解四个条件的必要性

```java
// ❌ 误以为四个条件都需要满足才会死锁
// 实际上：四个条件必须同时满足才会死锁
// 破坏任意一个条件就能避免死锁

// ❌ 误以为互斥条件可以被破坏
// 互斥是锁的本质，无法破坏
// 所以死锁预防只能从其他三个条件入手
```

### 2. 认为"占有并等待"是必然的

```java
// ❌ 误以为"先获取锁再等待"是唯一模式
// 实际上：可以一次性获取所有需要的锁（破坏占有并等待）
// 或者完全不用锁（用无锁算法）

// ✅ 破坏占有并等待的方式：
// 1. 一次性获取所有锁
// 2. 先释放已持有的锁，再重新获取
```

### 3. 循环等待不等于死锁

```java
// ❌ 误以为有循环等待就一定死锁
// 实际上：需要所有线程都在等待其他线程持有的资源

// ✅ 正确的循环等待定义：
// T1 持有 R1，等待 R2
// T2 持有 R2，等待 R1
// ...形成闭环

// ⚠️ 如果 T1 持有 R1，等待 R2
//    T2 持有 R2，等待 R3（不是循环）
// 这不是死锁
```

### 4. 忽略"不可抢占"条件的打破方式

```java
// ❌ 误以为 synchronized 锁完全不可抢占
// 实际上：Lock 接口提供了 tryLock() 可以超时获取
// ✅ 破坏不可抢占的方式：
Lock lock = new ReentrantLock();
if (lock.tryLock(100, TimeUnit.MILLISECONDS)) {
    try {
        // 获取成功
    } finally {
        lock.unlock();
    }
} else {
    // 获取失败，可以做其他事情
}
```

### 5. 死锁必要条件的面试陷阱

```java
// ❌ 面试时问"如何破坏互斥条件"？
// 答：无法破坏，互斥是锁的本质

// ❌ 面试时问"为什么银行转账需要按顺序加锁"？
// 答：破坏循环等待条件，避免死锁

// ✅ 面试时问"四个条件中最容易破坏哪个"？
// 答：循环等待最容易破坏（按顺序获取锁）
```
