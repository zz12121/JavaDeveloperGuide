
# 饥饿

## Q1：什么是线程饥饿？

**A**：某个线程长期无法获得所需资源（CPU 时间、锁等），虽然线程是活的但没有进展。不像死锁那样完全停滞，而是极度缓慢或永远轮不到。

常见原因：高优先级线程持续抢占、非公平锁的插队机制、大量读线程导致写线程饥饿。

---

## Q2：如何避免饥饿？

**A**：

1. **使用公平锁**：`new ReentrantLock(true)`，按排队顺序获取锁
2. **合理设置线程优先级**：不要用极端的 1 和 10
3. **使用读写锁的公平模式**：避免写线程饥饿
4. **缩短持锁时间**：减少其他线程的等待时间

---

## Q3：非公平锁为什么可能导致饥饿？

**A**：非公平锁允许新到达的线程直接 CAS 尝试获取锁，如果成功就"插队"了。在高并发下，排队中的线程可能一直被新来的线程插队，导致长时间获取不到锁。

公平锁通过 `hasQueuedPredecessors()` 检查，有排队的线程就不插队。

---
```java
// 饥饿：非公平锁的插队导致低优先级线程长期等不到锁
ReentrantLock unfairLock = new ReentrantLock();  // 默认非公平

// 非公平锁允许新线程 CAS 抢锁（可能插队）
// 排队中的线程可能一直被插队 → 饥饿

// 解决：使用公平锁
ReentrantLock fairLock = new ReentrantLock(true);
// hasQueuedPredecessors() 检查是否有排队的线程
// 有排队 → 不插队，按 FIFO 顺序获取锁
```
